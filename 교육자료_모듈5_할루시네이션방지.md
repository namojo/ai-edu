## 1. 할루시네이션이란?

### 1.1 정의

**할루시네이션(Hallucination)** 이란 AI가 사실이 아닌 정보를 마치 사실인 것처럼 자신 있게 생성하는 현상입니다. AI는 "모르겠습니다"라고 말하는 대신, 그럴듯한 거짓말을 만들어냅니다.

> 🔑 핵심: AI는 "확률적으로 그럴듯한 다음 단어"를 예측하는 모델입니다. 사실 여부를 판단하는 것이 아니라, **그럴듯해 보이는 문장**을 생성하는 것이 본질입니다.

### 1.2 할루시네이션의 4가지 유형

| 유형 | 설명 | 위험도 | 실무 예시 |
|------|------|--------|----------|
| **사실 조작** | 존재하지 않는 사건, 인물, 데이터를 생성 | ⭐⭐⭐⭐⭐ | "삼성디스플레이가 2024년 IJP 양산 라인을 가동했다"(실제로는 미가동) |
| **출처 날조** | 존재하지 않는 논문, 판례, 보고서를 인용 | ⭐⭐⭐⭐⭐ | "대법원 2023다12345 판결에 따르면..."(존재하지 않는 판례) |
| **수치 왜곡** | 통계, 재무 데이터, 기술 사양을 잘못 제시 | ⭐⭐⭐⭐ | "IJP 공정 수율이 97%에 달한다"(실제 85~91%) |
| **법률 인용 오류** | 법 조항 번호, 내용을 잘못 인용하거나 혼합 | ⭐⭐⭐⭐⭐ | "산업안전보건법 제42조에 따르면..."(실제 내용과 상이) |

### 1.3 발생 원인

1. **통계적 언어 모델의 본질적 한계**: LLM은 "다음에 올 확률이 높은 토큰"을 예측합니다. 정답을 검색하는 것이 아니라, 패턴을 생성하는 것입니다.
2. **학습 데이터의 편향과 오류**: 인터넷의 부정확한 정보가 학습 데이터에 포함되어 있습니다.
3. **지식 컷오프**: 학습 시점 이후의 법률 개정, 최신 데이터를 반영하지 못합니다.
4. **과잉 확신(Overconfidence)**: 확률이 낮은 출력에도 확신 있는 어투를 사용합니다.
5. **맥락 혼동**: 긴 대화나 복잡한 질문에서 앞뒤 맥락을 혼합합니다.

> 💡 팁: 2025년 수학적 증명을 통해 "현재 LLM 아키텍처에서 할루시네이션은 완전히 제거할 수 없는 본질적 특성"임이 확인되었습니다. 따라서 **검증 프로세스**를 갖추는 것이 유일한 해법입니다.

## 2. 실제 사고 사례

> ⚠ 주의: 아래 사례들은 "AI를 안 쓰면 된다"는 교훈이 아닙니다. "AI를 쓰되, 반드시 검증해야 한다"는 교훈입니다.

### 2.1 법률 분야 — 가짜 판례 제출 사건

**사건 개요**: 2023년 뉴욕의 변호사 Steven Schwartz는 ChatGPT가 생성한 6건의 가짜 판례를 연방법원에 제출했습니다. AI가 만들어낸 판례에는 실존하지 않는 사건명, 판결문 인용, 법원명이 포함되어 있었습니다.

**결과**: 법원은 변호사에게 5,000달러의 제재금을 부과했습니다.

**2025~2026년 확산**: 이 문제는 줄어들기는커녕 급증하고 있습니다. 2026년 기준 전 세계 법원에 제출된 AI 날조 인용 사례가 1,200건 이상으로 집계되었으며, 2026년 1분기에만 미국 법원이 부과한 제재금이 14만 5천 달러를 넘었습니다. 6차 순회 항소법원은 테네시주 변호사들이 3건의 항소에서 20개 이상의 가짜 인용을 제출한 사건에 대해 역대 최고 수준의 제재를 부과했습니다.

| 연도 | 주요 사건 | 제재 내용 |
|------|----------|----------|
| 2023 | Mata v. Avianca (뉴욕) | $5,000 제재금 |
| 2025 | Noland v. Land of the Free | 23개 중 21개 인용이 날조로 판명 |
| 2026 | 6th Circuit 통합 항소 | 20개 이상 가짜 인용, 역대 최고 제재 |

> 🔑 핵심: 하루에 5~6건의 새로운 AI 날조 인용 사례가 발견되고 있습니다. "AI가 만든 법률 정보는 100% 검증 대상"이라는 원칙을 세우세요.

### 2.2 금융 분야 — 잘못된 금융 데이터와 규제 인용

2026년 1분기, AI 금융 분석 도구의 할루시네이션으로 인한 잘못된 실적 전망이 업계 전반에 영향을 미쳤습니다. FINRA(미국 금융산업규제기관)는 2026년 감독 보고서에서 AI 할루시네이션과 편향을 주요 리스크로 지목하며, 증권사들에게 AI 할루시네이션 탐지 절차를 의무화할 것을 권고했습니다.

**실무에서 발생하는 금융 할루시네이션 유형:**
- AI가 존재하지 않는 금융 상품의 약관을 생성
- 잘못된 이자율, 수수료율을 제시
- 폐지된 규정을 현행 법규처럼 인용
- 타사 상품의 조건을 혼합하여 제시

> ⚠ 주의: 금융 규제 위반은 과징금, 영업정지, 형사처벌로 이어질 수 있습니다. AI가 제시한 금융 법규는 **금융감독원 법규 검색 시스템**이나 **법제처 국가법령정보센터**에서 반드시 원문을 확인하세요.

### 2.3 의료 분야 — 건강 정보 오류

2025년 연구에 따르면, LLM이 의료 질의에 대해 할루시네이션을 일으킨 비율이 짧은 질의에서 67.6%, 긴 질의에서 64.1%에 달했습니다. ECRI(글로벌 의료안전 비영리기관)는 AI 리스크를 2025년 의료기술 위험요소 1위로 선정했습니다.

**위험 사례:**
- AI가 잘못된 약물 복용량을 제시
- 존재하지 않는 의학 저널 논문을 인용
- 질병 간 연관성을 잘못 생성

### 2.4 제조 분야 — 잘못된 공정 데이터와 안전 기준

**시나리오**: AI에게 "OLED IJP 공정에서 사용하는 용제의 허용 노출 기준"을 물었을 때, AI가 실제와 다른 수치를 제시하는 경우가 발생합니다.

**위험 사례:**
- AI가 허용 농도 기준을 10배 높게 제시 → 작업자 건강 위험
- 존재하지 않는 안전 인증 규격을 인용 → 수출 통관 실패
- 잘못된 장비 사양을 제시 → 설비 호환성 문제 발생
- 폐지된 안전 규정을 현행으로 제시 → 안전 감사 부적합

> 💡 팁: 제조 현장에서 AI가 제시한 안전 기준이나 공정 조건은 **반드시 안전관리부서와 교차 검증**하세요. 잘못된 수치 하나가 현장 사고로 이어질 수 있습니다.

## 3. AI가 자신 있게 거짓을 말하는 메커니즘

### 3.1 통계적 언어 모델의 작동 원리

```
[사람의 이해 방식]
"산업안전보건법 제42조" → 법령 DB에서 해당 조항 검색 → 원문 확인

[AI의 작동 방식]
"산업안전보건법 제" → "42" (학습 데이터에서 자주 등장한 숫자)
"42조에 따르면" → "유해화학물질을..." (통계적으로 그럴듯한 연결)
→ 실제 42조의 내용과 무관하게, "그럴듯한 문장"이 완성됨
```

AI는 **검색 엔진이 아닙니다**. 데이터베이스에서 정보를 찾아오는 것이 아니라, "이 맥락 다음에 올 확률이 가장 높은 단어"를 순서대로 생성합니다. 그래서 조항 번호, 수치, 날짜처럼 **구체적인 팩트**에서 오류가 발생합니다.

### 3.2 학습 데이터 편향의 영향

| 편향 유형 | 설명 | 결과 |
|----------|------|------|
| 빈도 편향 | 자주 등장한 정보를 우선 생성 | 인기 있는(하지만 부정확한) 통설을 반복 |
| 시간 편향 | 학습 데이터 시점의 정보만 반영 | 개정된 법률의 이전 버전을 인용 |
| 출처 편향 | 블로그·뉴스의 해석을 원문처럼 제시 | 2차 자료를 1차 출처로 오인 |
| 혼합 편향 | 여러 출처의 정보를 합성 | A법의 조항 번호 + B법의 내용이 결합 |

### 3.3 할루시네이션 경고 신호

다음과 같은 패턴이 나타나면 할루시네이션을 의심하세요:

1. **지나치게 구체적인 숫자**: "정확히 94.7%의 효율 향상" — 출처 없이 소수점까지 제시
2. **완벽한 형식의 인용**: "대법원 2024다56789 판결문" — 형식은 완벽하지만 실존하지 않음
3. **단정적 어투**: "반드시 ~해야 합니다", "법적으로 의무입니다" — 검증 없는 단정
4. **맥락과 동떨어진 정보**: 질문과 관련 없는 세부 정보를 장황하게 서술
5. **최신 데이터 주장**: "2026년 최신 통계에 따르면..." — 학습 데이터 범위 밖의 시점

## 4. 팩트체크 3단계

### Step 1: AI 출처가 실제 존재하는지 확인

```
[확인 방법]
1. AI가 인용한 법령 → 국가법령정보센터(law.go.kr)에서 검색
2. AI가 인용한 논문 → Google Scholar, RISS에서 검색
3. AI가 인용한 판례 → 대법원 종합법률정보에서 검색
4. AI가 인용한 통계 → 원본 기관(통계청, 금감원 등) 사이트에서 검색
5. AI가 인용한 뉴스 → 해당 언론사 사이트에서 직접 검색
```

> 💡 팁: Perplexity는 모든 주장에 인라인 출처 링크를 제공합니다. 출처 링크를 클릭하여 실제 페이지가 존재하는지부터 확인하세요. 링크가 404 오류를 반환하면 날조된 출처입니다.

### Step 2: 원문을 찾아서 AI 요약과 대조

```
[대조 체크리스트]
□ 조항 번호가 정확한가?
□ 조항의 내용이 AI 요약과 일치하는가?
□ 수치(금액, 비율, 기간)가 원문과 같은가?
□ 적용 대상(업종, 규모, 지역)이 올바른가?
□ 시행일과 개정 이력이 최신인가?
```

**실전 프롬프트 — Claude로 원문 대조**

```
[법령 원문 대조 프롬프트]

아래 두 텍스트를 비교하여 차이점을 찾아주세요.

[텍스트 A — AI가 생성한 법규 요약]
(여기에 AI가 생성한 내용을 붙여넣기)

[텍스트 B — 법령 원문]
(여기에 law.go.kr에서 복사한 원문을 붙여넣기)

비교 항목:
1. 조항 번호의 정확성
2. 내용의 일치 여부 (누락, 추가, 왜곡)
3. 수치(금액, 기간, 비율)의 일치 여부
4. 적용 범위의 정확성

차이가 있는 부분을 표로 정리하고, 각 차이의 심각도를
"치명적 / 중요 / 경미"로 분류해주세요.
```

### Step 3: 전문가/법무팀 최종 검증

| 검증 대상 | 검증 주체 | 검증 방법 |
|----------|----------|----------|
| 법률·규제 조항 | 법무팀/컴플라이언스팀 | 원문 대조 + 최신 개정 확인 |
| 재무 데이터 | 재무팀/회계팀 | 공시 자료·감사보고서 대조 |
| 기술 사양/공정 데이터 | 기술팀/엔지니어 | 장비 매뉴얼·실측 데이터 대조 |
| 안전 기준 | 안전관리팀 | 안전보건공단 기준 대조 |
| 보험 약관 | 보험 전문가/계리사 | 실제 약관 원문 대조 |

> ⚠ 주의: "AI가 확인해줬으니 맞겠지"는 가장 위험한 판단입니다. AI의 자신감 있는 어투에 속지 마세요. **최종 책임은 항상 사람에게 있습니다.**

## 5. 보고서 작성 시 할루시네이션 제거 체크리스트

보고서를 AI로 작성한 후, 제출 전에 아래 10개 항목을 반드시 점검하세요.

| # | 점검 항목 | 점검 방법 | 통과 기준 |
|---|----------|----------|----------|
| 1 | **인용된 법령·규정이 실존하는가?** | 국가법령정보센터에서 조항 번호 검색 | 원문 확인 완료 |
| 2 | **수치 데이터의 출처가 명시되어 있는가?** | 각 수치 옆에 (출처: ○○, 20XX) 표기 확인 | 모든 수치에 출처 존재 |
| 3 | **통계 수치가 원본과 일치하는가?** | 원본 보고서/DB에서 동일 수치 확인 | 오차 없음 |
| 4 | **인용된 논문/보고서가 실존하는가?** | Google Scholar, 해당 기관 사이트에서 검색 | 실제 문서 확인 |
| 5 | **법령의 시행일과 개정 이력이 최신인가?** | 법령 시행일 확인, 최근 개정 여부 확인 | 현행 법령 기준 |
| 6 | **AI가 단정적으로 서술한 부분이 있는가?** | "반드시", "의무", "위반 시" 등 단정 표현 검색 | 전문가 확인 완료 |
| 7 | **경쟁사/타사 정보가 정확한가?** | 공시자료, 공식 발표 자료로 대조 | 공식 출처 확인 |
| 8 | **기술 사양이 실측 데이터와 일치하는가?** | 장비 매뉴얼, 실험 결과와 대조 | 사내 데이터 기준 |
| 9 | **날짜·일정이 현실적인가?** | 프로젝트 일정표, 캘린더와 대조 | 물리적으로 가능한 일정 |
| 10 | **면책 문구가 포함되어 있는가?** | "본 보고서의 법률 정보는 참고용이며..." 문구 확인 | 면책 문구 삽입 완료 |

> 💡 팁: 이 체크리스트를 프롬프트에 포함시키면 AI가 자체적으로 점검한 결과를 함께 제공합니다. 하지만 **AI의 자체 점검만으로는 부족**합니다. 사람이 직접 확인하는 단계가 반드시 필요합니다.

**상황**: 신규 화학물질 사용 시 관련 법규를 AI로 조사

## 8. 기술적 대응 방법

### 8.1 RAG (Retrieval-Augmented Generation, 검색 증강 생성)

AI가 답변을 생성하기 전에, 관련 문서를 **검색**하여 참고 자료로 제공하는 방식입니다.

```
[RAG 없이 질문]
사용자: "화학물질관리법에서 유기 용제 취급 기준은?"
AI: (학습 데이터에서 추론) → 할루시네이션 위험 높음

[RAG 적용]
사용자: "화학물질관리법에서 유기 용제 취급 기준은?"
시스템: (법령 DB에서 관련 조항 검색 → AI에게 원문 전달)
AI: (원문을 참고하여 답변 생성) → 정확도 대폭 향상
```

**사내 적용**: 사내 LLM에 RAG를 구축하면, 사내 규정집·매뉴얼·기술 문서를 참조하여 답변을 생성할 수 있습니다.

### 8.2 Grounding (근거 기반 생성)

AI가 생성한 각 문장에 대해 **근거 문서의 위치**를 표시하는 방식입니다. Google의 Gemini API에서 "Grounding with Google Search" 기능으로 제공됩니다.

### 8.3 출처 표시 활용 — Perplexity 인라인 인용

Perplexity는 모든 주요 주장에 대해 출처를 인라인으로 표시합니다. 법규·수치·사실관계를 검색할 때 가장 먼저 활용하기 좋은 도구입니다.

```
[Perplexity 활용 프롬프트]
IJP OLED 공정에서 사용하는 유기 용제의 산업안전보건법상 규제 사항을
정리해주세요. 각 조항에 대해 원문 확인이 가능한 출처 링크를 포함해주세요.
```

### 8.4 실무 활용 조합

| 상황 | 추천 접근법 |
|------|-----------|
| 법규 조사 초안 | Perplexity (출처 링크 확인) → 법령 원문 직접 대조 |
| 사내 기술 문서 기반 보고서 | 사내 LLM + RAG (사내 DB 연동) |
| 금융 상품 비교 분석 | AI로 프레임워크 생성 → 공시 자료로 수치 직접 입력 |
| 기술 동향 리서치 | ChatGPT Deep Research 또는 Gemini Deep Research → 출처 교차 확인 |

(C) 다른 AI 모델에게 같은 질문을 해서 답변을 비교한다
(D) 판례 번호 형식이 올바른지 확인한다

**정답**: (B) — AI에게 재질문하면 같은 할루시네이션을 반복하거나 새로운 거짓을 생성할 수 있습니다. 원본 소스(대법원 종합법률정보)에서 직접 확인하는 것이 팩트체크 1단계의 핵심입니다.

### 퀴즈 2 — 팩트체크 우선순위

Q. AI가 작성한 보고서 초안에서 다음 중 **팩트체크 우선순위가 가장 높은** 항목은?

(A) "최근 디스플레이 시장이 성장하고 있습니다" — 일반적 트렌드 서술

**정답**: (B) — 구체적인 조항 번호와 의무 사항이 명시된 법령 인용은 할루시네이션 위험이 가장 높고, 오류 시 법적 리스크가 가장 큽니다. 숫자, 조항, 의무 표현이 포함된 문장일수록 우선 검증하세요.
