# 모듈 4: 리서치와 자료 검색 — "AI로 전문 자료를 빠르고 정확하게 찾는 법"

> **대상**: 제조업 및 금융권 직장인
> **학습 시간**: 약 35분
> **핵심 키워드**: AI 검색 전략, 교차 검증, 법률/규제 검색, Perplexity, Deep Research, 할루시네이션 방지

---

## 학습 목표

1. AI 검색 도구별 강점과 약점을 이해하고 목적에 맞게 선택할 수 있다
2. 제조업/금융업 특화 리서치 전략을 수립하고 실행할 수 있다
3. 법률/규제 검색 시 AI 검색 → 원문 확인 → 전문가 검증의 3단계를 적용할 수 있다
4. 교차 검증 워크플로로 AI 검색 결과의 신뢰도를 확보할 수 있다

---

## 1. AI 검색 전략 — 도구별 강점/약점 비교

### 왜 도구 선택이 중요한가?

전문 자료를 AI로 검색할 때, **어떤 도구를 쓰느냐에 따라 결과의 품질과 신뢰도가 크게 달라집니다.** 모든 AI가 검색에 강한 것은 아닙니다. 각 도구의 특성을 이해하고 목적에 맞게 골라 써야 합니다.

> 🔑 **핵심**: AI 검색은 "시작점"이지 "결론"이 아닙니다. 특히 법률, 수치, 조항 번호는 반드시 원문과 대조하세요.

### AI 검색 도구 비교표 (2025~2026년 기준)

| 도구 | 핵심 강점 | 검색 전략 | 한계 | 비용 |
|------|----------|----------|------|------|
| **Perplexity** | 실시간 웹 검색 + 인라인 출처 링크, SimpleQA 정확도 93.9% | 최신 판례, 사고사례, 해외 규제 검색. "출처를 표시하고 원문 링크를 포함해줘" 명시 | 법령 원문보다 해석이 섞일 수 있음. 긴 문서 분석은 약함 | 무료/Pro $20/월 |
| **ChatGPT Deep Research** | o3 모델 기반 멀티스텝 리서치, 수백 개 소스 자동 분석 | 종합 기술 동향, 시장 조사, 해외 사례 심층 분석 | 5~30분 소요. 실시간 속보에는 Perplexity보다 느림 | Plus $20/월 이상 |
| **Claude Research** | 261개 이상 소스 고속 처리, 긴 문서 정밀 분석, 복잡한 조건 준수 | 법령 전문을 직접 붙여넣고 핵심 조항 추출. 긴 계약서/약관 분석 | Max($100/월) 이상 필요. 웹 검색은 Research 모드 한정 | Pro $20/월, Max $100/월 |
| **Gemini Deep Research** | Google 검색 직접 연동, 최신 뉴스/보도자료 강점, Google 생태계 통합 | 최신 규제 변경 사항, 국내외 사고 보도, Google Scholar 논문 탐색 | 긴 법령 문서 상세 분석보다 요약에 집중 | Advanced $20/월 |

### 상황별 도구 선택 가이드

| 검색 목적 | 1순위 도구 | 2순위 도구 | 이유 |
|----------|----------|----------|------|
| 최신 뉴스/사고 사례 | Perplexity | Gemini | 실시간 웹 검색 + 출처 링크 |
| 심층 기술 보고서 작성 | ChatGPT Deep Research | Claude Research | 수백 개 소스 자동 종합 분석 |
| 법령 원문 정밀 분석 | Claude | ChatGPT | 긴 문서 분석 + 조건 준수 능력 |
| 특허/학술 논문 탐색 | Gemini | Perplexity | Google Scholar 연동 |
| 빠른 팩트체크 | Perplexity | Gemini | 출처 링크로 즉시 원문 확인 |
| 보험 약관/계약서 비교 | Claude | ChatGPT | 긴 문서 비교 분석에 최적화 |

💡 **팁**: 하나의 도구만 고집하지 마세요. **목적에 따라 2~3개 도구를 조합**하면 검색 품질과 신뢰도가 크게 올라갑니다. 예를 들어 "Perplexity로 최신 사례 수집 → Claude로 법령 정밀 분석 → 원문 대조"가 효과적입니다.

## 2. 금융업 자료 빠르게 찾기

### 2-1. 금감원 공시(DART) 검색

금융권에서 경쟁사 분석, 시장 동향 파악 시 **DART(전자공시시스템)** 자료가 필수입니다. AI를 활용하면 방대한 공시 자료를 빠르게 요약하고 비교할 수 있습니다.

**프롬프트 예시 — DART 공시 분석 요청**:

```
당신은 금융 애널리스트입니다.

아래는 [보험회사명]의 2024년 4분기 사업보고서에서 발췌한
보험 수익 현황 테이블입니다.

(DART에서 복사한 표 데이터를 여기에 붙여넣기)

요청:
1. 전년 동기 대비 주요 지표 변동 분석 (표 형식)
2. 손해율·사업비율 추이에서 주목할 점 3가지
3. 경영진 보고용 1문단 요약

조건: 수치는 원 데이터를 그대로 인용하고, AI 추론 부분은 "[분석]"으로 표시
```

> ⚠ **주의**: DART 공시 데이터를 AI에 붙여넣을 때, **민감한 비공개 정보가 포함되어 있지 않은지 반드시 확인**하세요. 공개된 공시 자료만 외부 AI에 입력해야 합니다.

### 2-2. 보험업법, IFRS17, 바젤 규제 검색

보험/금융 규제는 자주 개정되므로 **최신 버전 확인**이 필수입니다.

**프롬프트 예시 — 규제 검색**:

```
당신은 보험 규제 전문가입니다.

IFRS17(보험계약 국제회계기준) 도입 이후 보험회사의 책임준비금
산출 방식이 어떻게 변경되었는지 정리해주세요.

포함할 내용:
1. IFRS17 핵심 변경사항 (기존 IFRS4 대비, 표 형식)
2. 보험부채 측정 방식의 주요 차이점
3. 국내 보험사에 미치는 영향 3가지
4. 관련 금감원 가이드라인 요약

⚠ 중요: 확실하지 않은 내용은 "확인 필요"로 표시하고,
참고 가능한 원문 출처(법령, 금감원 보도자료 등)를 함께 안내해주세요.
```

### 2-3. 보험 약관 해석 및 상품 비교

보험 약관은 수십~수백 페이지에 달하는 장문 문서입니다. Claude와 같이 **긴 문서 처리에 강한 AI**를 활용하면 효율적입니다.

```
아래는 [상품명] 보험 약관 전문입니다. (PDF 파일 첨부 또는 전문 붙여넣기)

요청:
1. 보장 범위 요약 (일반암/소액암/유사암 구분)
2. 면책 사항 중 고객 민원이 발생하기 쉬운 조항 3가지
3. 경쟁사 [상품명] 약관과 비교 시 차별점
4. 약관 해석이 모호한 부분 표시

조건: 약관 원문의 조항 번호를 반드시 인용
```

## 3. 제조업 리서치

### 3-1. 기술 동향 조사 (특허, 학술 논문)

제조업에서 신규 기술 도입이나 R&D 방향 설정 시 **특허 및 논문 동향 조사**가 핵심입니다.

**프롬프트 예시 — 기술 동향 리서치**:

```
당신은 디스플레이 기술 리서치 애널리스트입니다.

주제: 잉크젯 프린팅(IJP) 방식의 OLED 제조 기술 동향
조사 범위: 2023~2025년

아래 항목을 조사하여 기술 동향 보고서를 작성해주세요:
1. 주요 업체별 기술 발표 현황 (삼성, LG, BOE, JOLED 등)
2. 핵심 기술 과제와 해결 현황 (표 형식)
   - 잉크 재료, 노즐 정밀도, 대면적 균일성, 수율
3. 최근 등록/공개 특허 동향 (주요 출원인, 기술 분류)
4. 양산 적용 시점 전망 및 근거
5. 우리 회사 시사점 3가지

출처를 반드시 표시하고, 확인되지 않은 정보는 "[미확인]"으로 표시
```

> 💡 **팁**: 특허 검색은 AI 검색 결과만으로 완결하지 마세요. **KIPRIS(한국 특허), Google Patents, Espacenet** 등에서 직접 원문을 확인해야 합니다. AI는 특허 번호를 잘못 생성하는 경우가 많습니다.

### 3-2. 공정 사고 사례 검색

안전 관련 보고서 작성 시 **유사 업종의 사고 사례**를 참고하면 리스크 평가의 설득력이 높아집니다.

```
잉크젯 프린팅 또는 유기 용제 취급 관련 공정에서 발생한
산업 사고 사례를 최근 5년간 조사해주세요.

각 사고에 대해:
1. 발생 일시 및 장소
2. 사고 유형 (화재, 폭발, 누출, 중독 등)
3. 원인 분석
4. 인명/재산 피해 규모
5. 재발 방지 대책
6. 출처 (뉴스 기사, 안전보건공단 자료 등)

표 형식으로 정리하고, 국내 사례와 해외 사례를 구분해주세요.
⚠ 확인되지 않은 사고 사례는 절대 포함하지 마세요.
```

### 3-3. 경쟁사 동향 분석

```
당신은 산업 분석 전문가입니다.

주제: [경쟁사명]의 최근 1년간 OLED 사업 전략 분석

아래 항목을 조사해주세요:
1. 최근 투자 발표 및 신규 라인 계획
2. 기술 방향 (증착 vs IJP vs 하이브리드)
3. 주요 고객사/공급 현황 변화
4. 실적 추이 (매출, 영업이익률)
5. 우리 회사 대비 강점/약점

출처를 반드시 표시하고, 뉴스/공시/IR자료 기반으로 작성
추측성 내용은 "[추정]"으로 명확히 구분
```

## 4. 법률/규제 검색법

### 4-1. 제조업 관련 법규

제조업 보고서에서 자주 참고하는 법규와 AI 검색 전략입니다.

| 법률 | 주요 검색 포인트 | AI 활용 팁 |
|------|----------------|-----------|
| **산업안전보건법** | 유해물질 취급 기준, 작업환경측정, PSM(공정안전관리) | 조항 번호 검증 필수. 시행규칙까지 확인 |
| **화학물질관리법** | 유독물질 분류, 취급시설 기준, 화학사고 예방 | 물질별 규제 수준이 다르므로 CAS 번호로 검색 |
| **전기사업법** | 자가용 전기설비 검사, 안전관리 | 시행령 개정이 잦으므로 최신 버전 확인 |
| **환경영향평가법** | 신규 설비 설치 시 환경 영향 평가 절차 | 사업 규모 기준이 수시로 변경됨 |

### 4-2. 금융업 관련 법규

| 법률 | 주요 검색 포인트 | AI 활용 팁 |
|------|----------------|-----------|
| **보험업법** | 상품 인가 요건, 보험모집 규제, 공시 의무 | 시행령과 감독규정까지 3단계로 확인 |
| **자본시장법** | 금융투자상품 규제, 불공정거래 금지 | 금융위원회 유권해석까지 확인 필요 |
| **금융소비자보호법** | 적합성 원칙, 적정성 원칙, 설명 의무, 불완전판매 | 2021년 시행 이후 판례 축적 중 |
| **개인정보보호법** | 고객 데이터 수집/처리/이전 기준 | AI에 고객 데이터 입력 시 이 법률 적용 |

### 4-3. 법규 검색 3단계 — AI 검색 → 원문 확인 → 전문가 검증

법률/규제 정보는 AI가 가장 많이 **할루시네이션을 일으키는 영역**입니다. 반드시 3단계 검증을 거쳐야 합니다.

```
[3단계 검증 프로세스]

Step 1: AI 검색 (시작점)
→ Perplexity/ChatGPT로 관련 법규 초안 수집
→ "출처 링크를 반드시 포함해줘" 명시
→ 확실하지 않은 부분은 "확인 필요"로 표시 요청

Step 2: 원문 확인 (검증)
→ 국가법령정보센터(law.go.kr)에서 조항 원문 대조
→ 금융감독원(fss.or.kr), 한국산업안전보건공단 등에서 세부 기준 확인
→ AI 요약과 원문 내용이 일치하는지 항목별 체크

Step 3: 전문가 검증 (최종)
→ 법무팀, 컴플라이언스팀, 안전환경팀에 최종 확인 요청
→ 보고서에 "검증 완료/미완료" 상태를 명시
→ 중요한 의사결정에 영향을 미치는 조항은 반드시 전문가 확인 후 반영
```

> ⚠ **주의**: AI가 "산업안전보건법 제42조에 따르면..."이라고 구체적으로 인용해도, **실제로 해당 조항 내용이 전혀 다를 수 있습니다.** 구체적 숫자(조항 번호, 횟수, 기간, 금액)가 나올수록 오히려 더 의심하고 검증 우선순위를 높이세요.

## 5. Before/After 예시 1 — 신규 화학물질 사용 관련 법규 검색 (제조업)

**상황**: OLED 잉크젯 프린팅 공정에서 사용할 신규 유기 용제(솔벤트)에 대한 법적 규제 사항을 조사해야 합니다.

**Before — 막연한 검색 요청**

```
OLED 공정 유기 용제 관련 법률 알려줘.
```

**AI 결과:**
> 유기 용제를 사용하는 공정에서는 산업안전보건법과 화학물질관리법을 준수해야 합니다. 산업안전보건법에서는 유해물질 취급 시 작업환경측정과 건강검진을 의무화하고 있으며...

→ **문제점**: 일반적인 설명만 나열. 구체적 조항 없음. 어떤 규제가 우리 공정에 적용되는지 알 수 없음.

**After — 3단계 검증 워크플로 적용**

**Step 1: Perplexity로 관련 법규 초안 수집**

```
잉크젯 프린팅 OLED 공정에서 사용하는 유기 용제(솔벤트, 예: 시클로헥사논,
아니솔 등)와 관련하여, 산업안전보건법과 화학물질관리법에서 확인해야 할
핵심 규제 조항을 정리해주세요.

각 조항에 대해 표 형식으로 작성:
- 법령명 및 정확한 조항 번호
- 핵심 규제 내용 (50자 이내)
- 잉크젯 공정에 미치는 구체적 영향
- 원문 확인 가능한 URL

⚠ 확실하지 않은 조항은 "확인 필요"로 반드시 표시해주세요.
출처가 없는 정보는 포함하지 마세요.
```

**Step 2: Claude로 법령 원문 정밀 분석**

```
아래는 국가법령정보센터(law.go.kr)에서 복사한 화학물질관리법 원문입니다.

(법령 원문을 여기에 붙여넣기)

위 법령 원문에서 잉크젯 프린팅 공정의 유기 용제 취급과 관련된
조항을 모두 찾아 다음 형식으로 정리해주세요:
- 정확한 조항 번호와 제목
- 해당 조항의 핵심 내용 요약 (150자 이내)
- 우리 공정 도입 시 준수해야 할 구체적 사항
- 위반 시 제재 내용 (벌금, 영업정지 등)

⚠ 법령 원문에 명시되지 않은 내용은 절대 추가하지 마세요.
원문에 근거한 내용만 작성하세요.
```

**Step 3: 검증 결과 정리**

```
[검증 결과 요약]
AI 1차 검색 결과: 관련 조항 5개 제시
→ 원문 대조 결과: 4개 정확, 1개 조항 번호 오류 식별 및 수정
→ 검증된 4개 조항만 보고서 반영
→ 1개는 법무팀 확인 요청 후 최종 반영 예정

총 소요 시간: AI 검색 5분 + 원문 대조 15분 = 20분
(기존 수동 조사 대비: 3~4시간 → 20분으로 단축)
```

→ **개선 효과**: 구체적 조항까지 빠르게 파악하면서도, 3단계 검증으로 정확성 확보.

| 개선 요소 | 적용 내용 | 효과 |
|----------|----------|------|
| 물질 특정 | 시클로헥사논, 아니솔 등 구체적 물질명 | 범용적 답변이 아닌 우리 공정 맞춤 결과 |
| 출처 요구 | URL 포함 요청 | 원문 대조 즉시 가능 |
| 불확실성 표시 | "확인 필요" 태그 요청 | AI의 자체 불확실성 인지 활용 |
| 도구 분리 | Perplexity(검색) → Claude(분석) | 각 도구의 강점 극대화 |

## 6. Before/After 예시 2 — 보험 상품 리스크 관련 규제 검토 (금융업)

**상황**: 신규 실손보험 상품 출시 전, 금융소비자보호법 준수 여부를 검토해야 합니다.

**Before — 단순 질문**

```
실손보험 출시할 때 지켜야 할 법률이 뭐야?
```

**AI 결과:**
> 실손보험 상품 출시 시 보험업법, 금융소비자보호법, 개인정보보호법 등을 준수해야 합니다. 보험업법에서는 상품 신고 절차를 규정하고 있으며...

→ **문제점**: 법률 이름 나열 수준. 구체적 체크리스트 없음. 실무 적용 불가.

**After — 역할+구조+검증 요청 결합**

```
당신은 보험 컴플라이언스 전문가입니다.

상황: 우리 회사가 4세대 실손보험 상품을 신규 출시하려 합니다.
출시 전 금융소비자보호법 준수 여부를 검토해야 합니다.

아래 구조로 규제 준수 체크리스트를 작성해주세요:

1. 적용 법규 매핑
   - 법률 → 시행령 → 감독규정 → 세부 지침 단계별 정리
   - 금융소비자보호법, 보험업법, 관련 금감원 가이드라인 포함

2. 준수 항목 체크리스트 (표 형식)
   - 항목명
   - 관련 조항 (법률명 + 조항 번호)
   - 준수 기준 요약
   - 현재 상태 (준수/미확인/미준수)
   - 비고

3. 특히 주의해야 할 사항
   - 설명 의무 관련 (불완전판매 리스크)
   - 약관 명확성 기준
   - 고객 정보 활용 범위

4. 검증 필요 사항
   - 위 체크리스트 중 원문 대조가 필요한 항목 표시
   - 법무팀/컴플라이언스팀 확인 필요 항목 별도 표시

⚠ AI가 제시하는 조항 번호는 검증 전이므로 "검증 필요"를 기본 표시하고,
확신할 수 있는 조항만 "확인됨"으로 표시해주세요.
```

→ **개선 효과**: 법규 매핑부터 체크리스트까지 실무에 즉시 활용 가능한 구조. 검증 필요 항목이 명시되어 후속 작업 우선순위 파악 용이.

| 개선 요소 | 적용 내용 | 효과 |
|----------|----------|------|
| 역할 특정 | 보험 컴플라이언스 전문가 | 규제 실무 관점의 답변 |
| 법규 매핑 요청 | 법률→시행령→감독규정 단계 | 규제 체계 전체를 조망 |
| 체크리스트 구조 | 표 형식 + 상태 표시 | 실무 검토 문서로 즉시 활용 |
| 불확실성 관리 | "검증 필요" 기본 표시 요청 | 과신 방지, 검증 우선순위 명확화 |

## 7. 교차 검증 워크플로 — 신뢰도 높은 리서치의 비결

### 왜 교차 검증이 필요한가?

AI는 자신 있게 **존재하지 않는 법률 조항을 만들어내거나, 실제와 다른 수치를 제시**할 수 있습니다. 이것이 할루시네이션(Hallucination)입니다. 단일 AI의 결과를 그대로 신뢰하면 보고서의 신뢰도가 무너집니다.

> 🔑 **핵심**: 전문가들은 **서로 다른 AI에게 역할을 분담**시킵니다. A 도구로 초안 수집 → B 도구로 정밀 분석 → C 도구로 교차 검증 → 원문 대조. 이 방식이 할루시네이션을 효과적으로 걸러냅니다.

### 교차 검증 4단계 워크플로

```
[Step 1] Perplexity로 초안 수집
  → 실시간 웹 검색으로 최신 자료 확보
  → 인라인 출처 링크 확인
  → 관련 법규, 사례, 데이터 1차 수집

     ↓

[Step 2] Claude/ChatGPT로 정밀 분석
  → 수집된 자료를 구조화·요약
  → 법령 원문을 직접 붙여넣고 핵심 조항 추출
  → 모호한 부분, 불확실한 부분 식별

     ↓

[Step 3] 다른 AI로 교차 검증
  → AI가 생성한 내용을 다른 AI에게 검증 요청
  → "아래 내용에서 사실과 다를 가능성이 있는 부분을 찾아줘"
  → 불확실한 항목 목록 도출

     ↓

[Step 4] 원문 대조 및 전문가 확인
  → 불확실 항목을 원문(law.go.kr, DART 등)과 직접 대조
  → 중요 사항은 전문가(법무팀, 컴플라이언스팀)에게 최종 확인
  → 검증 완료된 정보만 보고서에 반영
```

### 교차 검증 프롬프트 예시

**다른 AI에게 검증을 요청하는 프롬프트**:

```
아래 AI가 생성한 법규 요약문에서 사실과 다를 가능성이 있는
부분을 찾아주세요.

검토 대상:
"산업안전보건법 시행규칙 제87조에 따르면, 유기화합물을 취급하는
사업장은 작업환경측정을 연 2회 실시해야 합니다. 또한 화학물질관리법
제35조는 유독물질 취급시설의 정기검사를 3년마다 의무화하고 있습니다."

검토 방법:
1. 조항 번호(제87조, 제35조)의 실제 내용이 일치하는지 확인
2. 구체적 수치(연 2회, 3년)가 정확한지 확인
3. 유기화합물/유독물질 범위가 적절한지 확인
4. 최신 개정으로 바뀌었을 수 있는 부분 표시
5. 불확실한 부분은 확실성 수준(높음/중간/낮음)을 표시
```

**AI 자체 진단 프롬프트** (생성 직후 같은 AI에게 바로 사용):

```
위에서 네가 작성한 법규 내용 중:
1. 조항 번호가 틀릴 가능성이 있는 부분은?
2. 수치(횟수, 기간, 비율)가 불확실한 부분은?
3. 최신 개정으로 바뀌었을 수 있는 부분은?

각 항목에 확실성 수준(높음/중간/낮음)을 표시해줘.
```

> 💡 **팁**: "자기 진단 요청"은 AI가 스스로 불확실한 부분을 표시하게 하는 기법입니다. 완벽하지는 않지만, **원문 확인 우선순위를 빠르게 파악**하는 데 효과적입니다.

## 8. 검색 프롬프트 최적화 — 법률/기술 검색에 특화된 작성법

### 8-1. 법률 검색 프롬프트 최적화 원칙

| 원칙 | 나쁜 예 | 좋은 예 | 이유 |
|------|--------|--------|------|
| **물질/대상 특정** | "화학물질 규제" | "시클로헥사논(CAS 108-94-1) 취급 규제" | 범용 답변 방지 |
| **법률 단계 명시** | "관련 법률" | "법률 → 시행령 → 시행규칙 단계별" | 규제 체계 전체 조망 |
| **출처 요구** | (별도 언급 없음) | "원문 URL 포함, 출처 없으면 제외" | 검증 가능성 확보 |
| **불확실성 표시** | (별도 언급 없음) | "확실하지 않으면 '확인 필요' 표시" | 과신 방지 |
| **최신성 확인** | (별도 언급 없음) | "2024년 이후 개정 사항 포함" | 구 법령 인용 방지 |

### 8-2. 기술 검색 프롬프트 최적화 원칙

| 원칙 | 나쁜 예 | 좋은 예 | 이유 |
|------|--------|--------|------|
| **기술 범위 특정** | "OLED 기술 동향" | "IJP 방식 OLED 대면적 양산 기술, 2023~2025년" | 검색 범위 좁혀 품질 향상 |
| **비교 기준 명시** | "업체 비교" | "삼성/LG/BOE의 IJP 투자 규모, 양산 시점, 기술 과제 비교" | 비교축 명확화 |
| **데이터 유형 지정** | "자료 찾아줘" | "특허 출원 건수, 학술 논문 발표 추이, 투자 금액" | 정량적 데이터 확보 |
| **추측/사실 구분** | (별도 언급 없음) | "확인된 사실과 업계 추측을 구분해서 표시" | 정보 신뢰도 관리 |

### 8-3. 검색 효율을 높이는 프롬프트 패턴

**패턴 1: 단계적 검색 (Layered Search)**

```
[1단계 — 전체 조망]
"OLED IJP 공정 관련 규제 분야를 카테고리별로 분류해줘"

[2단계 — 특정 분야 심화]
"위 카테고리 중 '화학물질 규제' 부분을 상세히 조사해줘"

[3단계 — 구체적 조항 확인]
"화학물질관리법 중 유기 용제 취급 관련 조항만 정리해줘"
```

> 💡 **팁**: 처음부터 세부 사항을 검색하면 빠뜨리는 부분이 생깁니다. **전체 조망 → 분야 심화 → 세부 확인** 순서로 진행하면 누락 없이 체계적으로 조사할 수 있습니다.

**패턴 2: 대비 검색 (Contrastive Search)**

```
"IJP 공정 도입 시 규제 관점에서의 장점과 단점을 모두 조사해줘.
장점만이 아니라, 추가로 준수해야 할 규제 부담도 함께 정리해줘."
```

**패턴 3: 시간축 검색 (Temporal Search)**

```
"산업안전보건법의 유기화합물 관련 규제가 2020년 이후 어떻게
변경되었는지 연도별로 정리해줘. 현재 시행 중인 최신 기준과
향후 예정된 개정 사항도 포함해줘."
```

## 실습 & 퀴즈

### 퀴즈 1 — AI 검색 도구 선택

Q. 보험 약관 전문(50페이지)을 분석하여 핵심 조항을 추출해야 합니다. 가장 적합한 도구와 전략은?

(A) Perplexity로 약관 내용을 검색한다
(B) Claude에 약관 PDF를 업로드하고 핵심 조항 추출을 요청한다
(C) ChatGPT에게 "보험 약관 요약해줘"라고 요청한다
(D) Gemini에서 Google 검색으로 약관 내용을 찾는다

**정답**: (B) Claude에 약관 PDF를 업로드하고 핵심 조항 추출을 요청한다 — Claude는 긴 문서 분석과 복잡한 조건 준수에 강점이 있습니다. 50페이지 약관을 통째로 업로드하고 특정 조건(예: "면책 조항만 추출")으로 분석을 요청하면 가장 정확한 결과를 얻을 수 있습니다.

### 퀴즈 2 — 할루시네이션 대응

Q. AI가 "화학물질관리법 제35조는 유독물질 취급시설의 정기검사를 3년마다 의무화하고 있습니다"라고 답변했습니다. 올바른 대응은?

(A) AI가 조항 번호까지 구체적으로 제시했으므로 그대로 보고서에 인용한다
(B) 다른 AI에게 같은 질문을 해서 같은 답이 나오면 신뢰한다
(C) 국가법령정보센터(law.go.kr)에서 제35조 원문을 직접 확인한 후 반영 여부를 결정한다
(D) AI에게 "정말 맞아?"라고 물어봐서 "맞다"고 하면 사용한다

**정답**: (C) 국가법령정보센터에서 원문을 직접 확인한다 — AI가 구체적 조항 번호와 수치를 제시할수록 오히려 검증 우선순위가 높습니다. 원문 대조 없이는 어떤 AI 답변도 법률 정보로 사용할 수 없습니다. (B)의 "다른 AI 교차 확인"은 1차 필터로는 유용하지만, 원문 대조를 대체할 수는 없습니다.

### 퀴즈 3 — 제조업 기술 리서치

Q. 반도체 신규 식각(Etching) 공정의 기술 동향을 조사해야 합니다. 가장 효과적인 AI 활용 전략은?

(A) ChatGPT에게 "반도체 식각 공정 최신 기술 알려줘"라고 질문한다
(B) Perplexity로 최신 논문/특허 동향을 검색하고, Claude에 핵심 논문 PDF를 업로드하여 기술 비교표를 작성한다
(C) Gemini에게 "etching technology trend"를 검색시키고 결과를 그대로 보고서에 인용한다
(D) 사내 LLM에게 경쟁사 기술 정보를 물어본다

**정답**: (B) — 기술 리서치는 **수집과 분석을 분리**하는 것이 핵심입니다. Perplexity로 최신 출처가 표시된 동향을 수집하고, Claude에 원문 논문을 업로드하여 정밀 분석하면 깊이 있는 기술 비교가 가능합니다. (A)는 최신 정보가 누락될 수 있고, (C)는 원문 검증이 빠져 있으며, (D)의 사내 LLM은 외부 기술 동향 데이터가 제한적입니다.
