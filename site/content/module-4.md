# 모듈 4: AI 기반 보고서 작성 방법론

> **대상**: KDB생명 임직원
> **학습 시간**: 약 50~60분
> **핵심 키워드**: KDB생명 보고서 유형, 5단계 AI 보고서 워크플로, 업무별 프롬프트 템플릿, 반복 보고서 자동화, Executive Summary

---

## 학습 목표
1. KDB생명 업무별 보고서 유형의 특성을 파악하고 AI 활용 전략을 수립할 수 있다
2. 5단계 AI 보고서 워크플로를 실무 보고서에 적용할 수 있다
3. 인수심사 의견서, 보험시장 분석 브리핑, Executive Summary 등 업무별 프롬프트 템플릿을 바로 활용할 수 있다
4. 반복 보고서를 프롬프트 템플릿으로 표준화하여 매주·매월 재사용할 수 있다

---

## 1. KDB생명 보고서 유형 지도

KDB생명의 보고서는 목적, 독자, 형식이 명확히 구분됩니다. AI를 활용할 때도 보고서 유형에 따라 프롬프트 전략이 달라져야 합니다.

### 1-1. 유형별 특징 비교표

| 보고서 유형 | 주요 독자 | 핵심 목적 | 분량 | 톤·문체 | AI 활용 포인트 |
|------------|----------|----------|------|---------|--------------|
| **기안서** | 결재 라인(팀장→본부장→임원) | 의사결정 승인 요청 | 1~3쪽 | 격식체, 개조식 | 목차·골격 설계, 근거 자료 정리 |
| **인수심사 의견서** | 심사위원회, 언더라이팅 담당 임원 | 보험계약 인수 승인/거절 근거 제시 | 5~15쪽 | 분석적, 수치 중심 | 의료 리스크 요약, 통계 분석 정리, 리스크 항목 구조화 |
| **결재 보고서** | 경영진, 이사회 | 주요 안건 보고·승인 | 3~5쪽 | 결론 선행, 간결 | Executive Summary 추출, 핵심 수치 강조 |
| **보험시장 분석 리포트** | 상품전략부, 유관 부서 | 시장 분석, 경쟁 상품 비교 | 10~30쪽 | 논리적, 출처 명시 | 자료 수집·요약, 해외 사례 번역·정리, 참고문헌 구조화 |
| **브리핑 노트** | 임원, 회의 참석자 | 신속한 상황 전달 | 1~2쪽 | 핵심 위주, 불릿 포인트 | 장문 자료를 1페이지로 압축 |
| **Executive Summary** | 최고경영진, 외부 이해관계자 | 전체 보고서의 핵심 전달 | 0.5~1쪽 | 결론→근거→후속, 최상위 톤 | 긴 보고서에서 핵심 추출 |

> 🔑 **핵심**: 같은 주제라도 보고서 유형이 다르면 프롬프트가 달라야 합니다. "인수심사 의견서용으로 써줘"와 "시장 분석 리포트용으로 써줘"는 AI에게 전혀 다른 지시입니다. 독자와 목적을 프롬프트 첫 줄에 명시하세요.

### 1-2. AI 활용도가 높은 보고서 vs 사람이 주도해야 하는 보고서

| 구분 | AI 활용도 높음 | 사람 주도 필수 |
|------|-------------|-------------|
| **자료 수집·정리** | 보험시장 동향, 경쟁사 상품 비교, 해외 사례 번역 | 보험계약자 의료정보 분석, 내부 기밀 데이터 |
| **초안 작성** | 정형화된 구조의 보고서, 반복 보고 | 인수심사 최종 판단, 보험금 지급 결정, 리스크 판정 |
| **검증·마감** | 수치 정합성 체크, 오탈자 검수 | 보험업법·약관 최종 확인, 경영진 판단 사항 |

> ⚠️ **주의**: AI가 아무리 정교한 분석을 제시하더라도, **인수심사 승인 여부, 보험금 지급 판단, 자산운용 의사결정은 반드시 담당자와 결재 라인이 결정**합니다. 금융분야 AI 가이드라인의 '보조수단성' 원칙 — 최종 의사결정 책임은 임직원에게 있습니다.

## 2. 5단계 AI 보고서 워크플로

보고서를 AI와 함께 작성할 때 핵심 원칙은 **한 번에 전부를 요청하지 않는 것**입니다. 단계별로 나눠 진행하면 품질과 통제력이 크게 향상됩니다.

### 워크플로 전체 구조

| 단계 | 작업 | 담당 | 시간 비중 | 핵심 포인트 |
|------|------|------|----------|------------|
| **1단계** | 목적·독자 정의, 목차 설계 | 사람 주도 + AI 보조 | 10% | 역할+맥락+제약조건으로 목차 생성 |
| **2단계** | 자료 수집·정리 | AI 주도 | 20% | 원자료를 AI에 제공하여 구조화 |
| **3단계** | 초안 생성 | AI 주도 | 25% | 섹션별 분리 요청, Prompt Chaining |
| **4단계** | 검증·팩트체크 | 사람 필수 + AI 보조 | 25% | 수치·법령·통계 교차 검증 |
| **5단계** | 리뷰·마감 | 사람 필수 | 20% | Executive Summary 추출, 최종 결재 |

```
[1. 목적·독자 정의] → [2. 자료 수집] → [3. 초안 생성] → [4. 검증] → [5. 리뷰·마감]
   사람+AI             AI 주도          AI 주도         사람+AI       사람 필수
   ─────────── AI 보조 영역 ───────────   ─────── 사람 주도 영역 ───────
```

### 1단계: 목적·독자 정의 — 프롬프트로 목차 잡기

보고서의 방향을 결정하는 가장 중요한 단계입니다. AI에게 보고서의 목적, 독자, 결론 방향을 명확히 전달합니다.

```
당신은 KDB생명 언더라이팅부의 선임 인수심사역입니다.

저는 심사위원회에 올릴 [B고객 고액 종신보험 인수심사 의견서]를 준비 중입니다.
- 독자: 인수심사위원회 (의료·재무 심사 전문가 5인)
- 분량: 본문 10쪽 내외 + Executive Summary 1쪽
- 결론 방향: 표준 인수/조건부 인수(할증·부담보)/거절 중 하나로 수렴
- 반드시 포함: 피보험자 개요, 의료 리스크 평가, 직업·재정 리스크, 종합 인수 의견, 특별 조건 제안

이 조건에 맞춰 의견서 목차를 제안해주세요.
각 섹션별로 포함해야 할 핵심 항목을 2~3개씩 명시해주세요.
```

> 💡 **팁**: 목차를 AI가 제안하면 바로 채택하지 말고, **기존에 결재를 받은 의견서의 목차와 비교**하세요. "우리 팀 의견서는 보통 이 구조야"라고 예시를 제공하면 훨씬 정확한 목차가 나옵니다.

### 2단계: 자료 수집·정리 — AI에 인풋 제공

건강검진 결과, 의료 기록 요약, 보험개발원 통계 등 원자료를 AI에 넣어 구조화합니다.

```
아래는 B고객의 건강검진 결과 요약과 기존 보유계약 현황입니다.

(건강검진 결과 요약 텍스트를 여기에 붙여넣기)

요청:
1. 주요 의료 리스크 항목을 표로 정리 (항목, 수치, 정상범위, 위험도)
2. 보험의학적 관점에서 주목할 점 3가지 (불릿 포인트)
3. 직업 리스크 평가 (직업군 분류, 위험등급)
4. 인수심사위원회 보고용 1문단 요약

조건:
- 의료 수치는 원 데이터를 그대로 인용하고, AI 분석 부분은 [분석]으로 구분
- 보험의학 표준체 기준표와 비교하여 위험도 판단
```

> ⚠️ **주의**: 외부 AI(ChatGPT, Claude 등)에 자료를 입력할 때, **보험계약자의 실명·의료정보·건강정보는 절대 입력하지 마세요.** 가명 처리한 데이터만 활용하거나, 사내 AI 시스템이 있다면 해당 시스템을 우선 사용하세요.

### 3단계: 초안 생성 — AI가 작성

목차와 정리된 자료를 기반으로 섹션별로 초안을 작성합니다. **한 번에 전체를 쓰라고 하지 말고, 섹션별로 나눠서 요청**하는 것이 품질을 높이는 핵심입니다(Prompt Chaining 기법).

```
앞서 정리한 B고객 건강검진 데이터와 리스크 분석 자료를 바탕으로,
의견서의 "3장. 의료 리스크 평가" 섹션 초안을 작성해주세요.

조건:
- 분량: 1,500자 내외
- 톤: 인수심사 의견서체 (격식적, 분석적, 개조식)
- 필수 포함 항목:
  · 현병력 분석 (고혈압 경과, 투약 이력, 혈압 조절 상태)
  · 가족력 분석 (심혈관 질환 가족력의 보험의학적 의미)
  · 체격 분석 (BMI, 흡연력, 복합 리스크 평가)
- 각 분석 항목에 "표준/경계/위험" 등급 표시
- 수치 뒤에 [출처: 건강검진 결과서] 형태로 출처 표기
```

> 💡 **팁**: "이전 대화 내용을 바탕으로"라고 명시하면 AI가 앞서 정리한 데이터를 자동으로 참조합니다. 같은 대화창 안에서 단계를 이어가세요. 새 대화를 시작하면 맥락이 끊깁니다.

### 4단계: 검증·팩트체크

AI가 생성한 초안에서 의료 수치, 보험의학 기준, 통계 데이터 등의 사실 여부를 점검합니다.

```
다음은 AI가 작성한 인수심사 의견서 초안의 "의료 리스크 평가" 섹션입니다.

(초안 텍스트 붙여넣기)

이 초안에서 의료 수치·보험의학 기준·통계 등 팩트를 점검해주세요.
각 팩트마다 아래 형식으로 정리:

| 항목 | 본문 기재 내용 | 출처 확인 가능 여부 | 최신성 (기준 연도) | 검증 방법 제안 |
|------|-------------|-------------------|------------------|-------------|

특히 다음 사항에 주의:
- 의료 수치가 원자료(건강검진 결과서)와 일치하는지
- 보험의학 위험등급 판정이 표준체 기준표와 일치하는지
- AI가 추론한 내용이 사실처럼 기술되어 있지 않은지
```

> 🔑 **핵심**: 4단계는 AI가 보조하지만 **최종 판단은 반드시 사람**이 합니다. AI의 팩트체크 결과도 다시 한번 확인해야 합니다. 특히 의료 수치와 보험약관 조항은 원문 대조가 필수입니다.

### 5단계: 리뷰·마감 — Executive Summary 추출

전체 의견서 초안이 완성되면, 임원 보고용 요약을 추출합니다.

```
아래는 완성된 [B고객 고액 종신보험 인수심사 의견서] 전문입니다.

(의견서 전문 붙여넣기)

임원 보고용 1페이지 Executive Summary를 작성해주세요.

형식:
- 한 줄 결론 (인수 승인/조건부 인수/거절 + 핵심 근거 1문장)
- 핵심 포인트 5개 (각 1~2줄)
  · 피보험자 개요
  · 의료 리스크 평가
  · 직업·재정 리스크
  · 종합 인수 의견
  · 특별 조건 제안
- 추가 검토 필요 사항 (있을 경우)

톤: 경영진 보고체 (결론 선행, 간결, 수치 중심)
분량: A4 1쪽 이내 (약 500자)
```

## 3. 업무별 복사 가능 템플릿 3종

아래 프롬프트는 실무에서 **복사하여 바로 사용**할 수 있도록 설계되었습니다. `[대괄호]` 안의 내용만 실제 데이터로 교체하면 됩니다.

### 템플릿 1: 보험 인수심사(언더라이팅) 의견서

**상황**: 언더라이팅부에서 고액 보험계약의 인수심사를 진행하며, 심사위원회에 제출할 의견서를 작성해야 합니다.

```
당신은 KDB생명의 선임 언더라이터(15년차)입니다.
언더라이팅부에서 [B고객(남성, 52세, 건설업 현장관리직)]에 대한
고액 [종신보험(보험가입금액 5억원)] 건의 인수심사 의견서를 작성해야 합니다.

심사위원회 제출용이며, 독자는 의료·재무 심사에 익숙한 위원 5인입니다.

아래 구조로 인수심사 의견서 초안을 작성해주세요:

1. 피보험자 개요
   - 성명(가명), 연령, 성별, 직업, 기존 보유계약 현황

2. 보험계약 요청 내용
   - 보험종류, 보험가입금액, 보험기간, 납입기간, 수익자

3. 의료 리스크 평가
   - 현병력: 주요 질환, 투약 이력, 경과 상태
   - 가족력: 주요 질환 가족력 및 보험의학적 의미
   - 체격: BMI, 혈압, 흡연력 분석
   - [아래 건강검진 결과를 기반으로 작성]
   (건강검진 결과 요약 붙여넣기)

4. 직업·재정 리스크 평가
   - 직업 위험등급 분류
   - 보험가입금액 대비 소득 적정성
   - 도덕적 위험(Moral Hazard) 점검

5. 종합 인수 의견
   - 인수 판단 (표준 인수/조건부 인수/거절)
   - 주요 리스크와 완화 방안
   - 특별 조건 (할증, 부담보, 체감 보장, 재심사 조건)

형식 조건:
- 마크다운 표 적극 활용
- 의료 수치에는 출처(건강검진 결과서/의무기록) 표기
- AI 추론 부분은 [분석]으로 명시
- 분량: 본문 8~10쪽 + Executive Summary 1쪽
```

**예상 산출물 형태**:
- 구조화된 인수심사 의견서 초안 (표, 의료 수치 분석 포함)
- 의료 리스크 항목별 위험등급 표
- 직업·재정 리스크 매트릭스
- 1쪽 분량 Executive Summary

> 📌 **사례**: 교보생명은 AI 자동심사 시스템으로 보험금 지급 심사 시간을 평균 0.24일로 단축했습니다. iM라이프도 머신러닝 기반 재정심사 시스템(EUS)을 도입하여 인수심사 효율을 높이고 있습니다. KDB생명에서도 이 워크플로를 통해 인수심사 의견서 작성 시간을 단축하면서 분석 품질은 유지할 수 있습니다.

### 템플릿 2: 보험시장 분석 브리핑

**상황**: 상품전략부 또는 리스크관리부에서 특정 보험시장의 최근 동향을 1~2쪽으로 정리하여 임원에게 보고해야 합니다.

```
당신은 KDB생명 상품전략부의 수석 시장분석 담당자입니다.

[건강보험 시장]의 최근 동향 브리핑 자료를 작성해주세요.
독자는 KDB생명 경영진으로, 보험시장에 대한 기본 이해는 있으나
최근 변화를 빠르게 파악하고 싶어합니다.

구조:
1. 한 줄 요약 (시장의 현재 상황을 1문장으로)

2. 시장 현황 (표 형식)
   - 국내 건강보험 시장 규모 및 성장률
   - 주요 경쟁사 동향 (삼성생명, 한화생명, 교보생명)
   - 최근 3개월 주요 이벤트

3. 핵심 이슈 3가지 (각 3~5줄)
   - [예: 4세대 실손보험 전환율 하락, 고령화에 따른 간병보험 수요 급증, 디지털 헬스케어 보험 출시 경쟁]

4. KDB생명 관점 시사점
   - 상품개발: 신상품 출시 기회
   - 언더라이팅: 인수심사 기준 조정 필요성
   - 리스크관리: 손해율·K-ICS 영향

5. 참고 자료 목록 (출처·발행일 명시)

형식:
- 분량: A4 2쪽 이내 (약 1,500자)
- 톤: 간결한 브리핑체 (불릿 포인트 중심)
- 수치에는 반드시 출처와 기준 시점 병기
```

**예상 산출물 형태**:
- 1문장 핵심 요약
- 시장 현황 요약표
- 핵심 이슈 3가지 (간결한 분석)
- KDB생명 부서별 시사점
- 참고 자료 리스트

### 템플릿 3: 임원 보고용 요약 (Executive Summary)

**상황**: 이미 작성된 장문의 보고서(10쪽 이상)에서 임원 보고용 1쪽 요약을 추출해야 합니다.

```
당신은 KDB생명 전략기획부의 보고서 편집 전문가입니다.

아래는 [월간 보험 포트폴리오 리뷰 보고서] 전문입니다. (총 15쪽)

(보고서 전문을 여기에 붙여넣기)

이 보고서의 임원 보고용 Executive Summary를 작성해주세요.

형식 (반드시 이 순서로):
1. **결론** (1문장): 핵심 권고 사항
2. **배경** (2~3줄): 왜 이 보고서가 필요했는가
3. **핵심 발견 사항** (3~5개 불릿 포인트):
   - 각 포인트는 1~2줄, 핵심 수치 포함
4. **권고 사항** (2~3개):
   - 구체적 행동 항목 (누가, 무엇을, 언제까지)
5. **리스크 요인** (1~2개):
   - 권고안 실행 시 주의 사항
6. **추가 검토 필요 사항** (있을 경우)

조건:
- 분량: A4 1쪽 이내 (500자 내외)
- 원문에 없는 내용은 절대 추가하지 않기
- 수치는 원문 그대로 인용
- 경영진이 30초 안에 핵심을 파악할 수 있는 톤
```

**예상 산출물 형태**:
- 결론 1문장으로 시작하는 구조화된 요약
- 원문 수치를 그대로 인용한 핵심 포인트
- 구체적 행동 항목이 포함된 권고 사항

> 💡 **팁**: Claude는 1M 토큰(약 75만 단어)의 컨텍스트 윈도우를 지원하므로, 15~20쪽 분량의 보고서를 통째로 붙여넣어도 처리할 수 있습니다. 보고서 전문을 넣을수록 Executive Summary의 품질이 올라갑니다.

## 4. Before/After 비교 — AI 없이 vs AI로 작성한 보고서

### 4-1. 인수심사 의견서 작성 비교

| 비교 항목 | AI 없이 작성 | AI 워크플로 활용 |
|----------|------------|----------------|
| **소요 시간** | 약 8~12시간 | 약 2~3시간 |
| **자료 수집** | 건강검진 결과·의무기록 수동 분석, 보험의학 기준표 대조 (3~4시간) | AI에 건강검진 데이터 입력 후 리스크 자동 구조화 (30분) |
| **시장 동향** | 보험개발원·생명보험협회 수동 탐색·요약 (2~3시간) | AI로 보험시장 동향 초안 생성 후 검증 (40분) |
| **초안 작성** | 백지에서 시작, 구조 설계부터 (2~3시간) | AI 초안 기반 수정·보완 (30분) |
| **검증·마감** | 의료 수치 재확인, 약관 대조, 오탈자 검수 (1~2시간) | AI 팩트체크 보조 + 사람 최종 확인 (30분) |
| **품질 특성** | 작성자 역량에 크게 의존, 구조 편차 발생 | 일관된 구조, 누락 항목 최소화 |

> 🔑 **핵심**: AI 활용의 가치는 단순히 시간 절감이 아닙니다. **구조적 완결성이 높아지고, 분석의 누락이 줄어드는 것**이 더 큰 이점입니다. 다만 AI 산출물을 검증 없이 그대로 제출하는 것은 절대 금지입니다.

### 4-2. 보험시장 분석 브리핑 작성 비교

**Before (AI 없이)**:
- 보험연구원, 생명보험협회, 보험개발원 자료를 개별 검색 (2시간)
- 핵심 내용을 수동으로 발췌·정리 (1시간)
- 브리핑 문서 형태로 재구성 (1시간)
- **총 4시간, 자료 누락 가능성 있음**

**After (AI 워크플로 활용)**:
- 1단계: AI에게 보험시장 동향 검색 및 초안 생성 요청 (20분)
- 2단계: AI 결과를 KDB생명 관점으로 수정·보완 (20분)
- 3단계: 출처 확인 및 최신성 검증 (20분)
- **총 1시간, 구조화된 브리핑 완성**

## 5. 반복 보고서 자동화 — 프롬프트 템플릿을 저장하고 재사용

### 5-1. 반복 보고서의 특성

KDB생명 업무에서 매주·매월 반복되는 보고서가 있습니다:

| 보고서 유형 | 주기 | 특징 |
|------------|------|------|
| 주간 보험계약 현황 보고 | 매주 | 데이터만 변경, 구조 동일 |
| 월간 보험 포트폴리오 리뷰 | 매월 | 수입보험료, CSM, 해약률, 손해율, K-ICS 현황 분석 |
| 분기 K-ICS 건전성 보고 | 분기 | K-ICS 비율, 자본적정성 분석 |
| 월간 보험시장 동향 리포트 | 매월 | 시장별 동향 업데이트 |

이런 보고서는 **역할(Role) + 형식(Format)은 한 번만 작성해 저장**하고, **매번 데이터(Data)만 교체**하는 방식으로 자동화할 수 있습니다.

### 5-2. 프롬프트 템플릿 저장 방법

**원칙**: 프롬프트를 [고정 부분]과 [변동 부분]으로 분리합니다.

```
===== 월간 보험 포트폴리오 리뷰 프롬프트 (저장용) =====

[고정 부분 — 매번 동일]
당신은 KDB생명 리스크관리부의 포트폴리오 분석 담당자입니다.
월간 보험 포트폴리오 리뷰 보고서를 작성합니다.

구조:
1. 이달의 핵심 요약 (3줄)
2. 주요 지표 현황 (표: 수입보험료 / CSM 잔액 / 해약률 / 손해율 / K-ICS 비율)
3. 상품군별 포트폴리오 분석 (보장성/저축성/변액/연금)
4. 신규 리스크 모니터링 항목
5. 다음 달 주목할 이벤트
6. 참고 자료

형식: 마크다운, 표 중심, 분량 A4 3쪽 이내
톤: 분석적, 간결, 수치 기반

[변동 부분 — 매월 교체]
이달 기준: [2026년 4월]
대상 상품군: [종신보험, 건강보험, 변액보험, 연금보험]
이달 주요 이벤트:
- (여기에 이달의 주요 뉴스·이벤트를 붙여넣기)

이달 데이터:
- (여기에 이달의 포트폴리오 데이터를 붙여넣기)

위 데이터를 기반으로 월간 보험 포트폴리오 리뷰를 작성해주세요.
=========================================================
```

### 5-3. 플랫폼별 프롬프트 저장·재사용 방법

| 플랫폼 | 저장 방법 | 재사용 방법 |
|--------|----------|-----------|
| **ChatGPT** | Custom Instructions 또는 GPTs에 고정 프롬프트 설정 | 새 대화 시 자동 적용, 데이터만 입력 |
| **Claude** | Projects 기능에 프로젝트별 프롬프트 저장 | 프로젝트 내에서 자동 참조, 데이터만 추가 |
| **사내 시스템** | 프롬프트 라이브러리에 등록 (구축 시) | 목록에서 선택 후 데이터 입력 |
| **개인 관리** | 텍스트 파일·노션 등에 프롬프트 보관 | 복사·붙여넣기 후 변동 부분만 교체 |

> 💡 **팁**: ChatGPT의 GPTs 기능을 활용하면, "KDB생명 월간 포트폴리오 리뷰 작성기"처럼 용도별 맞춤 AI를 만들 수 있습니다. 고정 프롬프트를 Instructions에 넣어두면 매번 긴 프롬프트를 복사할 필요 없이 데이터만 입력하면 됩니다.

## 학습 정리

### 핵심 요약
- KDB생명 보고서는 기안서·인수심사 의견서·결재보고서·보험시장 분석 리포트·브리핑·Executive Summary로 나뉘며, 유형에 따라 AI 활용 전략이 달라야 한다
- 5단계 AI 보고서 워크플로(목적 정의 → 자료 수집 → 초안 생성 → 검증 → 리뷰)를 적용하면 시간을 절감하면서 구조적 완결성을 높일 수 있다
- 업무별 프롬프트 템플릿을 복사하여 `[대괄호]` 내용만 교체하면 바로 사용할 수 있다
- 반복 보고서는 프롬프트의 고정 부분과 변동 부분을 분리하여 저장하면 매번 재사용할 수 있다
- **최종 의사결정 책임은 항상 임직원에게 있다** — AI는 보조 도구이며, 인수심사 판단·보험금 지급 결정은 사람이 한다

### 즉시 적용 체크리스트
- [ ] 가장 자주 쓰는 보고서 유형 1가지를 선정하여 5단계 워크플로를 적용해본다
- [ ] 템플릿 3종 중 1개를 복사하여 실제 업무에 시험 사용해본다
- [ ] 반복 보고서 1건의 프롬프트를 [고정 부분]과 [변동 부분]으로 분리해 저장한다
- [ ] AI 초안을 검증할 때 "수치→출처→최신성" 3가지를 반드시 체크한다
- [ ] 외부 AI에 입력하는 자료가 공개 정보인지 반드시 확인한다
