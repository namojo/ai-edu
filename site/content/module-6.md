# 모듈 6: 안전한 AI 활용 — 할루시네이션·컴플라이언스·정보보안

> **대상**: KDB생명 임직원
> **학습 시간**: 약 50~60분
> **핵심 키워드**: 할루시네이션, 팩트체크 3단계, 보험업 규제, AI기본법, KDB생명 내부통제, 정보보안

---

## 학습 목표
1. 보험업 AI 활용이 일반 산업 대비 왜 특별히 높은 리스크를 수반하는지 설명할 수 있다
2. AI 할루시네이션의 5대 패턴을 보험 실무 관점에서 식별하고, 각 패턴에 맞는 예방·탐지 기법을 적용할 수 있다
3. 팩트체크 3단계 절차를 숙지하여 AI 생성 결과물의 신뢰성을 체계적으로 검증할 수 있다
4. AI기본법(2026.01 시행), 금융분야 AI 가이드라인 등 현행 규제 체계를 이해하고 준수할 수 있다
5. KDB생명형 AI 안전 활용 의사결정 트리를 적용하여, 업무별로 적절한 AI 활용 범위와 검증 수준을 판단할 수 있다

---

## 1. 왜 보험업 AI는 특별히 위험한가

일반 기업에서 AI가 오류를 내면 업무 비효율이 발생합니다. 보험회사에서 AI가 오류를 내면 **보험계약자 피해, 규제 위반, 건전성 악화**로 이어집니다. 보험업은 여기에 한 가지 차원이 더해집니다 — 고객의 생명·건강·재산과 직결되는 **보험계약의 공공적 성격**입니다.

### 1-1. 금융소비자 보호 의무

금융소비자보호법은 보험회사에 **설명의무**(제19조), **적합성 원칙**(제17조), **적정성 원칙**(제18조)을 부과합니다. AI가 생성한 정보를 기반으로 고객에게 보험상품을 권유하거나 설명했는데, 그 정보가 할루시네이션이었다면? 설명의무 위반으로 **손해배상 책임**이 발생합니다.

> 주의: 2024년 에어캐나다 판결이 선례입니다. 법원은 "기업은 AI 챗봇의 발언에 법적 책임을 진다"고 판시했습니다. AI가 말한 것이라고 면책되지 않습니다.

### 1-2. 보험계약자 보호와 보험업의 공익성

보험업법은 보험업의 건전한 육성과 **보험계약자의 권익 보호**를 목적으로 합니다(제1조). 보험회사는 고객의 의료정보, 건강정보, 재정정보를 대량으로 보유하고 있으며, AI 오류로 인한 잘못된 인수심사, 부당한 보험금 지급 거절, 부적절한 상품 추천은 **보험계약자의 직접적 피해**와 **보험 신뢰도 훼손**으로 직결됩니다.

### 1-3. 보험계약자 의료정보·건강정보의 민감성

KDB생명의 언더라이팅팀과 보험금심사팀이 다루는 정보에는 고객의 **의료정보, 건강검진 결과, 질병 이력** 등 극도로 민감한 개인정보가 포함됩니다. 이러한 정보가 외부 AI에 입력되는 순간:

| 리스크 | 결과 |
|--------|------|
| 의료정보 유출 | 외부 LLM 서버에 보험계약자의 의료정보가 학습 데이터로 편입될 가능성 |
| 개인정보보호법 위반 | 의료정보는 민감정보로 별도 동의 없이 처리 불가(개인정보보호법 제23조) |
| 보험업법 위반 | 보험계약자 정보의 부당 이용·제공 금지 위반 |
| 고객 신뢰 상실 | 의료정보 유출 시 보험사에 대한 사회적 비난과 대규모 민원 발생 |

### 1-4. 금융감독기관의 AI 활용 감시 강화

2024~2026년 금융당국은 AI 규제를 급격히 강화하고 있습니다:

- **인공지능기본법** 2026년 1월 22일 전면 시행 — 보험업은 **고영향 AI**로 분류, 강화된 관리 의무 부과
- **금융분야 AI 통합 가이드라인** 2026년 1분기 시행 — 7대 원칙, 내부 업무 지원용 AI까지 적용 범위 확대
- **금융위원회** "금융권 생성형 AI 활용 지원방안" — 오픈소스 AI 내부망 설치, 금융권 AI 플랫폼 구축
- **금감원 보험업 AI 감독 강화** — AI 기반 언더라이팅의 공정성·설명가능성, 보험금 자동심사의 적정성 감시
- 미국 FINRA도 2026년 감독 보고서에서 AI 할루시네이션을 **주요 리스크**로 지목

> 핵심: KDB생명 임직원의 AI 활용은 "개인의 업무 편의"가 아니라 "금융감독 대상 행위"입니다. 모든 AI 활용에 감독 관점의 기준을 적용해야 합니다.

## 2. 할루시네이션 5대 패턴 — 보험업 실제 예시

AI 할루시네이션은 "AI가 모른다고 말하지 못하고, 그럴듯한 거짓을 자신 있게 생성하는 현상"입니다. 2026년 벤치마크 기준, 주요 LLM의 할루시네이션율은 Gemini 2.0 Flash 0.7%에서 Grok 4.1 Fast 20.2%까지 분포합니다. 가장 비싼 모델이 가장 정확한 것은 아닙니다.

| 모델 | 할루시네이션율 | 비고 |
|------|-------------|------|
| Gemini 2.0 Flash | 0.7% (최저) | 빠른 모델 중 가장 신뢰 |
| Gemini 3.1 Pro | 10.4% | - |
| Claude Sonnet 4.6 | 10.6% | - |
| GPT-5.2 | 10.8% | - |
| Claude Opus 4.6 | 12.2% | - |
| DeepSeek R1 | 14.3% | 추론 모델이 사실 정확도에서 약함 |
| Grok 4.1 Fast | 20.2% (최고) | 빠른 추론 모드에서 급증 |

(출처: Vectara Hallucination Leaderboard, Suprmind, 2026)

> 주의: 추론 능력이 뛰어난 모델이 반드시 사실 정확도가 높은 것은 아닙니다. "가장 지능적으로 마케팅되는 모델이 기본적인 사실 확인에서는 가장 신뢰할 수 없는 경우가 많다"는 점을 기억하세요.

### 패턴 1: 존재하지 않는 보험 법령·감독규정 인용

**시나리오**: 준법감시팀 직원이 AI에게 "보험상품 판매 시 설명의무 관련 보험업감독규정을 정리해줘"라고 요청. AI가 "보험업감독규정 제4-18-2조에 따르면 보험설계사는 보장내역 설명서를 교부하고 서명을 받아야 한다"고 답변. **해당 조항은 존재하지 않음.**

| 항목 | 내용 |
|------|------|
| **탐지 방법** | 법령·규정의 정확한 조항 번호와 명칭을 국가법령정보센터(law.go.kr) 또는 금융감독원 법규검색(law.fss.or.kr)에서 직접 검색 |
| **경고 신호** | 지나치게 구체적인 조항 번호(제X-X-X조), "~에 따르면"이라는 단정적 인용 |
| **예방 프롬프트** | "관련 보험업감독규정을 정리하되, 각 규정의 정확한 조항 번호를 명시하고, 확실하지 않은 경우 '확인 필요'로 표시해줘. 최종적으로 금감원 법규 검색에서 원문을 대조해야 함을 안내해줘" |

### 패턴 2: 허위 보험 통계·지표 생성

**시나리오**: 리스크관리팀 직원이 "국내 주요 생보사의 K-ICS 비율 현황을 정리해줘"라고 요청. AI가 실제 보험사명(삼성생명, 한화생명)과 함께 "A생명의 K-ICS 비율 215.3%, 경과조치 적용 전 178.6%"처럼 **존재하지 않는 구체적인 수치**를 생성.

| 항목 | 내용 |
|------|------|
| **탐지 방법** | 금감원 보험통계(fss.or.kr), 각 보험사 경영공시, DART 사업보고서에서 K-ICS 비율 원문 확인 |
| **경고 신호** | 소수점까지 정밀한 K-ICS 비율, 다수 보험사의 수치가 한꺼번에 나열됨, 경과조치 전/후 비율이 모두 제시됨 |
| **예방 프롬프트** | "보험사별 K-ICS 비율을 나열하되, 각 수치의 출처(금감원 공시, 사업보고서 등)와 기준일을 명시해줘. 학습 데이터에 포함되지 않은 수치는 '공시 확인 필요'로 표시해줘" |

### 패턴 3: 오래된 규제를 현행 규제처럼 서술

**시나리오**: 상품개발팀 담당자가 "보험업감독규정상 건강보험 상품 인가 요건을 정리해줘"라고 요청. AI가 K-ICS 도입 이전의 구 지급여력(RBC) 기준 규정을 현행 규정인 것처럼 서술. IFRS 17 도입 이후 변경된 보험부채 평가 기준이 반영되지 않음.

| 항목 | 내용 |
|------|------|
| **탐지 방법** | 법령의 "시행일"과 "최종 개정일"을 국가법령정보센터에서 확인. AI 답변의 시점과 현행법 시점 비교 |
| **경고 신호** | K-ICS 이전의 RBC 기준 언급, IFRS 17 이전의 보험부채 평가 방식 서술, 최근 2~3년간 대폭 개정된 보험 규제에서 특히 빈발 |
| **예방 프롬프트** | "2026년 4월 현재 시행 중인 현행 보험업법·보험업감독규정 기준으로 답변해줘. K-ICS(2023년~)와 IFRS 17(2023년~) 도입 이후 변경된 사항을 반영해줘. 학습 데이터의 시점 한계로 최신 개정이 반영되지 않았을 수 있으니, 각 규정의 최종 개정일을 함께 표기해줘" |

### 패턴 4: 출처 혼동 (A 보고서 내용을 B라고 표기)

**시나리오**: 상품개발팀 직원이 "보험연구원(KIRI)의 건강보험 시장 전망 리포트를 요약해줘"라고 요청. AI가 생명보험협회 경영통계의 내용을 보험연구원 리포트의 내용인 것처럼 서술. 출처가 뒤바뀌어 인용됨.

| 항목 | 내용 |
|------|------|
| **탐지 방법** | AI가 명시한 출처의 원문을 직접 접근하여 해당 내용이 실제로 존재하는지 확인. 보험연구원(kiri.or.kr)에서 직접 검색 |
| **경고 신호** | 여러 기관의 보고서를 한꺼번에 요약 요청했을 때 빈발. 출처 간 내용이 자연스럽게 섞여 있으면 의심 |
| **예방 프롬프트** | "각 정보 항목마다 출처를 [기관명, 보고서명, 발간일] 형식으로 명시해줘. 두 개 이상의 출처가 혼합된 내용은 별도로 구분해줘" |

### 패턴 5: 보험 정량 수치 오류 (손해율, 사업비율, CSM 등)

**시나리오**: 보험계리팀 담당자가 "자사의 최근 3년간 손해율과 사업비율 추이를 업계 평균과 비교해줘"라고 AI에 요청. AI가 실제 경영공시와 다른 수치를 제시하거나, 연도별 수치를 뒤섞어서 표기. 손해율 72.3%를 65.8%로, 사업비율 28.1%를 22.5%로 오류 생성.

| 항목 | 내용 |
|------|------|
| **탐지 방법** | 생명보험협회 경영통계, 금감원 보험통계, DART 사업보고서에서 해당 보험사의 원문 확인. 보험개발원 통계와 교차검증 |
| **경고 신호** | 소수점 단위까지 구체적인 수치, 연도별 추이가 지나치게 매끄러운 패턴(실제 보험 경영지표는 변동성이 큼) |
| **예방 프롬프트** | "보험 경영지표를 정리하되, 각 수치의 출처(경영공시, 사업보고서 등)와 기준일을 명시해줘. 학습 데이터에 포함되지 않은 최신 수치는 '경영공시 확인 필요'로 표시해줘" |

> 핵심: 할루시네이션은 "AI가 틀릴 수도 있다"가 아니라 "AI는 반드시 틀린다"는 전제에서 출발하세요. 문제는 **언제, 어디서** 틀리는지를 빠르게 식별하는 것입니다.

## 3. 팩트체크 3단계 절차

AI 생성물의 신뢰성을 확보하기 위한 체계적 절차입니다. 모든 AI 결과물에 적용하되, 업무 중요도에 따라 검증 깊이를 조절합니다.

### 3-1. 1단계: AI에게 근거 요구

AI에게 답변을 요청할 때부터 "근거를 함께 제시하라"고 지시합니다. 이 단계에서 할루시네이션을 원천 차단할 수는 없지만, **검증 대상을 명확히** 할 수 있습니다.

**프롬프트 템플릿 — 근거 요구**

```
당신은 KDB생명의 [역할: 예) 언더라이터 / 보험계리사 / 리스크관리 담당]입니다.

[구체적 업무 요청 내용]

답변 시 다음 조건을 반드시 준수하세요:
1. 모든 수치에 출처(기관명, 보고서명, 기준 연도)를 명시
2. 법령·규정 인용 시 정확한 법명, 조항 번호, 시행일 표기
3. 확실하지 않은 정보는 "확인 필요" 또는 "추정"으로 명시 (추측을 사실처럼 서술하지 마세요)
4. 학습 데이터의 시점 한계를 밝히고, 최신 확인이 필요한 항목을 별도 표시
5. 참조한 출처 목록을 답변 말미에 정리
```

> 팁: "확실한 부분만 말해줘"보다 "확실하지 않은 부분을 '확인 필요'로 표시해줘"가 더 효과적입니다. AI에게 불확실성을 인정할 여지를 줄수록 할루시네이션이 감소합니다.

### 3-2. 2단계: 1차 검증 — 핵심 팩트 전수 점검

AI 답변에 포함된 **수치, 법령, 보험사명, 날짜** 등 구체적 팩트를 하나하나 점검합니다.

**검증 체크리스트**

| 검증 대상 | 검증 방법 | 검증 도구 |
|----------|----------|----------|
| 보험업법·감독규정 | 조항 번호 + 내용 일치 확인 | 국가법령정보센터(law.go.kr), 금감원 법규검색 |
| 보험사명·경영정보 | 실재 여부 + 정보 정확성 확인 | DART, 생명보험협회, 금감원 경영공시 |
| 보험 통계 수치 | 원출처 대조 (공시자료) | 보험개발원, 생명보험협회 경영통계, 금감원 보험통계 |
| K-ICS 관련 수치 | K-ICS 기준서 및 공시 대조 | 금감원 K-ICS 공시, DART 사업보고서 |
| 금리·투자수익률 | 기준일 + 수치 대조 | 한국은행(bok.or.kr), 금융투자협회 |
| 날짜·일정 | 시행일·발효일 정확성 | 관보, 법제처(moleg.go.kr) |

**프롬프트 템플릿 — AI를 활용한 자체 검증 요청**

```
아래 텍스트에 포함된 팩트를 점검해주세요.

[AI가 생성한 텍스트 붙여넣기]

각 팩트(수치, 법령, 보험사명, 날짜)에 대해 다음 표를 작성해주세요:
| # | 팩트 내용 | 유형(수치/법령/보험사/날짜) | 검증 가능 여부 | 신뢰도(높음/중간/낮음) | 추천 검증 방법 |

신뢰도가 "낮음"인 항목은 반드시 별도 표시해주세요.
주의: 이 검증도 AI가 수행하는 것이므로, 최종적으로 사람이 원출처를 확인해야 합니다.
```

> 주의: AI에게 자기 답변을 검증하라고 하면 "모든 정보가 정확합니다"라고 할루시네이션할 수 있습니다. AI 자체 검증은 **검증 대상을 식별하는 보조 도구**일 뿐, 검증 그 자체가 아닙니다.

### 3-3. 3단계: 권위 있는 소스 교차확인

업무 중요도가 높을수록 반드시 **원출처 기관의 공식 자료**와 대조합니다.

| 업무 유형 | 교차확인 필수 소스 | 최종 검증 주체 |
|----------|-----------------|-------------|
| 보험 인수심사 | 보험개발원 위험률, 경험생명표, 의학 참고자료 | 언더라이터 + 심사팀장 |
| 보험금 심사 | 약관 원문, 의료기록, 보험금 지급 기준 | 보험금심사역 + 심사팀장 |
| 상품 개발 | 보험업감독규정, 보험개발원 참조순보험요율, 경쟁사 공시 | 상품개발팀 + 보험계리팀 |
| K-ICS·건전성 | 금감원 K-ICS 기준서, DART 공시, 보험업감독규정 | 리스크관리팀 + 보험계리팀 |
| 자산운용 | 투자 실사 자료, 시장 데이터, ALM 분석 | 자산운용팀 + 리스크관리팀 |
| 컴플라이언스 | 법령 원문, 감독기관 유권해석, 제재 사례 | 준법감시팀 + 법무팀 |

**프롬프트 템플릿 — 교차확인 가이드 요청**

```
아래 AI 생성 내용에 대해 교차확인 계획을 수립해주세요.

[AI 생성 텍스트 중 검증이 필요한 부분 붙여넣기]

각 항목에 대해:
1. 확인해야 할 원출처 (기관명 + URL)
2. 확인 방법 (검색어, 접근 경로)
3. 우선순위 (상/중/하 — 오류 시 업무 영향도 기준)

을 표로 정리해주세요.
```

| 구분 | 내용 | KDB생명 영향 |
|------|------|---------|
| **위험 기반 분류** | AI를 위험 수준별로 분류, 보험업은 "고영향 AI" | KDB생명의 모든 AI 활용이 고영향 범주에 해당 |
| **5대 의무** | 투명성, 안전성, 고영향 AI 특별 책무, AI 영향 평가, 국내 대리인 지정 | AI 도입 시 사전 영향 평가 필수 |
| **생성형 AI 표시** | AI 생성 콘텐츠 표시(워터마크 등) 의무 | 대외 보고서·고객 안내에 AI 활용 사실 표시 필요 |
| **고영향 AI 특별 책무** | 위험관리방안·설명방안·이용자보호방안 수립, 사람의 관리·감독 의무 | AI 인수심사·보험금심사 결과에 대한 인간 검토 프로세스 구축 필수 |
| **계도 기간** | 최소 1년간 과태료 미부과 (시정명령은 가능) | 2027년부터 본격 제재 시작 전에 체계 정비 필요 |

### 4-4. KDB생명 직원이 반드시 지켜야 할 10대 원칙

아래 체크리스트는 모든 KDB생명 임직원이 AI를 활용할 때 반드시 준수해야 할 원칙입니다.

| # | 원칙 | 위반 시 리스크 |
|---|------|-------------|
| 1 | **보험계약자 의료정보·건강정보를 외부 LLM에 입력 금지** | 개인정보보호법 위반(민감정보), 보험업법 위반, 과징금·형사처벌 |
| 2 | **미출시 상품 설계서·보험료 산출 자료 입력 금지** | 영업 기밀 유출, 경쟁사 정보 노출, 기업 경쟁력 훼손 |
| 3 | **보험금심사 상세 내역·인수심사 기록 입력 금지** | 보험계약자 정보 유출, 심사 기밀 위반, 대규모 민원 |
| 4 | **AI 생성 내용은 반드시 인간이 최종 책임** | 금융분야 AI 가이드라인 "보조수단성" 원칙 위반 |
| 5 | **AI 생성 보험 법령·규정은 반드시 원문 대조 후 사용** | 존재하지 않는 보험업법·보험업감독규정 인용으로 감독 제재 |
| 6 | **AI 생성 보험 통계·K-ICS 수치는 반드시 원출처와 교차검증** | 오류 수치 기반 건전성 보고·상품 설계 시 규제 위반 리스크 |
| 7 | **대외 제출 문서에 AI 활용 시 검증 이력 기록** | AI기본법 투명성 의무, 감사 시 검증 증적 필요 |
| 8 | **사내 승인된 AI 도구만 사용** | 미승인 도구 사용 시 정보보안 위반, 데이터 유출 |
| 9 | **AI 답변을 고객에게 직접 전달하지 않음** | 금융소비자보호법 설명의무 위반, 할루시네이션 전달 위험 |
| 10 | **AI 사고 발생 시 즉시 정보보안팀·준법감시팀에 보고** | 사고 은폐 시 징계·법적 책임 가중 |

> 핵심: 10대 원칙의 근본은 하나입니다 — **AI는 도구이고, 책임은 사람에게 있다.** "AI가 그렇게 말했습니다"는 어떤 상황에서도 면책 사유가 되지 않습니다.

### 사례 2: 뉴욕·밴쿠버 변호사 허위 판례 인용 (2023~2024)

| 항목 | 내용 |
|------|------|
| **기관** | 뉴욕 Levidow, Levidow & Oberman (2023), 캐나다 밴쿠버 변호사 (2024) |

### 사례 3: 에어캐나다 챗봇 허위 할인 안내 (2024)

| 항목 | 내용 |
|------|------|
| **기관** | Air Canada |
| **사건 개요** | AI 챗봇이 사별 할인 환불 정책을 **허위로 안내**. 실제 존재하지 않는 할인 정책을 고객에게 약속 |

## 6. KDB생명형 AI 안전 활용 의사결정 트리

AI를 활용하기 전에, 아래 의사결정 트리를 따라 판단하세요.

```
[시작] 이 업무에 AI를 활용하려 한다
    │
    ▼
[Q1] 이 업무에 보험계약자 의료정보·건강정보·개인정보·미출시 상품정보가 포함되어 있는가?
    │
    ├─ YES ──▶ [Q1-1] 해당 정보를 제거/익명화하고도 AI 활용이 가능한가?
    │              ├─ YES ──▶ 정보를 제거/익명화 후 Q2로 이동
    │              └─ NO ───▶ ⛔ AI 활용 불가. 수작업 처리
    │
    └─ NO ───▶ Q2로 이동
    │
    ▼
[Q2] 어떤 AI 도구를 사용할 것인가?
    │
    ├─ [사내 승인 AI] ──▶ Q3로 이동
    ├─ [외부 상용 AI (ChatGPT, Claude 등)]
    │       └─▶ [Q2-1] 입력할 정보가 공개 정보(경영공시, DART 공시, 보도자료, 일반 지식)인가?
    │              ├─ YES ──▶ Q3로 이동
    │              └─ NO ───▶ ⛔ 외부 AI 사용 금지. 사내 AI 또는 수작업 처리
    │
    └─ [미승인 AI] ──▶ ⛔ 사용 금지
    │
    ▼
[Q3] AI 결과물의 용도는 무엇인가?
    │
    ├─ [내부 참고·아이디어 수집] ──▶ 기본 검증 (1단계 팩트체크)
    ├─ [내부 보고서·분석 자료] ──▶ 표준 검증 (팩트체크 1~2단계)
    ├─ [대외 제출·고객 대면·의사결정 근거] ──▶ 심층 검증 (팩트체크 1~3단계 전체)
    │
    ▼
[Q4] 검증 완료 후 최종 판단
    │
    ├─ 검증 통과 ──▶ ✅ 활용 (AI 활용 사실·검증 이력 기록)
    └─ 검증 미통과 ──▶ 수정 또는 재작성 후 재검증
```

**업무 중요도별 검증 수준 요약**

| 업무 중요도 | 예시 | 검증 수준 | 소요 시간 |
|------------|------|----------|----------|
| **낮음** | 일반 자료 초벌 요약, 브레인스토밍, 영문 번역 | 1단계(근거 요구) | 5분 이내 |
| **중간** | 보험시장 분석 초안, 내부 검토 보고서, 회의 자료 | 1~2단계(근거 요구 + 핵심 팩트 점검) | 15~30분 |
| **높음** | 인수심사 의견서, 보험금 심사 보고, K-ICS 건전성 보고서 | 1~3단계 전체(원출처 교차확인 + 전문가 검증) | 1시간 이상 |
| **최고** | 금감원 제출 자료, 이사회 보고, 경영공시 문서 | AI 직접 활용 지양. 사용 시 3단계 + 준법감시·보험계리 검토 | 반나절 이상 |

> 팁: "AI로 시간을 절약했다"는 검증 시간까지 포함한 총시간으로 판단하세요. 고중요도 업무에서는 AI 초안 작성(10분) + 검증(2시간) = 총 130분이 될 수 있습니다. 그래도 백지에서 시작하는 것보다 낫지만, **검증 시간을 미리 확보**해야 합니다.

## 7. 즉시 적용 체크리스트 (1페이지 요약)

### AI 사용 전 점검 5항목

- [ ] 이 업무에 보험계약자 의료정보·건강정보·개인정보·미출시 상품정보가 포함되어 있는가? → 포함 시 제거/익명화 또는 AI 사용 포기
- [ ] 사내 승인된 AI 도구를 사용하고 있는가? → 미승인 도구 사용 금지
- [ ] 입력할 정보의 민감도를 확인했는가? → 외부 AI에는 공개 정보만 입력
- [ ] AI 결과물의 검증에 필요한 시간을 확보했는가? → 마감 직전 AI 활용은 검증 부실 초래
- [ ] 이 업무에 맞는 검증 수준(1~3단계)을 사전에 결정했는가?

### AI 사용 중 점검 5항목

- [ ] 프롬프트에 출처·근거 요구 조건을 포함했는가?
- [ ] "확실하지 않은 부분은 '확인 필요'로 표시하라"고 지시했는가?
- [ ] AI가 단정적으로 서술하는 보험 법령·통계 수치를 발견하면 즉시 의심하고 있는가?
- [ ] 대화가 길어질수록 맥락 혼동이 증가함을 인지하고, 주기적으로 새 대화를 시작하고 있는가?
- [ ] AI 답변 중 "지나치게 구체적인 K-ICS 수치", "완벽한 형식의 보험업법 인용"을 경고 신호로 인식하고 있는가?

### AI 결과 활용 전 점검 5항목

- [ ] 모든 보험업법·보험업감독규정 인용을 국가법령정보센터(law.go.kr) 또는 금감원 법규검색에서 원문 대조했는가?
- [ ] 모든 보험 통계·K-ICS 수치를 보험개발원, 생명보험협회, 금감원 보험통계와 교차검증했는가?
- [ ] AI가 언급한 보험사·기관·보고서가 실재하는지 확인했는가?
- [ ] 업무 중요도에 맞는 검증 단계(1~3단계)를 모두 완료했는가?
- [ ] 대외 제출 문서의 경우, AI 활용 사실과 검증 이력을 기록했는가?

### 사고 발생 시 대응 5단계

| 단계 | 행동 | 담당 |
|------|------|------|
| **1단계: 즉시 보고** | 정보보안팀·준법감시팀에 사고 내용 즉시 보고 | 사고 발견자 |
| **2단계: 피해 범위 확인** | 어떤 정보가(의료정보·보험계약정보 등), 어떤 AI에, 언제 입력되었는지 파악 | 정보보안팀 |
| **3단계: 확산 방지** | 해당 AI 서비스 접속 차단, 관련 계정 비밀번호 변경 | IT 보안팀 |
| **4단계: 영향 평가** | 유출 정보의 성격·규모, 보험계약자 피해 여부, 법적 리스크 평가 | 준법감시팀 + 법무팀 |
| **5단계: 시정 조치** | 감독기관 보고(필요 시), 보험계약자 통지(필요 시), 재발 방지 대책 수립, 관련자 교육 | 경영진·관련 부서 |

> 주의: 사고 은폐는 사고 자체보다 훨씬 큰 리스크를 초래합니다. 즉시 보고가 피해를 최소화하는 가장 확실한 방법입니다.

## 학습 정리

### 핵심 요약

- **보험업 AI 리스크는 특별하다**: 금융소비자 보호 의무, 보험계약자 의료정보의 민감성, 인수심사·보험금심사의 공정성, 감독기관의 감시 강화가 복합적으로 작용한다
- **할루시네이션은 "가끔 발생하는 오류"가 아니다**: 현행 LLM 아키텍처의 본질적 특성으로, 완전 제거가 불가능하다. 팩트체크 프로세스가 유일한 대응책이다
- **팩트체크 3단계를 습관화하라**: (1) AI에게 근거 요구 → (2) 핵심 팩트 전수 점검 → (3) 권위 있는 소스 교차확인
- **AI기본법(2026.01 시행)은 보험업을 고영향 AI로 분류한다**: KDB생명의 모든 AI 활용에 강화된 관리 의무가 적용된다
- **AI는 도구이고, 책임은 사람에게 있다**: "AI가 그렇게 말했습니다"는 어떤 상황에서도 면책 사유가 되지 않는다

### 즉시 적용 체크리스트

- [ ] 외부 AI에 보험계약자 의료정보·건강정보·미출시 상품정보 입력하지 않기
- [ ] 모든 AI 결과물에 팩트체크 3단계 적용하기
- [ ] 보험 법령·통계 수치는 반드시 원출처에서 확인하기
- [ ] 사내 승인된 AI 도구만 사용하기
- [ ] AI 사고 발생 시 즉시 보고하기

### 다음 단계

이 모듈에서 학습한 안전 원칙은 모듈 1~5에서 배운 모든 AI 활용 기법의 **기반**입니다. 프롬프트를 아무리 잘 설계해도, 검증 없이 결과를 사용하면 리스크가 커집니다. **"AI를 잘 쓰는 것"과 "AI를 안전하게 쓰는 것"은 분리할 수 없는 하나의 역량**입니다.
