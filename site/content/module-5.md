## 1. 할루시네이션이란?

### 1.1 정의

**할루시네이션(Hallucination)** 이란 AI가 사실이 아닌 정보를 마치 사실인 것처럼 자신 있게 생성하는 현상입니다. AI는 "모르겠습니다"라고 말하는 대신, 그럴듯한 거짓말을 만들어냅니다.

> 🔑 핵심: AI는 "확률적으로 그럴듯한 다음 단어"를 예측하는 모델입니다. 사실 여부를 판단하는 것이 아니라, **그럴듯해 보이는 문장**을 생성하는 것이 본질입니다.

### 1.2 할루시네이션의 4가지 유형

| 유형 | 설명 | 위험도 | 실무 예시 |
|------|------|--------|----------|
| **사실 조작** | 존재하지 않는 사건, 인물, 데이터를 생성 | ⭐⭐⭐⭐⭐ | "A기업이 2026년 3분기 5,000억 원 규모 회사채를 발행했다"(실제 미발행) |
| **출처 날조** | 존재하지 않는 논문, 판례, 보고서를 인용 | ⭐⭐⭐⭐⭐ | "대법원 2023다12345 판결에 따르면..."(존재하지 않는 판례) |
| **수치 왜곡** | 통계, 재무 데이터, 신용지표를 잘못 제시 | ⭐⭐⭐⭐ | "B기업 DSCR이 1.5배"(실제 1.2배), "한은 기준금리 3.75%"(실제 상이) |
| **법률 인용 오류** | 법 조항 번호, 내용을 잘못 인용하거나 혼합 | ⭐⭐⭐⭐⭐ | "자본시장법 제42조에 따르면..."(실제 내용과 상이) |

### 1.3 발생 원인

1. **통계적 언어 모델의 본질적 한계**: LLM은 "다음에 올 확률이 높은 토큰"을 예측합니다. 정답을 검색하는 것이 아니라, 패턴을 생성하는 것입니다.
2. **학습 데이터의 편향과 오류**: 인터넷의 부정확한 정보가 학습 데이터에 포함되어 있습니다.
3. **지식 컷오프**: 학습 시점 이후의 법률 개정, 최신 데이터를 반영하지 못합니다.
4. **과잉 확신(Overconfidence)**: 확률이 낮은 출력에도 확신 있는 어투를 사용합니다.
5. **맥락 혼동**: 긴 대화나 복잡한 질문에서 앞뒤 맥락을 혼합합니다.

> 💡 팁: 2026년 수학적 증명을 통해 "현재 LLM 아키텍처에서 할루시네이션은 완전히 제거할 수 없는 본질적 특성"임이 확인되었습니다. 따라서 **검증 프로세스**를 갖추는 것이 유일한 해법입니다.

## 2. 실제 사고 사례

> ⚠ 주의: 아래 사례들은 "AI를 안 쓰면 된다"는 교훈이 아닙니다. "AI를 쓰되, 반드시 검증해야 한다"는 교훈입니다.

### 2.1 법률 분야 — 가짜 판례 제출 사건

**사건 개요**: 2023년 뉴욕의 변호사 Steven Schwartz는 ChatGPT가 생성한 6건의 가짜 판례를 연방법원에 제출했습니다. AI가 만들어낸 판례에는 실존하지 않는 사건명, 판결문 인용, 법원명이 포함되어 있었습니다.

**결과**: 법원은 변호사에게 5,000달러의 제재금을 부과했습니다.

**2026년 확산**: 이 문제는 줄어들기는커녕 급증하고 있습니다. 2026년 기준 전 세계 법원에 제출된 AI 날조 인용 사례가 1,200건 이상으로 집계되었으며, 2026년 1분기에만 미국 법원이 부과한 제재금이 14만 5천 달러를 넘었습니다. 6차 순회 항소법원은 테네시주 변호사들이 3건의 항소에서 20개 이상의 가짜 인용을 제출한 사건에 대해 역대 최고 수준의 제재를 부과했습니다.

| 연도 | 주요 사건 | 제재 내용 |
|------|----------|----------|
| 2023 | Mata v. Avianca (뉴욕) | $5,000 제재금 |
| 2026 Q1 | Noland v. Land of the Free | 23개 중 21개 인용이 날조로 판명 |
| 2026 | 6th Circuit 통합 항소 | 20개 이상 가짜 인용, 역대 최고 제재 |

> 🔑 핵심: 하루에 5~6건의 새로운 AI 날조 인용 사례가 발견되고 있습니다. "AI가 만든 법률 정보는 100% 검증 대상"이라는 원칙을 세우세요.

### 2.2 금융 분야 — AI가 허위 재무제표·규제 조항을 생성한 사례

2026년 1분기, 글로벌 IB에서 AI 분석 도구의 할루시네이션으로 인한 잘못된 실적 전망이 시장 전반에 영향을 미쳤습니다. FINRA(미국 금융산업규제기관)는 2026년 감독 보고서에서 AI 할루시네이션과 편향을 주요 리스크로 지목하며, 증권사·은행들에게 AI 할루시네이션 탐지 절차를 의무화할 것을 권고했습니다.

**실무에서 발생하는 금융 할루시네이션 유형:**
- AI가 **존재하지 않는 기업 재무제표 항목**을 그럴듯하게 생성 (예: 매출 4,200억, 영업이익 380억 — 실제 공시 수치와 상이)
- **신용등급을 한 단계 상향**하여 서술 (예: "AA"가 실제로는 "A")
- **크레딧 스프레드·회사채 금리** 허위 인용 (예: "AA 3년물 스프레드 45bp" — 실제 70bp)
- **한은 기준금리·환율** 허위 수치 ("기준금리 3.25%" — 실제 상이)
- **폐지된 규제 조항**을 현행 법규처럼 인용 (예: 자본시장법 구 조항)
- **존재하지 않는 판례**로 여신 회수 가능성 판단 근거 제시

**실제 리스크 시나리오 예시:**
- 여신심사역이 AI에게 "C기업의 최근 3개년 영업현금흐름 추이"를 요청 → AI가 DART 공시 없이 수치를 꾸며냄 → 심사보고서에 반영 → 부실 대출 승인
- PF 심사 시 AI에게 "발전 프로젝트 DSCR 벤치마크"를 요청 → AI가 존재하지 않는 사례의 DSCR 1.45배를 인용 → 과도하게 낙관적 심사 결론
- 애널리스트가 AI에게 "2차전지 업종 주요 기업 수주잔고"를 요청 → AI가 IR 자료에 없는 숫자를 생성 → 산업분석 리포트가 왜곡

> ⚠ 주의: 금융 규제 위반이나 허위 재무정보 인용은 과징금, 영업정지, 형사처벌, 손해배상으로 이어질 수 있습니다. AI가 제시한 **재무 수치는 DART**, **신용등급은 KIS/NICE 원본 리포트**, **금리·환율은 한은 ECOS**, **법규는 금융감독원 법규 검색 또는 법제처 국가법령정보센터**에서 반드시 원문을 확인하세요.

### 2.3 의료 분야 — 건강 정보 오류

2026년 연구에 따르면, LLM이 의료 질의에 대해 할루시네이션을 일으킨 비율이 짧은 질의에서 67.6%, 긴 질의에서 64.1%에 달했습니다. ECRI(글로벌 의료안전 비영리기관)는 AI 리스크를 2026년 의료기술 위험요소 1위로 선정했습니다.

**위험 사례:**
- AI가 잘못된 약물 복용량을 제시
- 존재하지 않는 의학 저널 논문을 인용
- 질병 간 연관성을 잘못 생성

### 2.4 은행·여신 실무 — 허위 재무·시장 데이터 사고

**시나리오**: 심사역이 AI에게 "D기업의 최근 유동비율·부채비율, 동종업계 평균 대비 수준"을 물었을 때, AI가 실제 DART 공시와 다른 수치를 제시하는 경우가 발생합니다.

**위험 사례:**
- AI가 **유동비율을 180%로 제시** → 실제 95% → 유동성 위험 간과, 대출 부실화
- AI가 **동종업계 평균 부채비율을 120%로 제시** → 실제 210% → 상대 평가 왜곡
- **존재하지 않는 KIS/NICE 신용평가 의견서**를 인용 → 내부 신용등급 오판
- **폐지된 여신전문금융업법 조항**을 현행으로 제시 → 여신 약정서 오류
- AI가 **한은 기준금리 인상 시점을 잘못 언급** → 금리 리프라이싱 시점 오판 → 금리 리스크 확대
- AI가 **존재하지 않는 크레딧 스프레드 데이터**를 제시 → 회사채 인수 가격 산정 오류

> 💡 팁: KDB 심사·IB·리서치 현장에서 AI가 제시한 재무 수치·신용지표·시장 데이터는 **반드시 원본 소스(DART, ECOS, KIS Value, Bloomberg)와 교차 검증**하세요. 수치 하나가 여신 승인·가격 산정·투자 결정의 부실로 직결될 수 있습니다.

## 3. AI가 자신 있게 거짓을 말하는 메커니즘

### 3.1 통계적 언어 모델의 작동 원리

```
[사람의 이해 방식]
"자본시장법 제42조" → 법령 DB에서 해당 조항 검색 → 원문 확인

[AI의 작동 방식]
"자본시장법 제" → "42" (학습 데이터에서 자주 등장한 숫자)
"42조에 따르면" → "금융투자업자는..." (통계적으로 그럴듯한 연결)
→ 실제 42조의 내용과 무관하게, "그럴듯한 문장"이 완성됨
```

AI는 **검색 엔진이 아닙니다**. 데이터베이스에서 정보를 찾아오는 것이 아니라, "이 맥락 다음에 올 확률이 가장 높은 단어"를 순서대로 생성합니다. 그래서 조항 번호, 수치, 날짜처럼 **구체적인 팩트**에서 오류가 발생합니다.

### 3.2 학습 데이터 편향의 영향

| 편향 유형 | 설명 | 결과 |
|----------|------|------|
| 빈도 편향 | 자주 등장한 정보를 우선 생성 | 인기 있는(하지만 부정확한) 통설을 반복 |
| 시간 편향 | 학습 데이터 시점의 정보만 반영 | 개정된 법률의 이전 버전을 인용 |
| 출처 편향 | 블로그·뉴스의 해석을 원문처럼 제시 | 2차 자료를 1차 출처로 오인 |
| 혼합 편향 | 여러 출처의 정보를 합성 | A법의 조항 번호 + B법의 내용이 결합 |

### 3.3 할루시네이션 경고 신호

다음과 같은 패턴이 나타나면 할루시네이션을 의심하세요:

1. **지나치게 구체적인 숫자**: "정확히 DSCR 1.473배" — 출처 없이 소수점까지 제시
2. **완벽한 형식의 인용**: "대법원 2024다56789 판결문" 또는 "KIS 2026-03 리포트 p.17" — 형식은 완벽하지만 실존하지 않음
3. **단정적 어투**: "반드시 ~해야 합니다", "법적으로 의무입니다" — 검증 없는 단정
4. **맥락과 동떨어진 정보**: 질문과 관련 없는 세부 정보를 장황하게 서술
5. **최신 데이터 주장**: "2026년 4월 한은 금통위 결정에 따르면..." — 학습 데이터 범위 밖의 시점

## 4. 팩트체크 3단계

### Step 1: AI 출처가 실제 존재하는지 확인

```
[확인 방법 — KDB 실무 기준]
1. AI가 인용한 법령 → 국가법령정보센터(law.go.kr), 금융감독원 법규 검색
2. AI가 인용한 논문 → Google Scholar, RISS, SSRN에서 검색
3. AI가 인용한 판례 → 대법원 종합법률정보에서 검색
4. AI가 인용한 기업 재무 수치 → DART(dart.fss.or.kr) 공시 원문
5. AI가 인용한 신용등급·의견 → KIS Value, NICE평가정보 원본 리포트
6. AI가 인용한 금리·환율·경제통계 → 한국은행 ECOS(ecos.bok.or.kr)
7. AI가 인용한 산업 통계 → 산업연구원(KIET), 통계청, KDB 미래전략연구소
8. AI가 인용한 뉴스 → 해당 언론사 사이트에서 직접 검색
```

> 💡 팁: Perplexity는 모든 주장에 인라인 출처 링크를 제공합니다. 출처 링크를 클릭하여 실제 페이지가 존재하는지부터 확인하세요. 링크가 404 오류를 반환하면 날조된 출처입니다.

### Step 2: 원문을 찾아서 AI 요약과 대조

```
[대조 체크리스트 — KDB 심사·리서치 기준]
□ 조항 번호(법령) / 공시 번호(DART) / 리포트 ID(KIS·NICE)가 정확한가?
□ 조항·공시·리포트의 내용이 AI 요약과 일치하는가?
□ 수치(재무, 신용지표, 금리, 환율, 금액)가 원문과 같은가?
□ 적용 대상(업종, 기업, 기간)이 올바른가?
□ 시행일·공시일·기준일이 최신인가?
```

**실전 프롬프트 — Claude로 원문 대조**

```
[재무제표·법령 원문 대조 프롬프트]

아래 두 텍스트를 비교하여 차이점을 찾아주세요.

[텍스트 A — AI가 생성한 요약 (재무 요약 / 법규 요약)]
(여기에 AI가 생성한 내용을 붙여넣기)

[텍스트 B — 원문]
(여기에 DART 공시 원문 / law.go.kr 조항 원문 / KIS·NICE 신용평가 원본을 붙여넣기)

비교 항목:
1. 수치(매출, 영업이익, 부채비율, DSCR, 신용등급 등)의 일치 여부
2. 조항 번호·공시 번호의 정확성
3. 내용의 일치 여부 (누락, 추가, 왜곡)
4. 기준일·공시일·시행일의 일치 여부

차이가 있는 부분을 표로 정리하고, 각 차이의 심각도를
"치명적 / 중요 / 경미"로 분류해주세요.
```

### Step 3: 전문가/법무팀 최종 검증

| 검증 대상 | 검증 주체 | 검증 방법 |
|----------|----------|----------|
| 법률·규제 조항 | 준법감시실/법무팀 | 원문 대조 + 최신 개정 확인 |
| 기업 재무 데이터 | 여신심사부/리스크관리부 | DART 공시·감사보고서 대조 |
| 신용등급·신용평가 의견 | 여신심사부/리스크관리부 | KIS/NICE 원본 리포트 대조 |
| 금리·환율·경제지표 | 리서치부/자금부 | 한은 ECOS, Bloomberg 대조 |
| PF 사업성·DSCR 가정 | 프로젝트금융부 | 재무 모델 원본·민감도 분석 대조 |
| ESG 평가·탄소 데이터 | 지속가능경영실 | MSCI/KCGS/서스틴베스트 원본 대조 |

> ⚠ 주의: "AI가 확인해줬으니 맞겠지"는 가장 위험한 판단입니다. AI의 자신감 있는 어투에 속지 마세요. **최종 책임은 항상 사람에게 있습니다.**

## 5. 보고서 작성 시 할루시네이션 제거 체크리스트

심사보고서·산업분석 리포트·PF 검토 메모를 AI로 작성한 후, 결재 상정 전에 아래 10개 항목을 반드시 점검하세요.

| # | 점검 항목 | 점검 방법 | 통과 기준 |
|---|----------|----------|----------|
| 1 | **인용된 법령·규정이 실존하는가?** | 국가법령정보센터·금감원 법규에서 조항 번호 검색 | 원문 확인 완료 |
| 2 | **재무 수치의 출처가 명시되어 있는가?** | 각 수치 옆에 (출처: DART 2026-03-15 감사보고서 등) 표기 확인 | 모든 수치에 출처 존재 |
| 3 | **재무 수치가 DART 공시 원문과 일치하는가?** | DART 원본 감사보고서/사업보고서와 대조 | 오차 없음 |
| 4 | **신용등급·스프레드·금리가 원본과 일치하는가?** | KIS/NICE 원본, 한은 ECOS, Bloomberg와 대조 | 오차 없음 |
| 5 | **인용된 논문/산업 리포트가 실존하는가?** | Google Scholar, KDB·산업연구원 홈페이지에서 검색 | 실제 문서 확인 |
| 6 | **법령의 시행일·개정 이력이 최신인가?** | 법령 시행일 확인, 최근 개정 여부 확인 | 현행 법령 기준 |
| 7 | **AI가 단정적으로 서술한 부분이 있는가?** | "반드시", "의무", "위반 시" 등 단정 표현 검색 | 전문가 확인 완료 |
| 8 | **경쟁사/타 여신처 정보가 정확한가?** | DART 공시, 공식 IR 자료로 대조 | 공식 출처 확인 |
| 9 | **PF·M&A 가정치가 원본 모델과 일치하는가?** | 재무 모델·민감도 분석 원본과 대조 | 사내 데이터 기준 |
| 10 | **면책 문구가 포함되어 있는가?** | "본 보고서의 법률·재무 정보는 참고용이며..." 문구 확인 | 면책 문구 삽입 완료 |

> 💡 팁: 이 체크리스트를 프롬프트에 포함시키면 AI가 자체적으로 점검한 결과를 함께 제공합니다. 하지만 **AI의 자체 점검만으로는 부족**합니다. 사람이 직접 확인하는 단계가 반드시 필요합니다.

**AI 결과 (개선된 응답):**

## 8. 기술적 대응 방법

### 8.1 RAG (Retrieval-Augmented Generation, 검색 증강 생성)

AI가 답변을 생성하기 전에, 관련 문서를 **검색**하여 참고 자료로 제공하는 방식입니다.

```
[RAG 없이 질문]
사용자: "D기업의 최근 3년 영업현금흐름 추이는?"
AI: (학습 데이터에서 추론) → 할루시네이션 위험 높음

[RAG 적용]
사용자: "D기업의 최근 3년 영업현금흐름 추이는?"
시스템: (사내 여신 DB + DART 미러링 DB에서 해당 기업
         감사보고서 검색 → AI에게 원문 전달)
AI: (원문을 참고하여 답변 생성) → 정확도 대폭 향상
```

**KDB 사내 적용**: KDB 사내 LLM에 RAG를 구축하면, **내부 심사보고서·산업분석 리포트·여신 DB·시장 데이터(한은 ECOS 미러링)·내규 및 여신 운용 기준**을 참조하여 답변을 생성할 수 있습니다. 이를 통해:
- 심사역은 **과거 유사 여신 건 심사보고서**를 빠르게 검색하여 참고
- 애널리스트는 **KDB 미래전략연구소·산업분석부 누적 리포트**를 기반으로 산업 전망 작성
- IB·PF 심사역은 **과거 유사 딜의 DSCR·보증구조·가격 조건**을 근거로 검토

### 8.2 Grounding (근거 기반 생성)

AI가 생성한 각 문장에 대해 **근거 문서의 위치**를 표시하는 방식입니다. Google의 Gemini API에서 "Grounding with Google Search" 기능으로 제공됩니다.

### 8.3 출처 표시 활용 — Perplexity 인라인 인용

Perplexity는 모든 주요 주장에 대해 출처를 인라인으로 표시합니다. 법규·재무 공시·시장 지표·판례를 검색할 때 가장 먼저 활용하기 좋은 도구입니다.

```
[Perplexity 활용 프롬프트]
2026년 2차전지 산업의 자본시장법상 공시 의무 변경 사항과
주요 대기업 발행 회사채의 최근 스프레드 추이를 정리해주세요.
각 주장에 대해 원문 확인이 가능한 출처 링크(DART, 금감원,
한은 ECOS 등)를 포함해주세요.
```

### 8.4 실무 활용 조합

| 상황 | 추천 접근법 |
|------|-----------|
| 법규·규제 조사 초안 | Perplexity (출처 링크 확인) → 법령·금감원 원문 직접 대조 |
| 사내 심사·산업분석 보고서 | KDB 사내 LLM + RAG (내부 심사 아카이브·여신 DB 연동) |
| 기업 재무·신용 비교 분석 | AI로 프레임워크 생성 → DART·KIS/NICE·Bloomberg에서 수치 직접 입력 |
| 산업·시장 동향 리서치 | ChatGPT Deep Research 또는 Gemini Deep Research → 산업연구원·KDB 리포트로 교차 확인 |

### 8.5 금융권 보안·컴플라이언스 요구사항

KDB 임직원이 AI를 활용할 때는 금융권 특유의 보안·내부통제 요구를 반드시 고려해야 합니다.

| 요구사항 | 내용 | 실무 영향 |
|---------|------|----------|
| **신용정보법·개인정보보호법** | 고객 기업의 대표자 개인정보, 거래 정보는 법적 보호 대상 | 외부 AI(ChatGPT, Claude 웹 등)에 입력 금지. 사내 LLM만 사용 |
| **금융회사 IT안전성 확보 기준** | 고객 정보·여신 정보 처리 시스템은 강화된 통제 | 외부 AI API 호출 시 전용회선·보안검토 거쳐야 함 |
| **금융권 클라우드 컴플라이언스(전자금융감독규정)** | 중요도 등급에 따른 클라우드 이용 절차 | 사내 LLM의 AWS/Azure 호스팅 시 사전 신고·보안성 심의 필수 |
| **KDB 내부통제 기준·정보보호 지침** | 여신 심사 정보·M&A 기밀·PF 비공개 정보 취급 | 사내 LLM 접근권한 분리, 로그 보관, 질의·응답 이력 감사 |
| **외부 유출 금지 정보** | 미공시 재무정보, 여신 신청서, 내부 신용등급, 딜 클라이언트 정보 | 외부 AI에 입력 시 내부통제 위반 → 징계·고발 대상 |

> ⚠ 주의: "테스트 삼아 외부 ChatGPT에 여신 신청서 일부를 붙여넣었다"는 행위 하나로도 신용정보법 위반·KDB 내부통제 기준 위반이 됩니다. **고객사·딜·여신 관련 정보는 반드시 KDB 사내 LLM에서만 처리**하세요.

> 💡 팁: 사내 LLM 사용 시에도 **로그가 남는다**는 점을 인지하고, 질의 내용이 감사·내부통제 점검에 적절한지 한 번 더 확인하세요. RAG로 사내 문서를 검색할 때도 **접근 권한이 있는 자료만 결과로 반환**되도록 권한 모델이 설계되어 있는지 확인해야 합니다.

## 9. 실습 & 퀴즈

### 퀴즈 1 — 할루시네이션 유형 식별

Q. AI가 "대법원 2024다56789 판결에서 사업주의 안전의무 위반을 인정하였습니다"라고 답변했습니다. 이것이 할루시네이션인지 확인하기 위해 **가장 먼저** 해야 할 행동은?

(A) AI에게 "이 판례가 맞는지 다시 확인해줘"라고 재질문한다
(B) 대법원 종합법률정보 사이트에서 해당 사건 번호를 검색한다
(C) 다른 AI 모델에게 같은 질문을 해서 답변을 비교한다
(D) 판례 번호 형식이 올바른지 확인한다

**정답**: (B) — AI에게 재질문하면 같은 할루시네이션을 반복하거나 새로운 거짓을 생성할 수 있습니다. 원본 소스(대법원 종합법률정보)에서 직접 확인하는 것이 팩트체크 1단계의 핵심입니다.

### 퀴즈 2 — 팩트체크 우선순위

Q. AI가 작성한 여신심사보고서 초안에서 다음 중 **팩트체크 우선순위가 가장 높은** 항목은?

(A) "최근 2차전지 시장이 성장하고 있습니다" — 일반적 트렌드 서술
(B) "자본시장법 시행령 제87조에 따르면 반기마다 공시가 의무입니다" — 구체적 법령 인용
(C) "차주 기업의 사업 모델은 안정적입니다" — 정성적 평가
(D) "경쟁사도 유사한 사업 확장을 검토 중입니다" — 업계 동향

**정답**: (B) — 구체적인 조항 번호와 의무 사항이 명시된 법령 인용은 할루시네이션 위험이 가장 높고, 오류 시 법적·여신 리스크가 가장 큽니다. 숫자, 조항, 의무 표현이 포함된 문장일수록 우선 검증하세요.

## 핵심 요약

| 번호 | 핵심 포인트 | 실전 적용 |
|------|-----------|----------|
| 1 | AI 할루시네이션은 버그가 아니라 **구조적 한계** | "AI 결과 = 초안"이라는 마인드셋 유지 |

| 7 | 금융권 보안·내부통제를 최우선으로 | 고객·여신·딜 정보는 사내 LLM에서만 처리, 외부 AI 입력 금지 |
