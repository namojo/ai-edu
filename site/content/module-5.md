# 모듈 5: AI 리서치와 보험 전문자료 탐색

> **대상**: KDB생명 임직원
> **학습 시간**: 약 50~60분
> **핵심 키워드**: AI 검색 전략, DART·보험개발원·국가법령정보, 보험업 전문자료 탐색, 팩트체크·출처 확보, Perplexity, 유료 도구 병행

---

## 학습 목표
1. KDB생명 실무에서 자주 찾는 자료 유형과 주요 리소스를 체계적으로 파악할 수 있다
2. AI 검색 전략 3가지(최신 정보 검색, 리소스 탐색, 자료 분석)를 목적에 맞게 활용할 수 있다
3. 실전 검색 시나리오 5가지를 자신의 업무에 응용할 수 있다
4. 보험업 팩트체크 3단계를 적용하여 AI 검색 결과의 신뢰도를 확보할 수 있다

## 2. AI 검색 전략 3가지

KDB생명 실무에서 AI를 활용한 리서치는 크게 3가지 전략으로 나뉩니다.

### 전략 1: Perplexity·SearchGPT로 최신 정보 찾기

**적합 상황**: 최근 뉴스, 보험시장 동향, 규제 변화 등 최신 정보가 필요할 때

Perplexity와 SearchGPT는 실시간 웹 검색을 기반으로 답변하며, **인라인 출처 링크를 함께 제공**하는 것이 핵심 장점입니다.

```
[Perplexity 프롬프트 예시]

2026년 1분기 국내 생명보험 시장의 주요 이슈를 정리해줘.

포함할 내용:
1. 주요 생보사별 실적 동향 (삼성생명, 한화생명, 교보생명)
2. K-ICS 규제 변화 및 보험사 건전성 영향
3. IFRS 17 도입 이후 CSM 관리 현황
4. 보장성 보험 시장 성장 동향
5. 디지털 헬스케어·인슈어테크 트렌드

각 항목에 출처와 날짜를 반드시 표시해줘.
```

> 팁: Perplexity는 금융권 특화 기능(Perplexity Finance)으로 SEC 파일링, 실적 콜 데이터, FactSet 연동 등을 지원합니다. 해외 보험사 리서치 시 특히 유용합니다. (출처: Perplexity 공식, 2026)

### 전략 2: ChatGPT/Claude에게 "어디서 찾을 수 있나" 묻기

**적합 상황**: 특정 자료가 어디에 있는지 모를 때, 리소스 탐색이 필요할 때

AI는 방대한 지식 기반을 가지고 있어, **자료의 소재지를 안내**하는 데 탁월합니다.

```
[리소스 탐색 프롬프트 예시]

나는 KDB생명의 보험계리사입니다.
신규 건강보험 상품 설계를 위해 아래 자료가 필요합니다.

각 자료를 어디서 구할 수 있는지 안내해주세요:
1. 최근 5년간 경험생명표 및 질병 발생률 데이터
2. 국내 건강보험 시장 규모 및 경쟁사별 점유율
3. K-ICS 기준 건강보험 위험계수
4. 보험업감독규정상 건강보험 관련 인가 요건
5. 해외 건강보험 트렌드 (디지털 헬스케어 연계형)

각 자료에 대해:
- 가능한 소스 2~3개
- 유료/무료 여부
- 접근 방법 (URL, 검색 키워드)
- 대안 자료 (직접 입수가 어려운 경우)
```

> 핵심: AI에게 "자료를 찾아줘"보다 "자료가 어디에 있는지 알려줘"가 더 효과적입니다. AI는 직접 데이터베이스를 검색하지 못하지만, 어떤 데이터베이스에 어떤 정보가 있는지는 잘 안내합니다.

### 전략 3: AI에게 자료 넣고 요약·비교 시키기

**적합 상황**: 이미 확보한 자료를 빠르게 분석·비교해야 할 때

보험개발원 통계, 생명보험협회 공시, 경영공시 자료 등을 AI에 직접 입력하여 구조화된 분석 결과를 얻습니다.

```
[분석 프롬프트 예시]

아래는 A생명과 B생명의 최근 경영공시에서 발췌한 주요 지표입니다.

[A생명 경영공시]
(여기에 붙여넣기)

[B생명 경영공시]
(여기에 붙여넣기)

요청:
1. 두 보험사의 핵심 경영지표 비교표 (수입보험료, 손해율, 사업비율, K-ICS 비율, CSM)
2. 3개년 추이 비교 (어느 쪽이 개선/악화 추세인지)
3. 재무건전성 관점에서 각 보험사의 강점/약점 (표 형식)
4. 두 보험사 중 보장성 보험 경쟁력이 더 높은 쪽과 그 근거

조건:
- 수치는 원자료 그대로 인용
- AI 판단 부분은 [분석]으로 구분
- 결론적 추천은 하지 말고, 판단 근거만 제시
```

> 주의: 외부 AI에 자료를 입력할 때는 반드시 **공개된 자료(경영공시, DART 공시 등)만 사용**하세요. 보험계약자 개인정보, 의료정보, 인수심사 기록, 미출시 상품 설계서는 외부 AI에 절대 입력하면 안 됩니다.

## 3. 실전 검색 시나리오 5가지

### 시나리오 1: 건강보험 시장 트렌드 및 신상품 기획 리서치

**상황**: KDB생명 상품개발팀에서 2026년 건강보험 신상품을 기획하기 위해, 시장 트렌드와 경쟁사 상품 현황을 분석해야 합니다.

**1단계: Perplexity로 최신 동향 수집**
```
2025~2026년 국내 생명보험 시장의 건강보험 트렌드를 정리해줘.

포함할 항목:
1. 건강보험 시장 규모 및 성장률 (최근 3년)
2. 주요 생보사(삼성생명, 한화생명, 교보생명) 건강보험 주력 상품 현황
3. 디지털 헬스케어 연계 보험 상품 트렌드
4. MZ세대 타겟 미니보험·간편보험 동향
5. AI 기반 언더라이팅 도입 현황

각 항목에 출처(기관명, 날짜)를 반드시 표시해줘.
```

**2단계: Claude로 심층 분석 구조화**
```
당신은 KDB생명 상품개발팀의 상품 기획 전문가입니다.

아래는 건강보험 시장 동향 자료입니다.
(1단계 수집 결과 + 생명보험협회 경영통계 붙여넣기)

이 자료를 바탕으로 "2026년 건강보험 신상품 기획 리서치" 보고서의
근거 섹션을 작성해주세요.

구조:
1. 건강보험 시장 환경 변화 요약 (표)
2. 주요 경쟁사 상품 비교 (보장 내용, 보험료, 차별 포인트)
3. 소비자 니즈 변화 분석 (건강관리형, 실손 보완형 등)
4. KDB생명 관점의 시장 기회 3가지
5. 상품 개발 방향 시사점

톤: 분석적, 객관적, 수치 기반
```

**3단계: 후속 질문으로 심화**
```
위 분석에서 가장 성장 가능성이 높은 상품 유형은 무엇인가?
해당 유형에 대해 보험개발원 참조순보험요율 기준으로
예상 보험료 구간과 위험률 산출 시 참고할 통계를 정리해줘.
```

**최종 활용**: 건강보험 신상품 기획 보고서의 "시장 분석" 및 "상품 전략" 섹션에 반영

### 시나리오 2: K-ICS 건전성 분석 및 동종업계 비교

**상황**: KDB생명 리스크관리팀에서 자사의 K-ICS 비율을 동종업계와 비교 분석하고, 건전성 개선 방안을 검토해야 합니다.

**프롬프트**:
```
당신은 KDB생명의 리스크관리 전문가입니다.

아래는 금융감독원 보험통계에서 발췌한
주요 생명보험사의 K-ICS 관련 공시 데이터입니다.

(금감원 보험통계·경영공시에서 복사한 데이터 붙여넣기)

요청:
1. K-ICS 비율 비교표 작성
   | 보험사 | K-ICS 비율 | 기본자본 | 보완자본 | 요구자본 | 추이 | 평가 |
   
2. 핵심 지표별 분석
   - 보험위험액, 금리위험액, 시장위험액 비교
   - 경과조치 적용 전/후 K-ICS 비율 비교
   
3. 종합 건전성 평가 1문단 (리스크관리 관점)

4. 건전성 개선을 위해 추가 확인이 필요한 항목

조건: 원 데이터 수치를 그대로 사용, 억원 단위로 환산 병기
```

**후속 질문**:
```
위 분석에서 K-ICS 비율이 가장 낮은 보험사에 대해,
```

## 4. 팩트체크와 출처 확보

### 4-1. 보험업 팩트체크의 중요성

보험업에서 **출처 없는 수치 사용은 금지**입니다. AI가 생성한 결과물에 포함된 모든 수치, 법령, 보험 통계는 반드시 원문과 대조해야 합니다. AI 할루시네이션으로 잘못된 수치가 보고서에 포함되면 인수심사 오류, 보험료 산출 오류, 규제 위반, 소비자 피해로 이어질 수 있습니다.

> 참고: 2024년 캐나다에서 변호사가 ChatGPT가 생성한 가짜 판례 2건을 법정 문서에 인용하여 징계와 비용 배상 명령을 받았습니다. 보험업에서 동일한 일이 발생하면, 잘못된 위험률 기반으로 보험료를 산출하거나 보험금을 지급하는 최악의 결과로 이어질 수 있습니다. (출처: CBC News, 2024)

### 4-2. AI 결과 검증 3단계

```
[Step 1] AI 검색 — 시작점 확보
  → Perplexity/ChatGPT로 관련 정보 초안 수집
  → "출처를 반드시 표시해줘" 명시
  → "확실하지 않은 부분은 '확인 필요'로 표시해줘" 요청

     ↓

[Step 2] 원문 대조 — 사실 검증
  → 보험개발원(insure.or.kr)에서 보험 통계·위험률 원문 대조
  → 국가법령정보센터(law.go.kr)에서 보험업법·보험업감독규정 조항 확인
  → 금감원 보험통계에서 K-ICS·경영공시 수치 확인
  → 생명보험협회(klia.or.kr)에서 시장 통계 확인
  → AI 요약과 원문이 일치하는지 항목별 체크

     ↓

[Step 3] 전문가·동료 확인 — 최종 검증
  → 준법감시팀에 법률 사항 최종 확인
  → 보험계리팀에 보험 통계·위험률 데이터 크로스체크
  → 보고서에 "검증 완료/미완료" 상태 명시
  → 결재 라인 제출 전 검증 상태 최종 확인
```

### 4-3. AI가 자주 틀리는 보험업 정보 유형

| 정보 유형 | 할루시네이션 패턴 | 검증 방법 |
|----------|-----------------|----------|
| **보험 통계 수치** | 실제와 다른 손해율·사업비율·K-ICS 비율 생성 | 금감원 보험통계, 생명보험협회 경영공시 원문 대조 |
| **법령 조항 번호** | 존재하지 않는 보험업법·보험업감독규정 조항을 자신 있게 인용 | 국가법령정보센터 원문 검색 |
| **K-ICS 기준** | K-ICS 세부 기준서의 위험계수·할인율을 부정확하게 제시 | 금감원 K-ICS 기준서 원문 확인 |
| **시장 규모 통계** | 출처 불명의 보험시장 규모 수치 제시 | 보험개발원, 생명보험협회, Swiss Re sigma 확인 |
| **날짜·시점** | 과거 데이터를 최신인 것처럼 제시 | 데이터 기준일 반드시 확인 |
| **보험사 정보** | 합병·상호변경·사업 철수 등을 반영하지 못함 | DART 최신 공시, 생명보험협회 공시실 확인 |

> 주의: AI가 **구체적인 숫자를 자신 있게 제시할수록 오히려 더 의심**해야 합니다. "손해율 68.5%", "K-ICS 비율 187.2%"처럼 세부적인 수치가 나올 때, 이것이 실제 데이터인지 AI가 생성한 수치인지 반드시 원문으로 확인하세요.

### 4-4. 팩트체크 프롬프트

AI에게 스스로 팩트체크를 요청하는 프롬프트도 유용합니다. (다만 이것만으로는 부족하고, 원문 대조가 반드시 병행되어야 합니다.)

```
각 항목에 대해:
| 항목 | 텍스트 기재 내용 | 신뢰도(높음/중간/낮음) | 확인 방법 | 확인 소스 |
```

### 5-1. 역할 분담 — 누가 무엇을 잘 하는가

KDB생명 실무에서는 Bloomberg, 보험개발원 같은 전문 도구와 AI를 함께 사용하게 됩니다. 각 도구의 강점에 따라 역할을 분담하면 최대 효과를 얻을 수 있습니다.

| **보험 통계 데이터 추출** | **최적** — 정확한 정형 데이터 | 보유 시 활용 | 보조 (공시 데이터 구조화) |
| **보험사 경영공시 비교** | 보조 | **최적** (내부 데이터) | **최적** — 다수 공시 종합 비교 |
| **보험시장 동향 요약** | 보조 (리서치 리포트) | 보조 | **최적** — 다수 소스 종합 요약 |
| **보험 법률·규제 정리** | - | 보조 | **최적** — 법령 텍스트 분석·요약 |
| **보고서 초안 작성** | - | - | **최적** — 구조화된 초안 생성 |
| **자료 비교·분석** | 보조 (정형 데이터) | 보조 | **최적** — 비정형 텍스트 비교 |
| **팩트체크·원문 대조** | **최적** (정확 데이터) | **최적** (내부 데이터) | 보조 (1차 스크리닝) |

### 5-2. 실무 병행 워크플로 예시

**보험 인수심사(언더라이팅) 시나리오에서의 도구 병행**:

```
1. 보험계약DB → 피보험자 기본정보·기존 계약 이력 확인
2. 보험개발원 → 해당 위험군의 참조순보험요율·경험생명표 데이터 확인
3. AI (ChatGPT/Claude) → 의학 정보 요약, 유사 인수심사 사례 리서치 초안 생성
4. 생명보험협회·금감원 → 관련 감독규정·인수기준 원문 대조 확인
5. AI → 심사 의견서 구조화 초안 작성
6. Perplexity → 최신 의학 연구·보험업 트렌드 검색
7. 사람 → 최종 인수심사 의견 작성, 결재 제출
```

> 핵심: AI는 **정형 데이터 추출 도구가 아니라 분석·요약·구조화 도구**로 활용해야 합니다. 정확한 보험 통계·위험률은 보험개발원·생명보험협회·사내 시스템에서 가져오고, AI에게는 그 데이터를 분석·요약·비교하는 역할을 맡기세요.

### 5-3. 도구별 접근 권한과 데이터 보안

| 도구 | 데이터 입력 범위 | 보안 수준 |
|------|---------------|----------|
| **사내 AI 시스템** (구축 시) | 내부 기밀 포함 가능 | 최고 (내부망) |
| **Bloomberg** | 유료 구독 데이터 | 높음 (기업 계약) |
| **ChatGPT/Claude** (외부) | **공개 데이터만** | 중간 (입력 데이터 학습 가능성) |
| **Perplexity** | **공개 데이터만** | 중간 |

> 주의: 삼성전자는 2023년 직원이 ChatGPT에 반도체 소스코드와 회의록을 입력하여 기밀이 유출된 후 사내 AI 사용을 전면 금지한 바 있습니다. **외부 AI에 입력하는 모든 데이터는 공개된 정보인지 반드시 확인**하세요. 특히 보험계약자 의료정보, 건강정보, 미출시 상품 설계서는 절대 외부 AI에 입력하면 안 됩니다. (출처: TechCrunch, 2023)

## 6. 정보 조회 영역 Before/After 사례 6선

KDB생명 임직원이 자주 수행하는 *감독규정·법령 검색, 약관 비교, 판례 검색, 회계정책서 문단 검색, 그룹사 정책금융 동향, 동업사 상품 구조 비교* 등 6개 정보 조회 영역의 실전 프롬프트를 정리한다. 모든 사례는 **AI를 *검색기*가 아닌 *해석·비교 도구*로 한정**하는 황금 패턴을 공유한다. 사람이 1차 자료를 직접 가져오고, AI는 그 자료의 *해석·비교·요약*만 수행한다.

> 🔑 **공통 원칙**: 정보 조회 영역에서 AI 환각의 가장 큰 원인은 *"AI에게 검색을 시키는 것"*이다. 조항 번호·사건번호·통계 수치는 AI가 가장 자신만만하게 지어내는 항목이다. 1차 자료 확보는 사람이, AI는 그 자료의 *해석·비교·요약*만 — 이 한 줄을 모든 사례에 적용한다.

### 6-1. 감독규정·법령 조항 해석 — 변액보험 자산운용

**상황**: 준법감시팀에서 변액보험 자산운용 관련 외부위탁 비율 규제를 점검하기 위해 보험업법·보험업감독규정의 조항 해석이 필요하다. 두 조항이 어떤 관계인지를 정확히 정리해야 한다.

**❌ Before — 나쁜 프롬프트:**
```
보험업감독규정에서 변액보험 자산운용 관련 조항 알려줘
```
→ AI가 *"보험업감독규정 제352조의2에 따르면…"* 같은 가짜 조문을 자신만만하게 출력한다. 같은 질문을 다시 물으면 다른 조문 번호를 제시한다. 그대로 결재 라인에 올리면 사고로 이어진다.

**✅ After — 좋은 프롬프트:**
```
당신은 보험업법·보험업감독규정에 능숙한 KDB생명 준법감시 검토 담당자입니다.

아래는 제가 법제처 국가법령정보센터(law.go.kr)에서 직접 가져온
원문입니다. 이 원문만을 근거로 질문에 답해주세요.
원문에 없는 내용은 절대 추측하지 말아주세요.

[원문 1 — 보험업법 제OO조 (최종 개정 YYYY.MM.DD)]
{법제처에서 복사한 원문 그대로}

[원문 2 — 보험업감독규정 제OO조 (최종 개정 YYYY.MM.DD)]
{금감원 법규정보에서 복사한 원문 그대로}

질문:
1. 두 조항이 다루는 대상이 동일한가, 보완 관계인가, 상하위 관계인가?
2. 변액보험 자산운용에서 외부위탁 비율이 50%를 초과할 경우
   적용되는 조항은 어느 쪽이며, 그 근거 문장은 무엇인가?
3. 원문에 명시되지 않아 추가 확인이 필요한 사항을 별도로 표시.

조건:
- 원문에 없는 내용을 만들지 마세요.
- 조항 번호를 새로 만들지 마세요.
- 시행일·개정일은 원문에 적힌 것만 사용하세요.
- 답변 끝에 '본 검토는 원문 발췌만 근거로 함'을 명시.
```

**기법 해설:** AI를 검색기로 쓰지 않고, 사람이 가져온 원문의 *해석자*로 한정한다. *"원문에 없는 내용 추측 금지"*를 명시적으로 못 박아 조항 번호 환각을 차단한다. Role(준법감시 검토 담당자) + Context(법제처 원문) + Task(2개 조항 비교 해석) 구조로 작업 범위를 좁힌다.

**검증 포인트:**
- AI 답변에 등장하는 조항 번호가 입력한 원문 발췌의 조항 번호와 일치하는지 확인
- *"본 검토는 원문 발췌만 근거로 함"* 문구가 답변 끝에 포함됐는지 확인
- 시행일·개정일이 원문 표기 그대로인지 확인 (AI가 임의로 *"시행 2024.OO.OO"* 식으로 만든 경우 즉시 폐기)

**보안 판단:**
- 외부 LLM 입력 가능 여부: **가능** — 보험업법·보험업감독규정은 공개 법령이다.
- 마스킹 필요 항목: 사내 검토 의견서·사내 결재 메모·사내 검토자 이름은 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례의 조항 번호(`제OO조`)는 더미 표기다. 실제 검토 시에는 법제처 국가법령정보센터에서 정확한 원문을 가져오고, 사내 검토 의견서·내부 메모는 외부 LLM에 입력하지 말 것.

### 6-2. 약관 비교 — 자사 vs 경쟁사 면책 사유

**상황**: 상품개발팀에서 자사 종신보험 약관의 면책 사유를 경쟁사(A생명) 약관과 비교 분석한다. 약관심사 가이드라인 관점에서 잠재 쟁점을 사전에 도출해야 한다.

**❌ Before — 나쁜 프롬프트:**
```
A생명 종신보험 약관이랑 우리 약관 비교해줘
```
→ A생명 약관을 AI가 임의로 만들어내거나, *"일반적으로 종신보험 면책 사유는…"* 식의 일반론적 비교만 출력한다. 실제 약관 문구와 무관한 답변이 나와 약관심사 검토에 사용 불가하다.

**✅ After — 좋은 프롬프트:**
```
당신은 KDB생명 상품 담당자이며, 자사와 경쟁사 약관을 비교 분석합니다.

아래는 두 보험사의 종신보험 약관에서 '면책 사유' 부분만 발췌한 원문입니다.

[자사 약관 — 종신보험 약관 제OO조 면책 사유 (시행 YYYY.MM.DD)]
{원문 발췌}

[경쟁사 약관 — A생명 종신보험 약관 제OO조 면책 사유 (생명보험협회 공시 발췌)]
{원문 발췌}

다음을 비교해주세요:
1. 양사 공통 면책 사유 (조항 번호 함께)
2. 자사에만 있는 면책 사유
3. 경쟁사에만 있는 면책 사유 → 자사 추가 검토 필요한지 의견
4. 표현 차이가 있는 사유 (실질이 같은데 표현만 다른 경우)
5. 약관심사 가이드라인 관점에서의 잠재 쟁점

출력: 위 5개 항목을 각각 표로 정리.

조건:
- 원문에 없는 면책 사유를 만들지 말 것
- 경쟁사 약관 출처는 '생명보험협회 공시 발췌'로 표기
- '자사 추가 검토'는 의견이지 결정이 아님을 명시
- 조항 번호 추측 금지
```

**기법 해설:** 두 약관 원문을 함께 입력하고 AI는 *비교만* 수행하도록 작업을 좁힌다. *"가짜 약관 조항 생성"* 환각을 차단하기 위해 출처를 명시(생명보험협회 공시 발췌)하고, *"자사 추가 검토는 의견이지 결정이 아님"*을 강제 명시 항목으로 둔다.

**검증 포인트:**
- AI가 정리한 *"양사 공통 면책 사유"* 표의 조항 번호가 입력한 원문의 조항 번호와 일치하는지 항목별 대조
- *"경쟁사에만 있는 면책 사유"* 목록이 입력한 A생명 약관 발췌에 실제로 존재하는지 확인
- AI가 *"일반적으로 면책 사유는…"*처럼 입력 자료 외 일반론을 끌어들였는지 확인

**보안 판단:**
- 외부 LLM 입력 가능 여부: **가능** — 자사 약관 시판본·경쟁사 공시 약관은 모두 공개 자료.
- 마스킹 필요 항목: 미출시 상품의 약관 초안·약관심사 내부 검토 메모·법무팀 의견서는 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례의 약관 조항 번호(`제OO조`)는 더미 표기다. 자사 약관 비교 시에도 *시판본 약관*만 외부 LLM에 입력하고, 약관심사 내부 검토 의견은 사내 LLM 또는 사내 RAG로 한정할 것.

### 6-3. 판례 검색 보조 — 보험금 지급 판례

**상황**: 보험금심사팀에서 질병사망 보험금 거절과 관련한 최근 판례 2건의 비교 분석이 필요하다. 향후 유사 사안 검토 시 적용 가능한 시사점을 도출해야 한다.

**❌ Before — 나쁜 프롬프트:**
```
질병사망 보험금 거절 관련 최근 판례 찾아줘
```
→ AI가 *"대법원 2024다XXXXX 판결"* 같은 가짜 사건번호를 자신만만하게 인용한다. 그대로 보고서에 넣으면 즉시 발각되어 사내 신뢰를 잃는다. 보험금 분쟁에서 가짜 판례 인용은 컴플라이언스 사고로 직결된다.

**✅ After — 좋은 프롬프트:**
```
당신은 보험 분쟁 판례 분석에 능숙한 보험금심사 보조 담당자입니다.

[중요 — 작업 분담]
- 사건번호 검색은 사람이 대법원 종합법률정보에서 직접 수행했습니다.
- AI에게 요청하는 것은 '아래 두 판례 원문의 비교 분석'뿐입니다.

[판례 1 — 사건번호: {사람이 직접 확인한 번호}, 선고일자: YYYY.MM.DD]
{판례 원문 또는 판시사항·판결요지 발췌}

[판례 2 — 사건번호: {사람이 직접 확인한 번호}, 선고일자: YYYY.MM.DD]
{원문 발췌}

비교해주세요:
1. 양 판례의 사실관계 핵심 차이
2. 법원 판단의 핵심 논리 차이
3. 약관 해석의 어느 부분이 쟁점이 됐는가
4. 두 판례가 향후 KDB생명 유사 사안 검토에 주는 시사점
5. 추가 검토가 필요한 점

조건:
- 입력된 두 판례만을 근거로 분석할 것
- 다른 판례 인용·언급 금지
- 사건번호를 새로 만들거나 변형하지 말 것
- 시사점은 '의견'이며 '결정'이 아님을 명시
- 끝에 '판례 적용 여부의 최종 판단은 법무팀 검토 영역'을 명시
```

**기법 해설:** 사건번호 확보를 사람이 했다고 *명시적으로* 적어 AI에게 *"검색 요청이 아님"*을 인지시킨다. *"다른 판례 인용·언급 금지"* 옵션으로 입력 자료 외 환각을 원천 차단한다. *"시사점은 의견, 최종 판단은 법무팀 영역"* 강조로 보고서에 결정형 문장이 들어가는 사고를 막는다.

**검증 포인트:**
- AI 답변에 등장하는 모든 사건번호가 입력 원문의 사건번호와 *문자 단위로* 일치하는지 확인
- AI가 *"또한 대법원 20XX다XXXXX 판결에서는…"* 같이 입력하지 않은 다른 판례를 끌어들였는지 확인 (1건이라도 발견 시 즉시 폐기)
- *"판례 적용 여부의 최종 판단은 법무팀 검토 영역"* 문구가 답변 끝에 포함됐는지 확인

**보안 판단:**
- 외부 LLM 입력 가능 여부: **가능** — 대법원 종합법률정보의 공개 판례는 외부 LLM 사용 가능.
- 마스킹 필요 항목: 사내 보험금 분쟁 진행 중인 사건의 내부 메모·당사자 식별정보(피보험자 성명·주민번호·계약번호)는 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례의 사건번호(`{사람이 직접 확인한 번호}`)는 더미 표기다. 실제 사용 시 대법원 종합법률정보에서 사람이 직접 확인한 사건번호만 입력하고, 진행 중 분쟁의 내부 검토 메모는 외부 LLM에 입력 금지.

### 6-4. 회계정책서 문단 검색 — IFRS 17 CSM 재산정

**상황**: IFRS 17 회계정책 담당자가 *"보험계약 변경 시 CSM 재산정 절차"*가 사내 회계정책서의 어느 문단에 정의돼 있는지 빠르게 찾아야 한다. 일반론적 IFRS 17 설명이 아니라 *사내 회계정책서 본문* 인용이 필요하다.

**❌ Before — 나쁜 프롬프트:**
```
회계정책서에서 보험계약 변경 시 CSM 재산정 부분 찾아줘
```
→ AI가 *"회계정책서 제5장 3.2절에 따르면…"*처럼 가짜 섹션 번호를 만들어낸다. AI는 사내 회계정책서를 본 적이 없으므로 IFRS 17 일반론을 회계정책서 인용처럼 답한다. 그대로 결재에 올리면 *"우리 회계정책서에 그런 섹션 없는데?"* 즉시 발각된다.

**✅ After — 좋은 프롬프트:**
```
당신은 KDB생명 IFRS 17 회계정책 담당자이며, 사내 회계정책서의
문단 검색·해설을 보조합니다.

아래는 사내 회계정책서 발췌본(공개 가능 부분)입니다.
이 발췌본 안에서만 검색·해설을 수행해주세요.

[회계정책서 발췌 — 챕터 단위로 붙여넣기]
{문단 또는 섹션 단위로 — 챕터 번호는 사내 표기 그대로}

질문: '보험계약 변경 시 CSM 재산정 절차'와 관련된 문단이 어느
부분인지 인용 표시([챕터 번호]·[섹션 번호] 형식과 함께)하고, 핵심 내용을 3줄로
요약해주세요.

조건:
- 발췌본에 없는 내용은 추정하지 말 것
- 인용 표시 시 [섹션 번호]와 첫 문장을 함께 적을 것
- 발췌본에서 관련 문단을 찾지 못하면 '발췌본 내 미포함 — 추가 발췌 필요'로 답할 것
- IFRS 17 일반론을 회계정책서 내용처럼 답변하지 말 것
```

**기법 해설:** *"발췌본 안에서만 검색"*으로 RAG(Retrieval-Augmented Generation)와 동일한 효과를 직접 만든다. *"발췌본 내 미포함 — 추가 발췌 필요"*라는 *안전한 응답 옵션*을 명시적으로 제공해 AI가 *"답이 없을 때 답을 만들어내는"* 기본 행동을 차단한다. IFRS 17 일반론을 회계정책서 본문처럼 답하는 패턴은 *명시적 금지 조건*으로 못 박는다.

**검증 포인트:**
- AI가 인용한 섹션 번호가 입력한 발췌본 안에 *문자 단위로* 존재하는지 확인
- 핵심 내용 3줄 요약이 발췌본 원문의 표현과 일치하는지 확인 (IFRS 17 일반론을 끌어들이지 않았는지)
- *"발췌본 내 미포함 — 추가 발췌 필요"* 응답을 받은 경우, 추가 발췌본을 준비해 다시 입력

**보안 판단:**
- 외부 LLM 입력 가능 여부: **제한적** — 사내 회계정책서는 공개 가능 부분만 외부 LLM 사용 가능. 미공개 부분(특정 추정 가정·내부 통제 절차 등)은 사내 LLM 또는 사내 RAG로 한정.
- 마스킹 필요 항목: 회계정책서의 *비공개 챕터*, 추정 가정 수치, 내부 통제 절차서, 외부 감사 의견서는 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례의 인용 표기(`[챕터 번호]`·`[섹션 번호]`)는 더미 형식이다. 실제 사내 회계정책서의 챕터 번호 체계는 외부 LLM에 노출하지 말고, 발췌본도 *공개 가능 부분*에 한정해 사용할 것.

### 6-5. 정책금융 펀드 / 그룹사 동향 조사

**상황**: 전략기획부 담당자가 KDB산업은행 그룹 차원의 정책금융 펀드 동향을 사내 보고용으로 정리한다. 그룹 내부 정보를 추측해 만들면 즉시 사고로 이어지므로, 1차 검색은 공식 공개 자료에서 사람이 수행하고 AI는 종합·요약만 한다.

**❌ Before — 나쁜 프롬프트:**
```
KDB산업은행 그룹의 정책금융 펀드 최근 동향 알려줘
```
→ AI가 출처 불명의 추측성 정보를 출력한다. *"OO펀드는 2025년 약 OO조 원 규모로…"* 같은 그럴듯한 수치가 나오지만 검증 불가하다. 그룹 내부 정보 추측은 그대로 보고에 올리면 그룹 차원 신뢰 사고로 직결된다.

**✅ After — 좋은 프롬프트:**
```
당신은 KDB생명 전략기획부 담당자이며, KDB산업은행 그룹의
정책금융 펀드 동향을 사내 보고용으로 정리합니다.

[중요 — 작업 분담]
- 1차 검색은 사람이 KDB산업은행 공식 보도자료, 금융위원회·기획재정부 보도자료,
  주요 언론 보도에서 직접 수행했습니다.
- AI에게 요청하는 것은 '아래 자료의 종합·요약'뿐입니다.

[자료 1 — KDB산업은행 보도자료 'YYYY.MM.DD' 정책금융 펀드 OO 발표]
{원문 발췌}

[자료 2 — 금융위원회 보도자료 'YYYY.MM.DD' OO 정책 방향]
{원문 발췌}

[자료 3 — OO일보 보도 'YYYY.MM.DD']
{기사 본문 발췌}

다음을 정리해주세요:
1. 정책금융 펀드의 최신 운용 방향 (3줄)
2. KDB생명에 직간접 관련될 수 있는 항목 (있다면)
3. 자료 간 시점·내용 불일치가 있는 부분
4. 추가 확인이 필요한 사항

출력 형식:
| 항목 | 내용 | 출처 (자료 번호) | 시점 |

조건:
- 입력된 자료에 없는 정보를 추측해 추가하지 말 것
- KDB생명 관련 시사점은 '의견'으로 표기
- 불확실한 항목은 '확인 필요'로 표시
- 그룹 내부 정보를 추측해 만들지 말 것
```

**기법 해설:** 그룹사 동향 정보는 환각 위험이 매우 크다. *"검색 작업을 사람이 했음"*을 명시하고 AI는 *종합·요약*에 한정한다. *"그룹 내부 정보를 추측해 만들지 말 것"*이라는 강한 부정 조건을 명시하고, 출처 자료 번호를 표 형식으로 강제해 *"어떤 자료에서 나온 정보인지"*를 항상 추적 가능하게 만든다.

**검증 포인트:**
- 출력 표의 *"출처 (자료 번호)"* 열이 모든 행에 채워져 있는지 확인 (비어있으면 환각 의심)
- 표에 등장하는 수치·일자가 입력 자료의 원문과 일치하는지 항목별 대조
- *"확인 필요"* 표시가 적절히 사용됐는지 — 모든 항목이 단정형으로 채워졌다면 오히려 의심
- KDB생명 관련 시사점이 *의견*으로 명시됐는지, 단정 결론으로 적혔는지 확인

**보안 판단:**
- 외부 LLM 입력 가능 여부: **가능** — 공식 보도자료·언론 보도는 모두 공개 자료.
- 마스킹 필요 항목: 그룹 내부 회의록, 비공개 운용 계획, KDB생명 내부 검토 의견, 그룹 차원 미공개 사업 계획은 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례에서 *"정책금융 펀드"*는 일반화 표기다. 실제 사내 보고에서는 KDB산업은행 그룹의 구체적 펀드명을 사용할 수 있으나, 외부 LLM 입력 시에는 *공식 보도자료에 명시된 명칭*까지만 사용하고 그룹 내부 호칭·비공식 약칭은 입력하지 말 것.

### 6-6. 동업사 상품 구조 비교

**상황**: 상품전략부에서 종신보험 시장의 자사·동업사 상품 구조를 비교 분석한다. 보험료 수준의 단순 비교는 가정·할인구조 차이로 부적절하므로, *구조* 비교에 한정해야 한다.

**❌ Before — 나쁜 프롬프트:**
```
동업사 종신보험 구조 어떻게 돼있어?
```
→ AI가 *"일반적으로 종신보험은 사망보험금을…"* 식의 일반론적 종신보험 구조를 답하거나, 동업사 상품을 임의로 추정해 *"A사 OO종신은 가입 연령 15~70세, 보험료는…"* 같은 가짜 사실을 출력한다.

**✅ After — 좋은 프롬프트:**
```
당신은 KDB생명 상품전략부 담당자이며, 생명보험협회 상품비교공시
자료를 기반으로 동업사 상품 구조를 분석합니다.

[자료 출처 — 사람이 직접 다운로드]
- 생명보험협회 상품비교공시 (klia.or.kr) — 종신보험 카테고리
- 발췌일: YYYY.MM.DD

[동업사 A — OO종신보험 핵심 항목 발췌]
- 가입 연령:
- 보장 한도:
- 주요 특약:
- 보험료 산출 기준:
- 환급 방식 (해지/만기):

[동업사 B — OO종신보험 핵심 항목 발췌]
{동일 형식}

[자사 — KDB생명 OO종신보험 핵심 항목]
{동일 형식}

다음을 비교해주세요:
1. 3사 공통 구조 항목과 차이 항목
2. 동업사 대비 자사 차별화 포인트 (강점)
3. 동업사 대비 자사 약점으로 보이는 부분
4. 시장 트렌드 관점에서 자사가 검토할 만한 추가 옵션

조건:
- 입력 자료에 없는 항목을 추측해 채우지 말 것
- 보험료 수준의 절대 비교는 하지 말 것
  (가정·할인구조가 달라 단순 비교가 부적절)
- 차별화 포인트와 약점은 '의견'으로 명시
- 자사 미공개 정보는 다루지 말 것
```

**기법 해설:** 공시 자료 출처(생명보험협회 상품비교공시)와 발췌일을 명시해 *"어떤 시점의 어떤 자료인지"*를 명확히 한다. AI가 자주 빠지는 *"보험료 수준 단순 비교"* 함정을 *명시적 금지 조건*으로 차단했다. *"자사 미공개 정보 금지"* 조건으로 자사 미출시 상품 정보가 답변에 끼어드는 사고를 막는다.

**검증 포인트:**
- AI가 채운 *"동업사 A 핵심 항목"*이 입력한 발췌 항목과 일치하는지 — AI가 *추측해 빈칸을 채운* 항목이 있는지 확인
- *"보험료 수준 비교"*가 답변에 들어갔다면 즉시 폐기 (조건 위반)
- 차별화 포인트·약점이 *"의견"*으로 명시됐는지 확인
- *"시장 트렌드 추가 옵션"*이 입력 자료 범위 안에서 도출됐는지, AI가 일반 트렌드를 끌어들였는지 확인

**보안 판단:**
- 외부 LLM 입력 가능 여부: **가능** — 생명보험협회 상품비교공시는 공개 자료, 자사 시판 상품도 공개 정보.
- 마스킹 필요 항목: 자사 미출시 상품 설계서, 신상품 보험료 산출 가정, 상품개발팀 내부 메모는 외부 LLM 절대 금지.

> 💡 **사이트 게재 주의**: 본 사례에서 *"OO종신보험"*은 더미 표기다. 실제 사용 시에도 공시 자료에 등장하는 명칭만 사용하고, 미출시 상품의 가칭·내부 코드명은 외부 LLM에 입력하지 말 것.

---

> ⚠ **섹션 6 종합 주의**: 본 사례 6선의 법령 조항·자료 참조·사건번호는 모두 일반화된 더미 예시다. 실제 사내 검토 의견서·심사 메모·내부 회의록은 외부 LLM 절대 금지이며, 1차 자료(법령·판례·공시) 확보는 사람이 공식 출처에서 직접 수행해야 한다. AI는 *해석·비교·요약*만 담당한다.

## 학습 정리

### 핵심 요약
- KDB생명 실무에서 자주 찾는 자료는 보험 통계(보험개발원, 생명보험협회), 기업 공시(DART, 금감원 보험통계), 법률 규제(국가법령정보센터, 보험업법·K-ICS), 글로벌(Bloomberg, IAIS, Geneva Association, Swiss Re)로 나뉜다
- AI 검색 전략은 3가지: (1) Perplexity로 최신 정보 찾기, (2) ChatGPT/Claude에게 리소스 소재지 묻기, (3) AI에게 자료를 넣고 분석·비교 시키기
- 보험업에서 출처 없는 수치 사용은 금지 — AI 결과는 반드시 3단계(AI 검색 → 원문 대조 → 전문가 확인) 검증을 거쳐야 한다
- 전문 도구(Bloomberg, 보험개발원)는 정확한 데이터 추출에, AI는 분석·요약·구조화에 각각 활용하는 병행 전략이 효과적이다
- 외부 AI에는 공개된 자료만 입력하고, 보험계약자 의료정보·건강정보·미출시 상품정보는 절대 입력하지 않는다

### 즉시 적용 체크리스트
- [ ] 자주 쓰는 자료 유형(보험통계/공시/법률/글로벌) 중 1가지를 선택하여 AI 검색 전략을 시험한다
- [ ] 실전 시나리오 5가지 중 자신의 업무와 가장 가까운 1가지를 직접 실행해본다
- [ ] AI 검색 결과에 대해 "원문 대조" 습관을 시작한다 (보험개발원, 생명보험협회, 국가법령정보센터)
- [ ] 팩트체크 3단계(AI 검색 → 원문 대조 → 전문가 확인)를 1건 이상 적용해본다
