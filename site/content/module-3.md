# 모듈 3: LLM 선택 전략 — 모델별 특성과 KDB생명 활용 가이드

> **대상**: KDB생명 임직원
> **학습 시간**: 약 40~60분
> **핵심 키워드**: LLM 비교, ChatGPT, Claude, Gemini, Perplexity, 사내 LLM, 업무별 모델 선택, 동일 프롬프트 비교

---

## 학습 목표
1. 주요 LLM 5종(ChatGPT, Claude, Gemini, Perplexity, 사내 LLM)의 강점·약점을 구분할 수 있다
2. 동일한 프롬프트를 각 모델에 입력했을 때 답변이 어떻게 달라지는지 체감할 수 있다
3. KDB생명 업무 특성에 맞는 최적 LLM을 선택하는 의사결정 기준을 적용할 수 있다
4. 사내 LLM 활용 시 고려사항과 외부 AI와의 역할 분담 원칙을 이해할 수 있다

## 1. 주요 LLM 5종 비교표

2026년 4월 현재, 업무에 활용할 수 있는 주요 AI 모델은 크게 5종으로 분류된다. "어떤 AI가 최고인가"보다 "**이 업무에는 어떤 AI가 적합한가**"를 먼저 생각하는 것이 핵심이다.

### 종합 비교표

| 항목 | ChatGPT (OpenAI) | Claude (Anthropic) | Gemini (Google) | Perplexity | 사내 LLM |
|------|-----------------|-------------------|----------------|------------|---------|
| **최신 모델** | GPT-5.5 (2026.04) | Opus 4.7 / Sonnet 4.6 (2026.04) | Gemini 3.1 Pro (2026.02) | Sonar (다중 모델 기반) | 기관별 상이 |
| **컨텍스트 윈도우** | **1M 토큰** (API) / 400K (Codex) | **1M 토큰** | **1M 토큰** | 모델별 상이 | 보통 8K~32K |
| **최대 강점** | 멀티모달 통합, 에이전트 도구, 폭넓은 생태계, 장문 처리 강화(GPT-5.5) | 장문 분석, 정밀한 지시 준수, 자연스러운 문장력 | 추론 벤치마크 선두, 이미지 생성, Google 연동 | 실시간 검색 + 출처 명시, 금융 데이터 연동 | 보안 보장, 사내 데이터 활용 |
| **최대 약점** | 복합 조건 일부 누락, 빠른 모델 사이클로 가격·기능 변동 | 이미지 생성 불가, 확장 생태계 작음 | 한국어 품질 다소 낮음, 장문 구조화 약함 | 장문 작성 부적합, 한국어 검색 제한 | 추론 능력 제한, 최신 정보 불가 |
| **최신 정보** | 웹 브라우징 가능 | 웹 검색 가능 | Google 검색 연동 | **핵심 기능** (실시간 검색) | 불가 (폐쇄망) |
| **보안 수준** | 외부 서버 | 외부 서버 | 외부 서버 | 외부 서버 | **사내 서버** (최고) |
| **할루시네이션율** | 10.8% | 10.6~12.2% | 0.7~10.4% | 낮음 (출처 기반) | 모델별 상이 |
| **KDB생명 대표 활용** | 보험상품 비교표 초안, 고객 안내문 작성 | 인수심사 의견서, 보험약관 분석 | 건강검진 결과표 해석, 보험통계 시각화 | 보험시장 동향, 규제 변화 검색 | 보험계약 포트폴리오, 인수심사 기록 RAG |

> 💡 팁: 만능 AI는 없다. 각 모델의 강점을 파악하고, 업무 목적에 따라 **적합한 모델을 선택하거나 조합**하는 것이 핵심이다.

### 모델별 강점·약점 상세

#### ChatGPT (GPT-5.5) — 가장 폭넓은 기능의 범용 모델

**강점:**
- 멀티모달 통합(텍스트·이미지·음성·비디오)이 가장 발전된 모델
- DALL-E(이미지 생성), Code Interpreter(코드 실행), GPTs(맞춤 AI) 등 올인원 플랫폼
- 대화형 반복 개선에 적합: "좀 더 구체적으로", "이 부분 수정해줘" 같은 대화로 점진적 완성
- GPT-5.5부터 **1M 토큰 컨텍스트**(API) / 400K(Codex) 지원으로 장문 처리 능력 대폭 강화 (128K~512K 구간 정확도 GPT-5.4 대비 약 2배 향상)
- GPT-5.5 Pro / GPT-5.5 Thinking 변형으로 에이전틱 코딩·복합 추론 최적화
- 가장 큰 사용자 커뮤니티와 활용 사례

**약점:**
- 복잡한 다단계 지시에서 일부 조건을 빠뜨리는 경우 발생
- 할루시네이션율(GPT-5.x 기준 약 10% 안팎)이 잔존 — 팩트 의존 업무에서는 검증 필수
- 빠른 모델 사이클(GPT-5.4 → 5.5가 6주)로 API 가격·기능이 자주 바뀌어 사내 표준화에 부담

**KDB생명 활용 시나리오:** 보험상품 비교표 초안, 고객 안내문 작성, 마케팅 콘텐츠 브레인스토밍, 일반적 분석 초안 생성

#### Claude (Opus 4.7 / Sonnet 4.6) — 장문 분석과 정밀 지시의 전문가

**강점:**
- **1M 토큰 컨텍스트**: 수백 페이지 분량의 보험약관·사업보고서를 통째로 넣어 분석 가능
- 복잡한 지시 정밀 준수: "5개 항목으로 나누되, 각 항목에 수치 근거를 포함하고, 임원 보고 톤으로" 같은 복합 조건을 정확히 이행
- 128K 토큰까지 한 번에 출력 가능 — 긴 보고서 초안을 한 번에 생성
- SWE-bench 80.8%로 코딩·에이전틱 태스크에서 최고 수준

**약점:**
- 이미지 생성 기능 없음 (이미지 분석은 가능)
- 할루시네이션율이 Gemini Flash 대비 상대적으로 높음(10.6~12.2%)
- 플러그인·확장 생태계가 ChatGPT 대비 작음

**KDB생명 활용 시나리오:** 인수심사 의견서 초안, 보험약관 비교 분석, 장문 사업보고서·감독규정 분석, 복합 조건 보고서 생성

#### Gemini (3.1 Pro) — 추론과 멀티모달의 강자

**강점:**
- GPQA 94.3%로 **추론 벤치마크 선두** — 복잡한 논리적 분석에 강함
- 1M 토큰 컨텍스트로 대용량 데이터 한 번에 처리
- 이미지, 음성, 영상까지 이해하는 가장 발전된 멀티모달 능력
- Google 생태계(검색, Workspace) 연동으로 최신 정보 접근 용이
- Gemini 2.0 Flash 모델은 할루시네이션율 **0.7%** 로 사실 정확도 최고

**약점:**
- 한국어 응답 품질이 ChatGPT/Claude 대비 다소 낮은 경우 존재
- 장문 문서 작성 시 구조화 능력이 Claude 대비 약한 편
- Pro 모델은 할루시네이션율(10.4%)이 Flash 대비 높음

**KDB생명 활용 시나리오:** 건강검진 결과표·의료차트 이미지 해석, 보험통계 시각 자료 분석, Google 검색 연동 실시간 리서치, 빠른 팩트체크(Flash 모델)

#### Perplexity — AI 시대의 검색 엔진

**강점:**
- **모든 답변에 출처 URL을 함께 제공** — 팩트체크가 즉시 가능
- SEC 파일링·실적 콜·FactSet 데이터 등 40개 이상의 금융 도구 연동 (Perplexity Finance)
- 인라인 출처 인용 정확도 89%
- Pro Search 모드로 심층 리서치 수행
- 여러 LLM(GPT, Claude 등)의 답변을 동시에 비교하는 Model Council 기능

**약점:**
- 장문 문서 작성이나 창의적 콘텐츠 생성에는 부적합
- 한국어 검색 결과의 품질이 영어 대비 제한적
- 문서 분석이나 대화형 편집 기능이 부족

**KDB생명 활용 시나리오:** 보험시장 동향 리서치, 보험업법·K-ICS 규제 변경 확인, 경쟁사·해외 보험사 사례 조사, 수치 팩트체크

#### 사내 LLM — 보안이 최우선인 업무의 파트너

**강점:**
- **데이터가 외부로 나가지 않음** — 민감한 사내 정보를 자유롭게 입력 가능
- 보험계약자 의료정보, 건강정보, 인수심사 기록 등을 입력하여 분석 가능
- 보험업 규제(개인정보보호법, 신용정보법, 보험업법) 컴플라이언스 충족

**약점:**
- 모델 크기가 외부 LLM 대비 작아 **추론 능력이 제한적**
- 인터넷 검색 불가 — 최신 정보 반영 불가
- 멀티모달, 코드 실행 등 고급 기능이 없는 경우가 많음
- 컨텍스트 윈도우가 보통 8K~32K로 장문 처리에 한계

**KDB생명 활용 시나리오:** 보험계약 포트폴리오 분석, 인수심사 기록 RAG, 보험계약자 의료정보 기반 리스크 평가, K-ICS 내부 산출 데이터 분석

> ⚠️ 주의: 사내 LLM은 성능이 외부 모델보다 제한적이다. 이를 보완하려면 프롬프트를 더 상세하게 작성해야 한다. 맥락을 2배로 풍부하게, 예시를 2~3개 첨부하고, 한 번에 하나의 작업만 요청하는 것이 원칙이다.

## 2. 동일 프롬프트, 다른 답변 — 3세트 비교 시연

같은 양질의 프롬프트를 서로 다른 LLM에 입력하면, 모델별 특성에 따라 답변의 방향·깊이·형태가 달라진다. 3가지 KDB생명 업무 시나리오로 직접 비교한다.

### 시연 1: 건강보험 시장 경쟁 환경 분석

**사용 프롬프트:**
```
당신은 KDB생명 상품전략부의 보험시장 분석 전문가입니다.

국내 건강보험 시장에서 경쟁 심화와 손해율 악화 가능성을 
분석해주세요. 2024~2026년 시장 동향을 반영하고,
리스크가 높은 상품군(실손보험/CI보험/간병보험)별로 
리스크 수준과 상품전략 방향을 정리해주세요.

형식:
1. 한 줄 핵심 판단
2. 상품군별 리스크 평가 표 (상품군/리스크 수준/주요 요인/전략 방향)
3. KDB생명 관점 시사점 3가지
```

#### ChatGPT (GPT-5.5) 답변 특징
- 균형 잡힌 서술: 긍정적 전망(고령화 수요 확대)과 부정적 요인(손해율 악화, 규제 강화)을 양면 분석
- 중간 길이의 구조적 답변: 표 포함, 각 상품군별 2~3줄 분석
- 일반적 수준의 보험 산업 지식에 기반, 최신 수치는 웹 브라우징 시에만 반영
- "건강보험 시장 리스크는 중간~높은 수준"이라는 온건한 결론

#### Claude (Opus 4.7) 답변 특징
- 가장 상세하고 뉘앙스 있는 분석: 각 상품군별 손해율 메커니즘을 심층 서술
- 장문(2,000자 이상): 상품군별 리스크 요인 간 상호 연관성까지 분석
- KDB생명의 보장성 보험 강화 전략을 맥락에 반영한 시사점 도출
- "실손보험 손해율은 2026년 하반기 추가 악화 가능성 높음"이라는 구체적 판단
- 조건부 결론: "다만, 4세대 실손 전환율과 보험료 인상 허용 폭에 따라 시나리오가 달라질 수 있음"

#### Gemini (3.1 Pro) 답변 특징
- Google 검색 연동으로 최신 데이터 반영: 2026년 1분기 실손보험 손해율, 건강보험 가입률 등
- 간결하고 빠른 답변: 핵심을 압축하여 전달
- 표 중심의 구조이나 각 항목의 서술이 Claude 대비 짧음
- "고령화·의료비 인상에 따른 장기 손해율 상승"을 핵심 리스크로 부각

#### Perplexity 답변 특징
- 모든 문장에 출처 링크 첨부: "[1] 보험연구원 2026.03", "[2] 생명보험협회 2026.02" 등
- 검색 기반으로 가장 최신의 수치와 기사를 인용
- 분석의 깊이보다는 팩트의 정확성과 출처 추적에 강점
- "실손보험 손해율 128.5% [출처]", "건강보험 신계약 CSM 비중 42% [출처]" 등 구체적 수치 제시

#### 비교 요약

| 축 | ChatGPT | Claude | Gemini | Perplexity |
|----|---------|--------|--------|------------|
| 분석 깊이 | 중간 | **최고** | 중간 | 낮음 (팩트 위주) |
| 최신 데이터 | 웹 검색 시 반영 | 제한적 | 검색 연동 반영 | **최고** (출처 포함) |
| 분량 | 중간 (~1,000자) | 장문 (~2,500자) | 짧음 (~800자) | 중간 (~1,200자) |
| KDB생명 맥락 반영 | 보통 | **높음** | 보통 | 낮음 |
| 출처 제시 | 없음 | 없음 | 부분적 | **모든 문장** |

**이 업무에 최적 모델**: **Claude** — 보험시장 경쟁 분석은 깊이 있는 맥락 해석과 뉘앙스가 중요하므로, 장문 분석에 강한 Claude가 가장 적합. 다만 최신 수치는 Perplexity로 보완 권장.

### 시연 2: KDB생명 인수심사 의견서 초안

**사용 프롬프트:**
```
당신은 KDB생명 언더라이팅부의 인수심사 전문가입니다.

B고객(남성, 52세)의 고액 종신보험(보험가입금액 5억원) 
인수심사를 진행합니다.
- 직업: 건설업 현장 관리직
- BMI: 28.5 (과체중)
- 혈압: 145/92mmHg (고혈압 1기, 복약 중)
- 흡연: 과거 흡연자 (3년 전 금연)
- 가족력: 부친 심근경색 (60세)
- 기존 보유계약: 건강보험 1건(보험가입금액 5천만원)

형식:
1. 피보험자 개요 (3줄)
2. 리스크 평가 표 (의료/직업/재정/도덕적 위험 — 각 현황/위험도/근거)
3. 종합 인수 의견 및 인수 조건 제안
4. 특별 조건(할증/부담보) 3가지

A4 1.5페이지 이내.
```

#### ChatGPT (GPT-5.5) 답변 특징
- 4개 섹션을 모두 작성, 표 포함
- 리스크 분석이 개별 항목별로 정리되나, 리스크 간 연관 분석은 약함
- 인수 등급을 "표준미달체~조건부 표준체"로 범위 제시 (결정적 판단을 회피하는 경향)
- 특별 조건이 일반적 수준 (혈압 할증, 정기 건강검진 등)

#### Claude (Opus 4.7) 답변 특징
- 가장 완성도 높은 의견서 형태: 섹션별 논리 흐름이 자연스러움
- 리스크 간 상호 연관 분석 포함: "고혈압 + 과체중 + 가족력(심근경색)이 심혈관 리스크를 복합적으로 높임"
- 인수 등급을 "조건부 인수(할증 25%)"로 **명확히 제시** + 상세 근거
- 특별 조건이 구체적: "심혈관 질환 부담보 2년, 사망보험금 체감 조건(1차년도 70%), 2년 후 건강검진 결과에 따른 조건 재심사"
- 보험업법 관점 반영: "피보험자 고지의무 충실 이행 확인, 적합성 원칙에 따른 보장 범위 적정성 검토 필요"

#### Gemini (3.1 Pro) 답변 특징
- 빠른 속도로 핵심만 압축한 답변
- 표는 제공하나 각 셀의 서술이 간결 (1줄 이내)
- Google 검색 연동으로 최신 인수심사 가이드라인, 표준체 기준표 참조
- 인수 등급을 "조건부 인수"로 제시, 근거는 간략

#### Perplexity 답변 특징
- 인수심사 의견서 형태보다는 의학적 리서치 리포트에 가까운 답변
- 고혈압·과체중 관련 사망률 통계, 보험의학 가이드라인에 출처를 풍부히 첨부
- 심사 의견서로서의 형식과 톤이 다른 모델 대비 약함
- "정확한 심사를 위해서는 상세 건강검진 결과 확인이 필요합니다"라는 면책 문구 빈번

#### 비교 요약

| 축 | ChatGPT | Claude | Gemini | Perplexity |
|----|---------|--------|--------|------------|
| 의견서 완성도 | 중간 | **최고** | 중간 | 낮음 |
| 논리적 깊이 | 중간 | **높음** (리스크 교차 분석) | 중간 | 낮음 |
| 형식 준수도 | 높음 | **최고** | 높음 | 중간 |
| 최신 의학 정보 | 제한적 | 제한적 | **반영** | **반영 + 출처** |
| 보험업 톤 적합도 | 보통 | **높음** | 보통 | 낮음 |

**이 업무에 최적 모델**: **Claude** — 인수심사 의견서는 복합 조건의 정밀한 이행, 논리적 깊이, 의견서 형식 준수가 핵심이므로 Claude가 가장 적합. 의학적 최신 동향은 Gemini나 Perplexity로 보완.

**사용 프롬프트:**

형식:

2. KDB생명 업무별 영향 표 (업무 영역 / 규제 영향 / 대응 필요 사항)

- AI기본법의 일반적 내용을 정리하나, 보험업 특화 규제에 대한 깊이는 보통
- "KDB생명은 고영향 AI 사업자에 해당할 수 있다"는 일반적 판단

#### Claude (Opus 4.7) 답변 특징
- AI기본법과 금융분야 가이드라인의 구체적 조항까지 서술
- KDB생명 업무별 영향을 매우 상세하게 매핑 (언더라이팅, 보험금심사, 자산운용 각각에 대해)
- 다만 학습 데이터 기준일에 따라 최신 시행령·세부 규칙이 누락될 가능성
- "보조수단성 원칙에 따라 AI 인수심사 결과의 최종 승인은 반드시 언더라이터가 수행해야 함"이라는 실무적 해석 포함

#### Gemini (3.1 Pro) 답변 특징
- Google 검색으로 AI기본법 시행 이후 최신 뉴스·해석 기사를 반영
- 법령 원문에 대한 링크 제공
- 요약이 간결하나 보험업 특화 해석은 상대적으로 약함

#### Perplexity 답변 특징
- 가장 최신의 정보를 출처와 함께 제공

- "금융위원회가 2025년 12월 22일 공개한 개정안 [출처: 금융위원회 보도자료]"처럼 정확한 날짜·출처 명시
- 보험업 특화 해석보다는 규제 팩트 자체의 정확한 전달에 강점

#### 비교 요약

| 축 | ChatGPT | Claude | Gemini | Perplexity |
|----|---------|--------|--------|------------|
| 규제 정확성 | 중간 | 높음 | 높음 | **최고** (출처 기반) |
| 최신성 | 제한적 | 제한적 | **높음** | **최고** |
| KDB생명 맞춤 해석 | 보통 | **최고** | 보통 | 낮음 |
| 실무 적용도 | 중간 | **높음** | 중간 | 중간 (팩트 확인용) |

## 3. 업무별 LLM 선택 의사결정 트리

KDB생명 업무 상황에서 어떤 LLM을 선택할지 빠르게 판단하기 위한 의사결정 가이드이다.

### 의사결정 플로차트

```
[시작] 이 업무에 AI를 활용하려 한다
    │
    ▼
[질문 1] 입력 데이터에 기밀 정보(보험계약자 의료정보·건강정보, 미출시 상품 설계서 등)가 포함되는가?
    │
    ├── YES ──▶ 사내 LLM 사용 (기밀 데이터 분석)
    │           └── 사내 LLM으로 부족하면? → 데이터를 가명 처리 후 외부 AI 활용
    │
    └── NO ──▶ [질문 2] 최신 정보·출처가 중요한 업무인가?
                   │
                   ├── YES ──▶ Perplexity (리서치·팩트체크)
                   │           └── 심층 분석이 추가로 필요하면? → Claude 병행
                   │
                   └── NO ──▶ [질문 3] 어떤 종류의 업무인가?
                                  │
                                  ├── 장문 보고서/인수심사 의견서/복합 분석
                                  │   └── Claude (정밀 지시 준수, 장문 처리)
                                  │
                                  ├── 이미지·차트·의료 서류 포함 분석
                                  │   └── Gemini (멀티모달)
                                  │
                                  ├── 브레인스토밍/아이디어/일반 초안
                                  │   └── ChatGPT (폭넓은 기능, 대화형)
                                  │
                                  └── 반복적 정형 업무
                                      └── ChatGPT 또는 Claude (GPTs/자동화)
```

### 업무별 추천 모델 매트릭스

| KDB생명 업무 | 1순위 모델 | 2순위(보완) | 이유 |
|------------|-----------|-----------|------|
| 인수심사 의견서 초안 | **Claude** | 사내 LLM (의료정보) | 복합 조건 준수 + 장문 보고서 |
| 보험시장 분석 리서치 | **Perplexity** | Claude (심층 분석) | 최신 데이터 + 출처 필수 |
| 보험약관 비교 분석 | **Claude** | Perplexity (최신 약관 검색) | 장문 문서 분석 + 구조화 |
| 손해율 악화 징후 분석 | **Claude** | 사내 LLM (보험계약 데이터) | 깊이 있는 다단계 분석 |
| 임원 보고 Executive Summary | **Claude** | ChatGPT (대화형 수정) | 정밀한 형식 준수 |
| 보험업 규제 동향 확인 | **Perplexity** | Gemini (Google 검색) | 최신 법령·출처 추적 |
| 해외 보험사 사례 조사 | **Perplexity** | Gemini (다국어) | 글로벌 실시간 검색 |
| 건강검진 결과·의료차트 분석 | **Gemini** | ChatGPT | 멀티모달 처리 |
| 보험계약자 의료정보 분석 | **사내 LLM** | — | 보안 필수 |
| 상품 아이디어 브레인스토밍 | **ChatGPT** | Claude | 폭넓은 기능, 대화형 |

> 🔑 핵심: 하나의 업무에 하나의 모델만 써야 하는 것이 아니다. **리서치는 Perplexity → 분석·작성은 Claude → 의료정보·계약정보는 사내 LLM**처럼 단계별로 모델을 조합하는 것이 가장 효과적이다.

## 4. 사내 LLM 사용 시 고려사항

### 왜 사내 LLM이 중요한가

KDB생명은 보험계약자의 의료정보·건강정보, 미출시 상품 설계서, 인수심사 기록, K-ICS 내부 산출 데이터 등 고도의 민감 정보를 다룬다. 이런 정보는 외부 AI에 입력할 수 없으므로, 사내 LLM의 역할이 필수적이다.

| 구분 | 외부 AI 사용 가능 | 사내 LLM 필수 |
|------|------------------|-------------|
| 언더라이팅 | 공개 보험의학 가이드라인, 질병 통계 프레임 | 보험계약자 의료정보, 인수심사 기록 |
| 자산운용 | 공개 시장 동향, 투자 방법론 | 투자 검토 중인 건, 내부 포트폴리오 분석 |
| 상품개발 | 경쟁사 공개 상품 비교, 시장 트렌드 | 미출시 상품 설계서, 내부 손해율 데이터 |
| 리스크관리 | K-ICS 제도 일반론, 공개 규제 정보 | K-ICS 내부 산출 데이터, CSM 상세 분석 |

### 사내 LLM 프롬프트 4대 원칙

사내 LLM은 외부 모델보다 모델 크기가 작고 학습 데이터가 제한적이다. 이를 보완하기 위한 프롬프트 원칙은 다음과 같다.

| 원칙 | 방법 | 이유 |
|------|------|------|
| **1. 더 상세한 맥락** | 외부 AI보다 배경 설명을 2배 풍부하게 | 자체 추론할 배경 지식이 제한적 |
| **2. Few-shot 적극 활용** | 기존 인수심사 의견서·보고서를 예시로 2~3개 첨부 | 패턴 학습으로 양식·톤 맞춤 가능 |
| **3. 출력 형식 확실히** | 표 구조, 칼럼명, 분량까지 명시 | 형식 이탈 시 재작업 비용이 큼 |
| **4. 한 번에 하나씩** | 복합 요청보다 단일 작업으로 분리 | 다중 태스크 시 품질 저하 가능 |

### 외부 AI와 사내 LLM의 역할 분담 예시

```
[1단계] 외부 AI (Perplexity/Claude)
  - 공개 정보 기반 보험시장 분석 프레임 생성
  - 가명 처리한 데이터로 분석 구조 설계
  - 규제·시장 동향 리서치

[2단계] 사내 LLM
  - 실제 보험계약자 의료정보를 대입하여 인수심사 완성
  - 내부 보험계약 포트폴리오 데이터로 손해율 분석
  - K-ICS 내부 산출 데이터와 교차 분석

[3단계] 담당 전문가
  - AI 결과물 팩트 검증
  - 전문적 판단 추가
  - 최종 보고서 완성
```

> ⚠️ 주의: 사내 LLM이라도 모든 데이터를 무분별하게 입력해서는 안 된다. 사내 LLM 접근 권한 관리, 로그 기록, 데이터 분류에 따른 입력 가부 판단은 여전히 필요하다.

## 학습 정리

### 핵심 요약
- **ChatGPT**: 가장 폭넓은 기능의 범용 모델, 브레인스토밍과 대화형 개선에 강점
- **Claude**: 장문 분석, 복합 조건 준수, 보고서 작성에 가장 적합 — KDB생명 인수심사·분석 업무의 1순위
- **Gemini**: 멀티모달과 추론에 강점, 이미지·의료차트 포함 분석에 적합
- **Perplexity**: 실시간 검색 + 출처 제시의 핵심 도구, 리서치·팩트체크에 필수
- **사내 LLM**: 의료정보·계약정보 등 기밀 데이터를 다루는 업무에 유일한 선택지, 프롬프트를 더 상세하게 작성해야 함
- 하나의 모델에 의존하지 말고, **업무 단계별로 모델을 조합**하는 것이 가장 효과적이다

### 즉시 적용 체크리스트
- [ ] 현재 자주 하는 업무 3가지에 대해 최적 LLM을 선택해본다
- [ ] 의사결정 플로차트(의료정보·계약정보 포함 여부 → 최신 정보 필요 → 업무 유형)를 업무에 적용한다
- [ ] 리서치 업무에 Perplexity를 한 번 사용하여 출처 기반 검색의 효과를 체험한다
- [ ] 사내 LLM 사용 시 4대 원칙(상세 맥락, Few-shot, 형식 명시, 단일 작업)을 적용한다
- [ ] 보험계약자 의료정보·건강정보는 반드시 사내 LLM만 사용하고, 외부 AI 사용 시 가명 처리를 준수한다
