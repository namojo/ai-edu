## 교육 목표

1. 생성형 AI를 **KDB 실제 업무**(여신심사, 산업분석, PF, M&A, ESG, 시장동향) 에 즉시 적용할 수 있는 프롬프트·워크플로 역량 확보
2. 제조/일반 예시가 아닌, **KDB 심사역·애널리스트·딜 매니저의 일상 업무 흐름**을 그대로 AI에 탑재
3. 금융권 특유의 **보안·컴플라이언스**(신용정보법, 전자금융감독규정, 사내LLM 가이드라인) 판단 기준 체득

## 모듈 1 — 프롬프트 엔지니어링 기초와 실전

### 학습 목표
- 프롬프트 실패 3원인(모호한 지시 / 맥락 부재 / 형식 미지정)을 진단
- 5대 기법(Zero-shot / Few-shot / Role-play / CoT / 구조화 출력)을 상황별로 선택
- 5단계 프롬프트 체크리스트로 즉시 업무에 적용

### KDB 실무 적용 포인트
- **Zero-shot**: 회의록·이메일 요약, 시장 동향 문구 번역
- **Few-shot**: 심사의견서 양식 통일, 기업 분석 리포트 패턴 학습
- **Role-play**: "20년차 기업여신 심사역", "PF 심사역", "산업분석 수석 애널리스트" 역할 부여
- **CoT**: 재무 → 업종 → 담보 → 종합 등급 단계별 추론 / PF 수익모델 → CF → 민감도 → DSCR
- **구조화 출력**: 리스크 매트릭스, 신용등급 비교표, 심사의견/리스크/조건/등급 매트릭스

### Before/After 대표 사례
- ❌ Before: "A기업 대출 심사해줘" → 교과서적 일반론
- ✅ After: 역할부여 + CoT + 구조화 출력 + Few-shot(과거 유사 여신 심사 양식) → 즉시 승인절차에 쓸 수 있는 심사의견서

### 5단계 프롬프트 체크리스트
1. **역할** — 누구의 관점에서 쓸 것인가 (심사역/RM/애널리스트)
2. **맥락** — 기업명·업종·여신금액·담보·경쟁사·규제를 모두 주입했는가
3. **지시** — 모호한 "분석"이 아니라 구체적인 "OO 관점에서 X를 Y 기준으로 비교"인가
4. **형식** — 표/리스트/매트릭스 등 받고 싶은 모양을 명시했는가
5. **검증** — 결과의 수치·법규·고유명사를 1차 검증할 루틴이 있는가

## 모듈 2 — AI/LLM 종류별 특성과 활용법

### 학습 목표
- ChatGPT / Claude / Gemini / Perplexity / 사내LLM 각각의 강점·한계를 실무 기준으로 비교
- 업무 유형별 최적 모델을 선택하는 판단 프레임 체득
- AI 에이전트·MCP의 개념과 금융권 적용 가능성

### 모델별 KDB 활용 가이드

| 모델 | 강점 | KDB 대표 활용 | 주의 |
|------|------|--------------|------|
| **ChatGPT** | 범용성·플러그인 | 초안 작성, 아이디어 발산 | 고객·기업 비공개 정보 입력 금지 |
| **Claude** | 장문 처리·논리성 | M&A DD 체크리스트, 여신 심사의견서 초안 | 한국어 실무 용어는 프롬프트로 강화 |
| **Gemini** | 검색 연동·멀티모달 | 시각 자료 해석, 최신 뉴스 요약 | 할루시네이션 검증 필수 |
| **Perplexity** | 출처 표시 | 산업분석 자료 수집, 해외 규제 요약 | 출처 원문 직접 대조 |
| **사내 LLM(KDB)** | 보안·내부자료 연결 | 여신 포트폴리오 분석, 내부 보고서 RAG | 외부 LLM 대비 성능 차이 — 용도 분리 |

### 금융권 보안 고려사항
- **신용정보법**: 개인신용정보·기업신용정보는 외부 LLM 입력 금지
- **전자금융감독규정**: 금융권 클라우드·AI 활용 가이드라인 준수
- **KDB 내부통제**: 미공개 딜 정보·심사 중 정보는 사내LLM 한정

### AI 에이전트 활용 예시
- **시장동향 수집 에이전트**: Perplexity·Bloomberg·한은 ECOS를 순회하며 일일 브리핑 자동화
- **여신심사 자동화 에이전트**: DART 공시 → 재무제표 추출 → KIS/NICE 등급 → 심사메모 초안

## 모듈 3 — 보고서 작성 실전 가이드

### 학습 목표
- AI 보고서 5단계 워크플로(목적 정의 → 자료 수집 → 초안 작성 → 검증 → 다듬기) 체화
- KDB 업무별 보고서 템플릿 6종(여신·시장·산업·PF·리스크·포트폴리오)의 Few-shot 활용
- 반복 보고서(일일·주간·월간)의 자동화 워크플로 설계

### KDB 보고서 템플릿 6종

| 유형 | 용도 | 핵심 섹션 |
|------|------|----------|
| 일일 시장동향 브리핑 | 경영진 아침 보고 | 금리·환율·회사채 스프레드·주요 뉴스 |
| 여신 심사보고서 | 기업여신 승인 | 재무 / 업종 / 담보 / 리스크 / 종합등급 |
| 주간 딜 파이프라인 | IB·PF 팀 조율 | 진행 단계 / 이슈 / 다음 액션 |
| 월간 여신 포트폴리오 리뷰 | 리스크 관리 | 업종별 비중 / NPL / 등급 변동 / 집중도 |
| 산업분석 리포트 | 산업별 전망 | 시장규모 / 성장률 / 리스크 / KDB 전략 |
| PF 타당성 보고서 | 프로젝트 승인 | 수익모델 / CF / DSCR / 리스크 매트릭스 |

### Before/After 대표 사례
- ❌ Before: "오늘 시장 상황 정리해줘" → 잡다한 헤드라인 나열
- ✅ After: "당신은 KDB 시장분석 담당자. 2026-04-23 기준 ① 한미 기준금리 ② 원/달러 환율 ③ 5년 회사채 AA 스프레드 ④ 전일 미국 고용지표 반응을 각각 3줄 이내, 수치-변동-KDB 여신/투자 영향 순으로 정리" → 승인자에게 즉시 전달 가능한 브리핑

### 자동화 워크플로
- Few-shot으로 사내 양식 고정 → LLM이 매일 동일 구조로 생성 → 담당자는 수치 검증만 수행

## 모듈 4 — 리서치와 자료 검색

### 학습 목표
- AI 검색의 한계를 이해하고, 1차 자료(원문)와 2차 자료(요약)를 구분
- KDB 업무에 필요한 전문 자료 소스를 분야별로 숙지
- 법률·규제 검색에서 AI가 범하기 쉬운 오인용을 방지

### KDB 필수 전문 자료 소스

| 영역 | 1차 자료 (원문) | 2차 자료 (요약·리서치) |
|------|---------------|----------------------|
| **기업 공시** | DART 전자공시, 반기/사업보고서 | FnGuide, NICE 평가정보, KIS Value |
| **거시 경제** | 한국은행 ECOS, 통계청 KOSIS | 산업연구원 KIET, KDB 미래전략연구소 |
| **시장 데이터** | Bloomberg, Refinitiv, WIND | CEIC, BNEF, IEA |
| **금융 규제** | 법제처 국가법령정보, 금감원 전자공시 | 금융연구원, 자본시장연구원 |
| **국제 규제** | BIS, FSB, IMF WEO | SSRN, NBER, 한국금융학회 |
| **산업 규제** | K-Taxonomy, TCFD, IFRS 재단 | MSCI ESG, 서스틴베스트, KCGS |

### AI 검색 도구별 전략
- **Perplexity**: 출처가 표시되므로 1차 자료 탐색에 유리. 답변의 링크를 반드시 원문 확인
- **Claude/ChatGPT**: 최신 자료에 약함 → 사용자가 자료를 첨부한 뒤 요약 지시
- **사내 LLM(KDB) + RAG**: 내부 심사보고서·여신 DB·ECOS 미러를 연결해 "우리 은행 맥락"의 답변 생성

### Before/After 대표 사례
- ❌ Before: "해상풍력 PF 외환·바젤·제재 규제 알려줘" → 법령명 혼동, 출처 없음
- ✅ After: "2026년 4월 기준, 해상풍력 PF 해외 투자 시 ① 외국환거래법 ② 바젤 III ③ 대러/대이란 제재 각각의 핵심 조항과 원문 링크를 표로" → 원문 검증 가능한 결과

### 법률·규제 검색 체크리스트
1. AI 답변의 법령명·조항 번호를 **법제처 원문**과 대조
2. "최근 개정" 주장은 반드시 **공포일자** 확인
3. 판례 인용 시 **대법원 종합법률정보**에서 사건번호 검색

## 모듈 5 — 할루시네이션 방지와 팩트체크

### 학습 목표
- 금융권 업무에서 할루시네이션이 야기하는 실제 리스크 사례 이해
- 팩트체크 3단계(수치 / 출처 / 규제) 루틴화
- RAG·사내LLM으로 할루시네이션을 구조적으로 줄이는 기술 이해

### KDB 업무에서 빈번한 할루시네이션 유형

| 유형 | 예시 | 탐지 방법 |
|------|------|----------|
| **수치 왜곡** | "A기업 신용등급 AA" (실제 A) / "DSCR 1.5" (실제 1.2) / "유동비율 180%" (실제 145%) | DART·KIS/NICE 원문 대조 |
| **존재하지 않는 법령·조항** | "자본시장법 제300조 ○항" (실존 안 함) | 법제처 검색으로 조항 유무 즉시 확인 |
| **허위 판례·사건번호** | 실제로 없는 판결문 인용 | 대법원 종합법률정보 사건번호 조회 |
| **잘못된 규제 연도** | "바젤 IV 2026년 전면 시행" (실제 일정 상이) | BIS 공식 일정표 확인 |
| **기업명·직함 혼동** | 대표이사 변경 전 이름 반복 | DART 최근 공시 대조 |

### 팩트체크 3단계
1. **수치** — AI가 제시한 모든 숫자는 DART·ECOS·Bloomberg 등 **1차 원본**과 대조
2. **출처** — "출처: Bloomberg 2026-04-15" 같은 주장은 URL·기사 ID까지 검증
3. **규제** — 법령명·조항 번호는 **법제처 국가법령정보**로 재확인

### 할루시네이션 제거 체크리스트
- [ ] 기업명·대표자명·사업자번호를 DART와 대조했는가
- [ ] 재무제표 수치를 반기/사업보고서 원문과 매칭했는가
- [ ] 신용등급은 KIS·NICE 공식 리포트로 확인했는가
- [ ] 법령 조항은 법제처로 재확인했는가
- [ ] 시장 데이터(금리·환율·스프레드)는 한은·Bloomberg 스냅으로 검증했는가

### 기술적 대응 — 사내LLM + RAG
- KDB 사내 LLM이 내부 심사보고서·여신 DB·ECOS 미러링·내규를 RAG로 연결
- "OO기업 최근 3년 재무지표"를 물으면 → DART 대신 **내부 DB 스냅샷**에서 응답 → 허위 생성 원천 차단

### 금융권 보안·컴플라이언스 원칙
- **외부 LLM 입력 금지**: 미공개 딜 정보, 고객 신용정보, 내부 심사 메모
- **사내 LLM 전용**: 기업 실명을 포함한 여신 포트폴리오 분석
- **마스킹 후 외부 LLM 허용**: 공개 공시 데이터의 해석·초안 작성

## 모듈 6 — KDB 실무 3대 시나리오 (종합 적용)

모듈 1~5에서 배운 모든 기법(역할부여 · 구조화 · CoT · Few-shot · 할루시네이션 방지 · 팩트체크)을 **6단계 구조**(상황 → Before → After → 기법 해설 → 검증 포인트 → 보안 판단)로 통합 적용합니다.

### 시나리오 1 — 기업여신 심사 리포트 작성

**상황**: KDB 여신심사부 심사역. 중견기업 A사(자동차 부품 업종)의 시설자금 500억원 대출 심사. 3년 재무제표·담보(공장부지+건물+기계설비)·업종을 근거로 2일 내 심사보고서 제출.

**적용 기법**:
- Role-play: "20년차 기업여신 심사역"
- CoT: 재무 → 업종 → 담보 → 종합등급
- 구조화 출력: 심사의견 / 리스크 / 조건 / 등급 매트릭스
- Few-shot: 과거 유사 여신 심사 양식 주입

**검증 포인트**:
- DART 반기/사업보고서와 재무 수치 대조
- KIS·NICE 신용등급 추이
- 담보 감정평가서 원문

**보안 판단**:
- 기업명·대표자명 등 민감정보는 **사내 LLM** 또는 **마스킹 후 외부 LLM**
- 심사 중 정보는 승인 공시 전까지 외부 입력 금지

---

### 시나리오 2 — 산업분석 리포트 (2차전지 2026년 전망)

**상황**: KDB 산업분석부 수석 애널리스트. IRA/EU CBAM 영향, 셀 3사(LG엔솔·SK온·삼성SDI) 2026년 설비투자 계획, 원자재 리스크를 포함한 2차전지 전망 리포트를 2주 내 작성.

**적용 기법**:
- Role-play: "산업분석 수석 애널리스트"
- 워크플로: **Perplexity로 최신 데이터 수집** → **Claude로 구조화 리포트 초안** → **3단계 팩트체크**
- 구조화 출력: 시장규모 / 성장률 / 리스크 매트릭스 / KDB 여신·투자 전략

**검증 포인트**:
- 한국전지산업협회 공식 통계
- 기업 공시(DART) — CAPEX 수치 대조
- 해외 원문: BNEF, IEA World Energy Outlook
- 규제 원문: IRA(미 재무부 가이던스), CBAM(EU Commission)

**보안 판단**:
- 공개 데이터는 외부 LLM 허용
- KDB 내부 여신 포트폴리오 데이터(셀 3사 여신 한도·잔액)는 **사내 LLM 전용**

---

### 시나리오 3 — 프로젝트파이낸스(PF) 타당성 검토 (해상풍력 500MW)

**상황**: KDB 프로젝트금융부 심사역. 총사업비 2.5조원, 자본구조(에쿼티 30% + 부채 70%), 20년 PPA, 정부 REC 정책 기반의 국내 해상풍력 500MW PF 참여 검토. DSCR/LLCR 분석, 정책·건설·운영 리스크 평가 심사보고서 필요.

**적용 기법**:
- Role-play: "PF 심사역, 15년 경력"
- CoT: 수익모델 → 현금흐름 추정 → 민감도 → DSCR/LLCR
- 구조화 출력: 리스크 매트릭스 (정책/건설/운영/환/원자재)
- Few-shot: KDB 과거 해상풍력 유사 딜 심사 양식

**검증 포인트**:
- 한국에너지공단 RPS 의무비율·REC 단가 공시
- 유사 PF 딜 DSCR 벤치마크 (KDB 내부 DB)
- 환리스크 (수입 터빈 가격 변동, 유로/달러 환율)
- 해상풍력 운영비(O&M) 글로벌 벤치마크

**보안 판단**:
- 프로젝트 사업자명·입찰 정보는 **사내 LLM 전용**
- 시장 공지 전까지 외부 LLM 사용 금지
